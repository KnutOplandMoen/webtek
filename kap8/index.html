01 · Helhetsbilde

Hvorfor forms?

Skal skrives

Forms er lim mellom bruker og tjener. Login, søk, registrering, oppgavelevering — alt går gjennom et skjema.

Kilde: pensum 1.9, slides presentasjoner/forms.md.

02 · Mental modell

Tre roller

Skal skrives

Form-elementet (containeren), input-feltene (data), submit-knappen (handlingen). action og method (URL og HTTP-verb) avgjør hvor og hvordan data sendes.

03 · Reisen

Bygg et kontaktskjema

Skal skrives

Stegvis: form med action og method → label + input(text) → input(email) med required → textarea → button(submit). Vis HTML5-validering ved tom innsending.

05 · Begreper

Nøkkelbegreper

action
URL data sendes til.
method
GET (i URL) vs POST (i body).
name-attributt
Hvordan inputen identifiseres på tjeneren.
label-for
Knytter etikett til input — viktig for tilgjengelighet.
required
HTML5-validering.
input-typer
text, email, url, number, date, password, checkbox, radio, file.
placeholder
Hint, ikke verdi.
pattern
Regex-validering på input.

08 · Vanlige feller

Vanlige feller

Skal skrives

Glemme name-attributt (input havner ikke i submit-data); bruke placeholder som label; tro at klientside-validering er sikkerhet (det er det ikke); blande GET og POST.

Skal skrives
