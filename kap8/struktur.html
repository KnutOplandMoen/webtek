01 · fieldset, legend, label

Gruppering og labels

Pensum 1.9.2 starter med tre elementer som strukturerer skjemaet uten å samle data selv: <fieldset>, <legend> og <label>.

A · fieldset
<fieldset>…</fieldset>

Grupperer relaterte kontroller i skjemaet. Typiske grupper: alle radioknappene som hører sammen, eller alle adressefeltene (navn, gate, by, postnummer, land).

B · legend
<legend>…</legend>

Beskriver formålet til en <fieldset>. Skal være første barn i fieldset-en. Skjermlesere bruker legenden som kontekst når de leser opp widgetene inne i gruppen.

C · label
<label for="id">…</label>

Etiketten til én kontroll. Beskriver hva feltet skal inneholde (f.eks. «E-post»). To måter å koble label til input:

  1. Med for-attributtet: <label for="navn"> peker på input-ens id.
  2. Wrap-stilen: Skriv input-en inne i label-elementet. Da trengs ikke for.

Begge gir samme effekt: brukeren kan klikke på label-teksten for å aktivere kontrollen, og skjermlesere leser label-teksten når feltet får fokus.

Hvorfor for er viktig på radio og checkbox

Radio-knapper og avkrysningsbokser har et lite klikkmål — den runde sirkelen eller den lille firkanten. Med riktig label kan brukeren klikke hvor som helst på label-teksten i stedet, noe som er mye lettere å treffe (særlig på touch-skjerm).

HTML — adresseskjema med fieldset, legend og labels
<form>
  <fieldset>
    <legend>Hva er adressen din?</legend>

    <label for="fullname">
      Fullt navn:
      <input type="text" id="fullname" name="address">
    </label>
    <br>
    <label for="street">
      Gate:
      <input type="text" id="street" name="address">
    </label>
    <br>
    <label for="city">
      By:
      <input type="text" id="city" name="address">
    </label>
    <label for="zip">
      Postnr:
      <input type="number" id="zip" name="address">
    </label>
  </fieldset>
</form>
Sjekk · Lett
Hvor i en <fieldset> skal <legend> stå?
  • A Som første barn av fieldset-en
  • B Etter alle input-feltene
  • C Utenfor fieldset, rett over
  • D Hvor som helst — rekkefølgen spiller ingen rolle
Riktig: A. <legend> skal være første barn så skjermlesere kan introdusere gruppen før widgetene leses opp.
02 · output

output-elementet

<output> brukes til å vise resultatet av en beregning — gjerne basert på verdier fra andre input-felt. Du kan bruke for-attributtet til å liste opp ID-ene til kontrollene som bidrar til beregningen, atskilt med mellomrom.

Skatt beregnes fra inntekt
<output for="income tax"></output>

Selve beregningen må du gjøre i JavaScript — <output> er bare beholderen for resultatet. for-attributtet er semantisk: det forteller hjelpemidler hvilke felter som er relevante, uten å automatisk beregne noe.

03 · input

input — kjernen i skjemaet

<input> er det viktigste elementet i et skjema — kilden til informasjon fra brukeren. Elementet kan endre seg radikalt avhengig av hvilken verdi type-attributtet får.

Et enkelt tekstfelt
<input type="text" name="street" id="street">
Fallback til text

Hvis nettleseren ikke støtter en type-verdi, faller kontrollen tilbake til type="text". Et felt med type="email" i en gammel nettleser oppfører seg som et vanlig tekstfelt — uten format-validering, men brukeren kan fortsatt fylle det ut.

<input> er et void-element: det har ingen sluttag og kan ikke ha innhold. All konfigurasjon skjer via attributter.

04 · type-attributtet

Alle input-typer

Pensum 1.9.2 grupperer type-verdiene i fire kategorier. Tabellen under er en oppsummering — kolonnen «Relevante attributter» viser hvilke ekstra attributter som har effekt for hver type.

A · Tekstfelt på én linje

text
Vanlig tekstfelt. Attr: autocomplete, maxlength, pattern, list, placeholder, required.
email
Felt for én eller flere e-poster. Attr: autocomplete, list, maxlength, multiple, pattern, placeholder, required.
password
Maskerer tegnene som prikker. Attr: autocomplete, list, maxlength, required.
search
Felt for søkestrenger. Attr: autocomplete, list, maxlength, pattern, placeholder, required.
tel
Telefonnummer. Attr: autocomplete, list, maxlength, pattern, placeholder, required.
url
Felt for absolutt URL. Attr: autocomplete, list, maxlength, pattern, placeholder, required.

B · Kontroller uten tekst-input

checkbox
Avkrysningsboks. Attr: checked, required.
file
Velg lokal fil. Attr: accept, multiple, required.
number
Tall, eksakt verdi viktig. Attr: autocomplete, list, min, max, required, step.
radio
Bare én radio i samme gruppe kan velges. Attr: checked, required.
range
Glidebryter — eksakt verdi ikke viktig. Attr: autocomplete, list, min, max, required, step.

C · Tid og dato

date
År, måned, dag. Attr: autocomplete, list, min, max, required.
time
Klokkeslett. Attr: autocomplete, list, min, max, required.

D · Knapper

button
Trykknapp uten standardatferd. Trenger JavaScript for å gjøre noe.
reset
Tilbakestiller alle felt i skjemaet til standardverdiene.
submit
Sender skjemaet til action-URL-en.
image
Submit-knapp som er et bilde. Trenger src og alt.
Sjekk · Middels
Forskjellen på type="number" og type="range"?
  • A Begge er identiske, bare visuelt forskjellige
  • B number brukes når eksakt verdi er viktig; range når den ikke er det (glidebryter)
  • C range tillater bare heltall, number tillater desimaler
  • D number sender ikke verdi til serveren
Riktig: B. Pensum: «range — A control for entering a number whose exact value is not important.»
Sjekk · Lett
Hva skjer hvis nettleseren ikke kjenner igjen type="email"?
  • A Skjemaet feiler å rendres
  • B Feltet blir helt usynlig
  • C Det fallback-er til type="text"
  • D Det blir automatisk konvertert til type="search"
Riktig: C. Pensum: «If the browser doesn't support the value in the type attribute, the input field will be rendered as a text input.»
05 · Attributter

Attributter med eksempler

Hvert attributt under er en del av pensum 1.9.2. Eksemplene viser typisk bruk og hvilke type-verdier attributtet er relevant for.

accept

For type="file": hvilke filtyper aksepteres. Verdien er en kommaseparert liste av filendelser eller MIME-typer. Filendelser starter med .; MIME-typer kan være spesifikke eller bruke * for hele kategorier (image/*, audio/*, video/*).

Bare bilder
<label for="file">Velg fil:
  <input type="file" name="file"
         id="file" accept="image/*">
</label>
JPG eller PNG
<label for="file">Velg fil:
  <input type="file" name="file"
         id="file" accept=".jpg, .png">
</label>
autocomplete

Indikerer om nettleseren kan auto-fullføre verdien (med tidligere verdier brukeren har skrevet inn).

E-post med autocomplete
<label for="e-mail">E-post:
  <input type="email" name="e-mail"
         id="e-mail" autocomplete>
</label>
checked

Boolean. For type="radio" eller type="checkbox": kontrollen er forhåndsvalgt. Ignoreres for andre typer.

Forhåndsavkrysset
<label for="alligator">Er du en alligator?
  <input type="checkbox" name="alligator"
         id="alligator" checked>
</label>
list

Identifiserer en liste av forslag som skal vises til brukeren. Verdien er ID-en til et <datalist>-element i samme dokument. Brukes typisk til autofullføring.

Foreslå språk
<label for="language">Foretrukket språk:
  <input type="text" name="language" list="prog-lang">
</label>
<datalist id="prog-lang">
  <option value="Python"></option>
  <option value="JavaScript"></option>
  <option value="Java"></option>
  <option value="Elm"></option>
  <option value="Haskell"></option>
</datalist>
max

Maks (numerisk eller dato/tid) verdi for kontrollen. Må ikke være mindre enn min.

Klassetrinn maks 5
<label for="class">Hvilket trinn er du i?
  <input type="number" name="class" id="class" max="5">
</label>
maxlength

For tekst-typer (text, email, search, password, tel, url): maks antall tegn brukeren kan skrive.

E-post på maks 40 tegn
<label for="email">E-post:
  <input type="email" name="email"
         id="email" maxlength="40">
</label>
min

Min (numerisk eller dato/tid) verdi. Må ikke være større enn max.

Trinn minst 1
<label for="class">Trinn:
  <input type="number" name="class" id="class" min="1">
</label>
multiple

Boolean. Lar brukeren legge inn flere verdier. Gjelder type="email" (flere e-postadresser, kommaseparert) og type="file" (flere filer i samme valg). Også gyldig på <select>.

Velg flere filer
<label for="file">Velg filer:
  <input type="file" name="file" id="file" multiple>
</label>
pattern

Et regulært uttrykk (RegEx) som verdien sjekkes mot. Samme språk som JavaScript RegExp, med implisitt u-flagget. Pattern omsluttes ikke av skråstreker (/.../).

IATA-flyplasskode (3 bokstaver)
<label for="iata">Flyplasskode:
  <input type="text" name="iata-code" id="iata"
         pattern="[A-Za-z]{3}"
         title="Tre bokstaver IATA-kode">
</label>

title-attributtet vises som hint hvis brukeren legger inn noe som ikke matcher pattern.

placeholder

Et hint i feltet om hva som kan skrives. Forsvinner så snart brukeren begynner å skrive. Kan ikke inneholde linjeskift.

Hint på e-post
<label for="email">E-post:
  <input type="email" name="email" id="email"
         placeholder="ditt@domene.no">
</label>
Ikke erstatt label

Pensum understreker: placeholder erstatter ikke <label>. Label beskriver feltets rolle; placeholder er bare et hint om format. I noen tilfeller vises placeholder aldri (f.eks. ved autofullføring).

required

Boolean. Brukeren må fylle ut feltet før skjemaet kan submit-es.

Obligatorisk passord
<label for="password">Passord:
  <input type="password" name="password"
         id="password" required>
</label>
step

Sammen med min og max: tillatte intervaller for numeriske eller dato-verdier. Kan være strengen any eller et positivt flyttall. Hvis ikke any, godtas bare verdier som er multiple av step over min.

Inntekt i tusen-trinn
<label for="income">Inntekt:
  <input type="range" name="income" id="income"
         step="1000" min="0" max="100000">
</label>
name

Hver input bør ha et name. Det er navnet som brukes når data sendes med GET eller POST. Uten name sendes kontrollen ikke i det hele tatt.

Spesielt for radio

Radio-knapper i samme gruppe deler name. Hver knapp har sin egen value. Det som sendes er name=value for den valgte knappen.

Sjekk · Middels
Hva er forskjellen på placeholder og <label>?
  • A De gjør akkurat det samme
  • B Label beskriver feltets rolle (permanent etikett); placeholder er et hint som forsvinner når brukeren skriver
  • C Placeholder er obligatorisk, label er valgfri
  • D Label vises bare på mobil, placeholder bare på desktop
Riktig: B. Bruk begge — placeholder er et format-hint, label er etiketten som forklarer hva feltet er.
Sjekk · Vanskelig
På hvilke type-verdier er multiple gyldig?
  • A Bare text
  • B email og file
  • C Alle typer
  • D Bare checkbox og radio
Riktig: B. multiple gjelder type="email" (flere e-postadresser) og type="file" (flere filer). Også gyldig på <select>.
06 · textarea

textarea — flere linjer tekst

<textarea> er et tekstfelt med flere linjer. Det oppfører seg som type="text", men brukeren kan trykke Enter for å lage linjeskift.

Tre attributter for layout
Attributt Default Beskrivelse
cols 20 Synlig bredde, målt i gjennomsnittlig tegnbredde.
rows — Antall synlige tekstlinjer.
wrap soft Hvordan teksten brytes — soft eller hard.
Innhold i taggen, ikke i value

I motsetning til <input> (som er et void-element) er <textarea> et vanlig element med åpning og lukking. Standardverdi skrives mellom taggene, ikke som value-attributt. HTML/markup i innholdet rendres som ren tekst.

Kommentar-tekstområde
<form>
  <label for="text">Skriv din historie:</label>
  <textarea name="text" id="text"
            rows="8" cols="80" wrap="soft">
Dette er min historie, og den handler om webteknologier!
  </textarea>
</form>
Sjekk · Middels
Hvor skriver du standardverdien til en <textarea>?
  • A I value-attributtet
  • B Mellom åpnings- og lukketagg
  • C I placeholder-attributtet
  • D I et eget <default>-element inni
Riktig: B. Textarea har lukketag — innhold mellom taggene blir standardverdien. HTML rendres som ren tekst.
07 · select, option, optgroup

select — nedtrekkslister

<select> lager en nedtrekksboks der brukeren velger blant forhåndsdefinerte verdier. Hver verdi er et <option>-element. <optgroup> grupperer relaterte options.

Velg programmeringsspråk
<form>
  <label for="language">Velg språk</label>
  <select id="language" name="language">
    <option value="python">Python</option>
    <option>JavaScript</option>
    <optgroup label="Funksjonelle">
      <option>Haskell</option>
      <option>Elm</option>
    </optgroup>
  </select>
</form>
Hva sendes ved submit?
  • Hvis <option> har et value-attributt: verdien sendes.
  • Hvis value mangler: tekstinnholdet til option-en sendes som verdi.

I eksempelet over: hvis brukeren velger «Python» sendes language=python; hvis hen velger «JavaScript» sendes language=JavaScript.

optgroup-label kan ikke velges

label-attributtet på <optgroup> vises som overskrift over options-ene i gruppen, men det er ikke selv et valg. Brukeren kan bare velge en <option>.

Sjekk · Lett
Hva sendes hvis et <option> ikke har value-attributt?
  • A Tomme strenger
  • B Tekstinnholdet til option-en
  • C Indeksen til option-en (0, 1, 2, …)
  • D Ingenting — option-en sendes ikke
Riktig: B. Tekstinnholdet brukes som fallback-verdi når value mangler.
08 · Vanlige feller

Vanlige feller

Felle 1 — glemt name

En kontroll uten name sendes ikke ved submit. Brukeren kan fylle ut feltet, men det havner ikke i form-dataen. Den vanligste «hvorfor mangler dette feltet?»-feilen.

Felle 2 — radio uten samme name

Radio-knapper i samme valg-gruppe må dele name. Med ulike navn oppfører de seg som uavhengige felt — brukeren kan velge flere samtidig, og hver sendes som et eget felt.

Felle 3 — pattern med skråstreker

I HTML pattern-attributtet skal du ikke ha skråstreker rundt verdien: skriv pattern="[A-Za-z]{3}", ikke pattern="/[A-Za-z]{3}/". ^ og $ er heller ikke nødvendig — pattern sjekkes mot hele verdien.

Felle 4 — textarea med value-attributt

<textarea value="Hei"></textarea> gir et tomt tekstområde. Standardverdi skal stå mellom taggene: <textarea>Hei</textarea>.

Felle 5 — label uten kobling

En <label> uten for og uten å omslutte input-en er bare tekst. Klikk på den gjør ingenting, og skjermlesere kobler den ikke til feltet.

09 · Test deg selv

Multiple choice

1 · Lett
Hvilket element grupperer relaterte form-kontroller?
  • A <section>
  • B <group>
  • C <fieldset>
  • D <form-group>
Riktig: C. <fieldset> grupperer kontroller; <legend> beskriver gruppen.
2 · Lett
Hvilken type maskerer tegnene som prikker?
  • A type="hidden"
  • B type="password"
  • C type="secret"
  • D type="text" + masked
Riktig: B. type="password" maskerer input visuelt.
3 · Middels
Hvordan grupperer du radio-knapper slik at bare én kan velges?
  • A Pakk dem i en <fieldset>
  • B Gi alle samme name-attributt
  • C Gi alle samme id
  • D Sett type="radio-group"
Riktig: B. Felles name binder dem som en gjensidig utelukkende gruppe. Hver knapp har egen value.
4 · Middels
Hvordan skriver du pattern-verdien for «nøyaktig tre bokstaver»?
  • A pattern="/^[A-Za-z]{3}$/"
  • B pattern="[A-Za-z]{3}"
  • C pattern="^[A-Za-z]{3}$"
  • D pattern="\w{3}"
Riktig: B. Pensum: «The pattern is not surrounded by forward slashes.» Pattern matches hele verdien — ingen ^/$ nødvendig.
5 · Middels
Hvilken kombinasjon gir en glidebryter med min/max og 1000 i steg?
  • A type="number" step="1000"
  • B type="range" min="0" max="100000" step="1000"
  • C type="slider" interval="1000"
  • D type="range" interval="1000"
Riktig: B. range = glidebryter, step styrer tillatte intervaller mellom min og max.
6 · Vanskelig
På et <option>-element: når brukes tekstinnholdet som submit-verdi?
  • A Alltid — value-attributtet ignoreres
  • B Når value-attributtet mangler
  • C Bare hvis option-en er inne i en <optgroup>
  • D Aldri — det må alltid være en value
Riktig: B. Hvis value er satt brukes den; ellers fallback-er det til tekstinnholdet.
7 · Vanskelig
Du har <textarea> uten cols og rows satt. Hva er standard bredde?
  • A Hele bredden av forelderen
  • B 20 tegn (cols-default)
  • C 80 tegn
  • D Ingen bredde — feltet blir usynlig
Riktig: B. Pensum-tabellen: cols har default 20.
