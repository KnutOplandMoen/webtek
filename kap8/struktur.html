01 · fieldset, legend, label

Gruppering og labels

Forelesningens slides om struktur starter med tre elementer som strukturerer skjemaet uten å samle data selv: <fieldset>, <legend> og <label>.

A · fieldset
<fieldset>…</fieldset>

Grupperer relaterte kontroller i skjemaet. Typiske grupper: alle radioknappene som hører sammen, eller alle adressefeltene (navn, gate, by, postnummer, land).

B · legend
<legend>…</legend>

Beskriver formålet til en <fieldset>. Skal være første barn i fieldset-en. Skjermlesere bruker legenden som kontekst når de leser opp widgetene inne i gruppen.

C · label
<label for="id">…</label>

Etiketten til én kontroll. Beskriver hva feltet skal inneholde (f.eks. «E-post»). To måter å koble label til input:

  1. Med for-attributtet: <label for="navn"> peker på input-ens id.
  2. Wrap-stilen: Skriv input-en inne i label-elementet. Da trengs ikke for.

Begge gir samme effekt: brukeren kan klikke på label-teksten for å aktivere kontrollen, og skjermlesere leser label-teksten når feltet får fokus.

Hvorfor for er viktig på radio og checkbox

Radio-knapper og avkrysningsbokser har et lite klikkmål — den runde sirkelen eller den lille firkanten. Med riktig label kan brukeren klikke hvor som helst på label-teksten i stedet, noe som er mye lettere å treffe (særlig på touch-skjerm).

HTML — adresseskjema med fieldset, legend og labels
<form>
  <fieldset>
    <legend>Hva er adressen din?</legend>

    <label for="fullname">
      Fullt navn:
      <input type="text" id="fullname" name="address">
    </label>
    <br>
    <label for="street">
      Gate:
      <input type="text" id="street" name="address">
    </label>
    <br>
    <label for="city">
      By:
      <input type="text" id="city" name="address">
    </label>
    <label for="zip">
      Postnr:
      <input type="text" id="zip" name="address">
    </label>
  </fieldset>
</form>
Sjekk · Lett
Hvor i en <fieldset> skal <legend> stå?
  • A Som første barn av fieldset-en
  • B Etter alle input-feltene
  • C Utenfor fieldset, rett over
  • D Hvor som helst — rekkefølgen spiller ingen rolle
Riktig: A. <legend> skal være første barn så skjermlesere kan introdusere gruppen før widgetene leses opp.
02 · input

input — kjernen i skjemaet

<input> er det viktigste elementet i et skjema — kilden til informasjon fra brukeren. Elementet kan endre seg radikalt avhengig av hvilken verdi type-attributtet får.

Et enkelt tekstfelt
<input type="text" name="street" id="street">
Fallback til text

Hvis nettleseren ikke støtter en type-verdi, faller kontrollen tilbake til type="text". Et felt med type="email" i en gammel nettleser oppfører seg som et vanlig tekstfelt — uten format-validering, men brukeren kan fortsatt fylle det ut.

<input> er et void-element: det har ingen sluttag og kan ikke ha innhold. All konfigurasjon skjer via attributter.

03 · type-attributtet

Input-typer fra forelesningen

Slides forms.md viser akkurat disse type-verdiene. Andre typer (som number, range, tel, time, reset) ble ikke gjennomgått, og er ikke i pensum.

A · Tekstfelt (Adding Text-slidene)

text
Vanlig énlinjes tekstfelt — slides nevner brukstilfeller som e-post, brukernavn osv.
password
«acts just like a single line input, except that the characters are hidden in the way that someone sees only dots».

B · Valg (Making Choices)

radio
«Radio button allow users to pick just one of the options.» Gruppen deler name; checked markerer forhåndsvalg.
checkbox
«checkboxes allow users to select (and deselect) one of more.» checked markerer forhåndsvalg.

C · «Creating Input Boxes»-slidene

image
«Displays an input image that can be clicked to perform an action.» Trenger src.
button
«Displays a button that can be clicked to perform an action from a script.»
file
«Displays a browse button to locate and select a file.» Krever method="post".
hidden
«Creates a hidden field, not viewable on the form.» Slides viser typisk bruk inni <button>-eksempelet.
submit
«the submit button is used to send form to the server.»

D · HTML5-typene (egne slides)

date
«If you are asking the user for a date, you can use an <input> element and give type="date"».
email
HTML5-felt for e-postadresser — nettleseren sjekker format.
url
HTML5-felt for nettsteds-adresse — nettleseren sjekker at det ligner en URL.
search
Søkefelt. Brukes typisk med placeholder.
Sjekk · Lett
Hva skjer hvis nettleseren ikke kjenner igjen type="email"?
  • A Skjemaet feiler å rendres
  • B Feltet blir helt usynlig
  • C Det fallback-er til type="text"
  • D Det blir automatisk konvertert til type="search"
Riktig: C. Nye HTML5-typer fallback-er til text hvis nettleseren ikke kjenner dem.
Sjekk · Middels
Hvilken kombinasjon brukes for å la brukeren laste opp en fil?
  • A type="upload"
  • B type="file" + method="post"
  • C type="file" + method="get"
  • D type="image"
Riktig: B. Slides forms.md: «Type="file" you can use only post method.»
04 · Attributter

Attributter med eksempler

Bare attributter foreleser viste — text-input-eksempelet med name, size, maxlength; HTML5-validering med required; placeholder; checked/selected; multiple.

name

Hver input bør ha et name. Slides sier: «the server needs to know the name of this information.» Uten name sendes ikke kontrollen.

Spesielt for radio

Radio-knapper i samme gruppe deler name. Hver knapp har sin egen value. Det som sendes er name=value for den valgte knappen.

size og maxlength

Slides forms.md viser begge på login-eksempelet: size="15" styrer synlig bredde i tegn, maxlength="30" setter maks antall tegn brukeren kan skrive.

Brukernavn med begrenset lengde
<input type="text" name="username"
       size="15" maxlength="30">
checked

Boolean. For type="radio" eller type="checkbox": kontrollen er forhåndsvalgt fra start. Slides bruker syntaksen checked="checked".

Forhåndsavkrysset radio
<input type="radio" name="genre"
       value="rock" checked="checked"> Rock
multiple (på select)

På <select multiple="multiple">: brukeren kan velge flere alternativer ved å holde Ctrl/Cmd. Slides advarer at nettlesere har «not very good support» for dette.

Velg flere instrumenter
<select name="instruments" multiple="multiple">
  <option value="guitar" selected="selected">Guitar</option>
  <option value="drums">Drums</option>
</select>
required

Boolean (slides bruker required="required"). Brukeren må fylle ut feltet før skjemaet kan submit-es — HTML5-validering.

Obligatorisk brukernavn
<input type="text" name="username"
       required="required">
placeholder

Slides: «placeholder, whose value will be shown in the text box until the user clicks in that area.» Forsvinner når brukeren begynner å skrive.

Søk med hint
<input type="search" name="search"
       placeholder="Enter keyword">
Ikke erstatt label

Placeholder er et hint, ikke en etikett. Slides legger vekt på at <label>-elementet er det som beskriver feltets rolle.

Sjekk · Middels
Hva er forskjellen på placeholder og <label>?
  • A De gjør akkurat det samme
  • B Label beskriver feltets rolle (permanent etikett); placeholder er et hint som forsvinner når brukeren skriver
  • C Placeholder er obligatorisk, label er valgfri
  • D Label vises bare på mobil, placeholder bare på desktop
Riktig: B. Bruk begge — placeholder er et format-hint, label er etiketten som forklarer hva feltet er.
Sjekk · Middels
På hvilke elementer kan checked brukes?
  • A På alle <input>-typer
  • B Kun på type="radio" og type="checkbox"
  • C Kun på <option>
  • D Kun på type="submit"
Riktig: B. checked markerer en radio eller checkbox som forhåndsvalgt. På <option> heter tilsvarende attributt selected.
05 · textarea

textarea — flere linjer tekst

<textarea> er et tekstfelt med flere linjer. Slides: «is multiline for longer areas of text». «This is not an empty element, it should therefore have an opening and closing tag.»

To attributter foreleser viste
Attributt Beskrivelse
cols Synlig bredde, målt i tegn (slides bruker cols="20").
rows Antall synlige tekstlinjer (slides bruker rows="4").
Innhold i taggen, ikke i value

I motsetning til <input> (som er et void-element) er <textarea> et vanlig element med åpning og lukking. Standardverdi skrives mellom taggene, ikke som value-attributt.

Kommentar-tekstområde (slides forms.md)
<form action="http://www.example.com/comments.php">
  <p>What did you think of this gig?</p>
  <textarea name="comments" cols="20" rows="4">
Enter your comments...
  </textarea>
</form>
Sjekk · Middels
Hvor skriver du standardverdien til en <textarea>?
  • A I value-attributtet
  • B Mellom åpnings- og lukketagg
  • C I placeholder-attributtet
  • D I et eget <default>-element inni
Riktig: B. Textarea har lukketag — innhold mellom taggene blir standardverdien. HTML rendres som ren tekst.
06 · select, option, optgroup

select — nedtrekkslister

<select> lager en nedtrekksboks der brukeren velger blant forhåndsdefinerte verdier. Hver verdi er et <option>-element. <optgroup> grupperer relaterte options.

Velg programmeringsspråk
<form>
  <label for="language">Velg språk</label>
  <select id="language" name="language">
    <option value="python">Python</option>
    <option>JavaScript</option>
    <optgroup label="Funksjonelle">
      <option>Haskell</option>
      <option>Elm</option>
    </optgroup>
  </select>
</form>
Hva sendes ved submit?
  • Hvis <option> har et value-attributt: verdien sendes.
  • Hvis value mangler: tekstinnholdet til option-en sendes som verdi.

I eksempelet over: hvis brukeren velger «Python» sendes language=python; hvis hen velger «JavaScript» sendes language=JavaScript.

optgroup-label kan ikke velges

label-attributtet på <optgroup> vises som overskrift over options-ene i gruppen, men det er ikke selv et valg. Brukeren kan bare velge en <option>.

Sjekk · Lett
Hva sendes hvis et <option> ikke har value-attributt?
  • A Tomme strenger
  • B Tekstinnholdet til option-en
  • C Indeksen til option-en (0, 1, 2, …)
  • D Ingenting — option-en sendes ikke
Riktig: B. Tekstinnholdet brukes som fallback-verdi når value mangler.
07 · Vanlige feller

Vanlige feller

Felle 1 — glemt name

En kontroll uten name sendes ikke ved submit. Brukeren kan fylle ut feltet, men det havner ikke i form-dataen. Den vanligste «hvorfor mangler dette feltet?»-feilen.

Felle 2 — radio uten samme name

Radio-knapper i samme valg-gruppe må dele name. Med ulike navn oppfører de seg som uavhengige felt — brukeren kan velge flere samtidig, og hver sendes som et eget felt.

Felle 3 — textarea med value-attributt

<textarea value="Hei"></textarea> gir et tomt tekstområde. Standardverdi skal stå mellom taggene: <textarea>Hei</textarea>.

Felle 4 — label uten kobling

En <label> uten for og uten å omslutte input-en er bare tekst. Klikk på den gjør ingenting, og skjermlesere kobler den ikke til feltet.

08 · Test deg selv

Multiple choice

1 · Lett
Hvilket element grupperer relaterte form-kontroller?
  • A <section>
  • B <group>
  • C <fieldset>
  • D <form-group>
Riktig: C. <fieldset> grupperer kontroller; <legend> beskriver gruppen.
2 · Lett
Hvilken type maskerer tegnene som prikker?
  • A type="hidden"
  • B type="password"
  • C type="secret"
  • D type="text" + masked
Riktig: B. type="password" maskerer input visuelt.
3 · Middels
Hvordan grupperer du radio-knapper slik at bare én kan velges?
  • A Pakk dem i en <fieldset>
  • B Gi alle samme name-attributt
  • C Gi alle samme id
  • D Sett type="radio-group"
Riktig: B. Felles name binder dem som en gjensidig utelukkende gruppe. Hver knapp har egen value.
4 · Middels
Hva gjør maxlength="30" på et <input type="text">?
  • A Setter visuell bredde til 30 tegn
  • B Begrenser brukeren til maks 30 tegn i feltet
  • C Krever minst 30 tegn
  • D Strekker feltet over 30 linjer
Riktig: B. Slides: «You can use maxlength attribute to limit the number of characters a user may enter into the text field.» Visuell bredde styres av size.
5 · Middels
Hvilket attributt forhåndsmerker en <option> i en <select>?
  • A checked
  • B selected
  • C active
  • D default
Riktig: B. Slides bruker selected="selected" for forhåndsvalgt option. checked er for radio/checkbox.
6 · Vanskelig
På et <option>-element: når brukes tekstinnholdet som submit-verdi?
  • A Alltid — value-attributtet ignoreres
  • B Når value-attributtet mangler
  • C Bare hvis option-en er inne i en <optgroup>
  • D Aldri — det må alltid være en value
Riktig: B. Hvis value er satt brukes den; ellers fallback-er det til tekstinnholdet.
7 · Vanskelig
Hvilken syntaksvariant for <label> brukes i forelesningen?
  • A Bare <label for="id"> med for
  • B Bare wrap-stilen <label>...<input></label>
  • C Begge — slides nevner eksplisitt at <label> kan «wrap around both text description and the form input» eller «indicate which control it is a label for» via for
  • D Ingen — slides bruker bare placeholder
Riktig: C. Slides forms.md viser begge mønstrene side om side.
