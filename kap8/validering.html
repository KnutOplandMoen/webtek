01 · Hvorfor

Hvorfor validere?

Slides forms.md har én slide om validering — den heter «HTML5: Form Validation» og åpner med to setninger:

Slides forms.md

«Validation ensures that user enters information that server will be able to understand.»
«Validation has been performed using JS.»

Foreleser nevner altså to spor: HTML5-attributter som required, og JavaScript-validering. Sistnevnte er det du oftest skriver på eksamen (typisk e-postvalidering uten å bruke type="email").

02 · HTML5-validering

HTML5-attributter

Slides viser required på input-elementer, og at riktig type gir gratis format-sjekk for e-post, URL og dato. maxlength brukes på login-eksempelet. Det er det forelesningen nevner.

A · type

type="email" krever en gyldig e-postadresse. type="url" krever en gyldig URL. type="date" krever en gyldig dato. Hvis verdien ikke matcher type-en, vil ikke skjemaet kunne submit-es.

B · required

Slides bruker syntaksen required="required". Feltet må fylles ut før submit. Hvis det er tomt, viser nettleseren en innebygd melding og hindrer innsendingen.

C · maxlength

Maks antall tegn for tekst-input. Brukeren kan ikke skrive flere tegn. Slides bruker dette på login-eksempelet: maxlength="30".

Forelesningens valideringseksempel
<form action="http://www.example.com/login/" method="post">
  <label for="username">Username:</label>
  <input type="text" name="username" required="required"><br>
  <label for="password">Password:</label>
  <input type="password" name="password" required="required">
  <input type="submit" value="Submit">
</form>

Skjemaet kan ikke submit-es før begge feltene er fylt ut. Hvis du bytter type til email får du i tillegg format-validering — uten en eneste linje JavaScript.

Sjekk · Lett
Hva gjør required?
  • A Skjuler feltet hvis det er tomt
  • B Hindrer skjemaet i å sendes hvis feltet er tomt
  • C Setter en standardverdi i feltet
  • D Forbyr brukeren å endre verdien
Riktig: B. Nettleseren viser en innebygd melding og blokkerer innsendingen til feltet er fylt ut.
03 · JavaScript-validering

Validering med JavaScript

Slides sier eksplisitt at «Validation has been performed using JS.» På eksamen er det typisk dette du må skrive — særlig e-postvalidering uten type="email". Regex er ikke i pensum; bruk indexOf.

Foreleser-mønster

En typisk eksamensoppgave gir deg HTML for et skjema og ber deg skrive JS som leser verdier, sjekker dem og melder fra ved feil. Lest opp av foreleser: «email validation … they may forbid type="email"; if forbidden and you use it, you get 0 for that part.»

JS-validering uten regex
<form onsubmit="return val()">
  <input id="nm" type="text">
  <input id="al" type="text">
  <input id="ep" type="text">
  <input type="submit">
</form>
<script>
function val(){
  let n  = document.getElementById("nm").value;
  let a  = document.getElementById("al").value;
  let ep = document.getElementById("ep").value;

  if (n.length === 0){
    alert("Navn er påkrevd"); return false;
  }
  if (a < 18){
    alert("Må være 18+"); return false;
  }
  if (ep.indexOf("@") === -1 || ep.indexOf(".") === -1){
    alert("Ugyldig e-post"); return false;
  }
  return true;
}
</script>

Returnerer val() false, kanselleres submit av onsubmit="return val()". Returnerer den true, går skjemaet videre. Dette er mønsteret som passer både kalkulator-skjemaer og e-postsjekk på eksamen.

Sjekk · Middels
Oppgaven forbyr type="email". Hvordan sjekker du at strengen inneholder en @?
  • A /[a-z]@/.test(ep)
  • B ep.indexOf("@") !== -1
  • C ep.contains("@") — finnes ikke i JS
  • D type="email" — gir 0 poeng på den deloppgaven
Riktig: B. indexOf returnerer -1 hvis tegnet ikke finnes. Regex (A) er ikke i pensum. Bruker du type="email" mot oppgaveinstruksen, mister du poeng.
04 · Vanlige feller

Vanlige feller

Felle 1 — feil type, ingen sjekk

type="text" validerer ikke noe — det godtar alt. Hvis du forventer en e-postadresse, bruk type="email". Da får du gratis format-validering uten ekstra arbeid (når oppgaven tillater det).

Felle 2 — glemt return false ved feil

Returnerer JS-funksjonen true uansett, sendes skjemaet selv om sjekken slo ut. Husk return false; etter alert-en og onsubmit="return val()" på form-elementet — da kan submit faktisk avbrytes.

Eksamensvarsel — e-postvalidering kan forby type="email"

Foreleser har sagt eksplisitt at en typisk eksamensoppgave er å skrive e-postvalidering i JavaScript — og oppgaveteksten kan kreve at du ikke bruker type="email". Bruker du det likevel, får du 0 poeng for den deloppgaven, selv om resten av koden er riktig.

Pensum dekker ikke JavaScript-regex. Sjekk i stedet for de delene en e-postadresse må ha med rene string-metoder: 

let ep = document.getElementById("ep").value;
if (ep.indexOf("@") === -1 || ep.indexOf(".") === -1) {
  alert("Ugyldig e-post");
  return false;
}
05 · Test deg selv

Multiple choice

1 · Lett
Hvilket attributt forhindrer skjemaet i å sendes hvis feltet er tomt?
  • A mandatory
  • B required
  • C validate
  • D not-empty
Riktig: B. required er den standard HTML5-måten å markere et felt som obligatorisk. Slides bruker syntaksen required="required".
2 · Middels
Hvilken type-verdi gjør at nettleseren validerer at innholdet ser ut som en e-postadresse?
  • A type="text" + required
  • B type="email"
  • C type="address"
  • D type="mail"
Riktig: B. type="email" validerer formatet og fallback-er til text i gamle nettlesere.
3 · Middels
Oppgaven ber deg validere e-post i JS uten å bruke type="email". Hvilken sjekk er tilstrekkelig for å fange opp at @ mangler?
  • A if (ep === "")
  • B if (ep.indexOf("@") === -1)
  • C if (ep.length > 0)
  • D if (/[a-z]/.test(ep))
Riktig: B. indexOf returnerer -1 hvis tegnet ikke finnes. Regex (D) er ikke i pensum.
4 · Middels
Hvilken syntaks bruker slides for required på et felt?
  • A required=true
  • B required="required"
  • C required={1}
  • D required: yes
Riktig: B. Slides forms.md bruker required="required". (Det er også gyldig HTML å bare skrive required uten verdi.)
5 · Vanskelig
Hvorfor sier foreleser at validering kan/bør gjøres i JavaScript selv om HTML5 har required og type-validering?
  • A HTML5-validering fungerer ikke i Chrome
  • B Slides sier eksplisitt at validering «has been performed using JS», og eksamen krever ofte at du skriver JS-sjekk (f.eks. e-post uten type="email")
  • C JavaScript er raskere enn HTML5
  • D Det er ikke lov å bruke HTML5-validering i kurset
Riktig: B. JS-validering er kursets hovedmønster og det eksamen typisk tester.
6 · Vanskelig
Hva skjer i nettleseren hvis valideringsfunksjonen val() returnerer false?
  • A Skjemaet sendes uansett
  • B Submit avbrytes (forutsatt at form-elementet har onsubmit="return val()")
  • C Siden lastes på nytt
  • D Alle felt tømmes
Riktig: B. onsubmit="return val()" kobler returverdien til submit-handlingen — false kansellerer.
