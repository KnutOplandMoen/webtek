Array-metoder, scope, tid, DOM og events. Det er her JavaScript slutter å være regneuttrykk og begynner å gjøre noe synlig på siden.

Eksamensfokus · Kap 10 Høy prioritet

Event propagation/bubbling er forventet langsvarsoppgave

DOM og events er stort: 2 av 3 sett har en stor langsvarsoppgave om event propagation og bubbling (10 %), og 1/3 har en kode-oppgave med mouse/focus-handlers (10 %). I tillegg dukker forskjellen mellom getElementById og getElementsByX opp som langsvar.

Må kunne

  • Tre faser: capturing (ned fra window) → target → bubbling (opp til root)
  • addEventListener: tredje arg true = capture, false (default) = bubble
  • stopPropagation() stopper videre forplantning, preventDefault() blokkerer default-oppførsel
  • DOM-tilgang: forskjellen mellom getElementById (én), getElementsByClassName/TagName/Name (HTMLCollection), querySelector(All)
  • Event-typer: click, mouseover/mouseout, focus/blur, submit, keydown
  • Endre stil/innhold: el.style.color, el.innerHTML, el.value
  • Eksempel for bubbling: nestede divs, klikk på indre, listener på ytre

Kan nedprioriteres

  • Custom events og dispatchEvent
  • Event delegation som mønster (godt å vite, ikke testet)
  • Skjemaers FormData-API
  • Touch-events og pointer-events
  • Avansert DOM-traversering (parentNode.children, nextSibling)

01 · Helhetsbilde

Hvorfor disse fem temaene?

I kapittel 9 lærte du syntaksen til JavaScript: variabler, funksjoner, løkker, objekter. Det gir deg språk, men ikke kontaktflate mot siden. Kapittel 10 dekker de fem verktøyene du må kombinere for å lage en faktisk interaktiv nettside:

  • Array-metoder for å filtrere og transformere data uten å skrive løkker selv.
  • Scope for å forstå hvor en variabel er synlig — og hvorfor du noen ganger får ReferenceError.
  • Tid (setTimeout, setInterval) for å utsette eller gjenta noe.
  • DOM for å velge, endre, legge til og fjerne elementer i sida.
  • Events for å la siden reagere på brukeren.

Kilde: pensum 1.13.

02 · Mental modell

JS gjør tre handlinger mot DOM-treet

DOM er et tre med document som rot. JavaScript gjør tre ting mot dette treet: velger elementer, endrer dem, og lytter etter at noe skjer i dem. Alt annet i kapittelet (arrays, scope, tid) er støtteverktøy for disse tre handlingene.

Husk

DOM er strukturert som et tre. document er roten. Hvert HTML-element er en node, og en node kan ha barn og foreldre.

Kilde: pensum 1.13.4.

03 · Reisen

Bygg en handleliste

Vi bruker eksempelet fra pensum: en ul med tre li-elementer (Milk, Bread, Cheese). Stegene under kombinerer alt fra delkapitlene 1.13.4 og 1.13.5.

  1. Velg lista. document.getElementsByTagName('ul')[0] gir oss ul-elementet.
  2. Endre et element. Sett groceries[0].innerText = 'Juice' for å bytte Milk til Juice.
  3. Legg til. document.createElement('li') + sett innerText + appendChild.
  4. Fjern. Pek på elementet (f.eks. groceryList.firstChild) og kall removeChild på forelderen.
  5. Lytt. Legg til en knapp med addEventListener('click', …) som gjør én av handlingene over.
04 · Pensum 1.13.1

Array-metoder

Når du har en Array, vil du som regel gjøre noe med innholdet. Pensum lister fem metoder du må kunne. Alle tar en funksjon som argument og kalles direkte på arrayet.

filter — behold elementer som oppfyller en betingelse

filter returnerer en ny array med de elementene som returnerer true i funksjonen. Pensumeksempelet er en silsk metafor: vi siler bort frø og pulp og beholder juice. 

['juice', 'seed', 'pulp'].filter(ingredient => ingredient === 'juice');
// => ['juice']

map — transformer hvert element

map returnerer en ny array hvor hvert element er resultatet av funksjonen anvendt på det opprinnelige elementet. 

['Norway', 'Sweden', 'Denmark'].map(country => country.toUpperCase());
// => ['NORWAY', 'SWEDEN', 'DENMARK']

some — finnes det minst ett element som oppfyller betingelsen?

some returnerer boolean. Den skiller seg fra filter/map ved at du får tilbake true eller false, ikke en ny array. 

[1, 3, 5, 7].some(n => n % 2 === 0);   // false (ingen partall)
[1, 3, 5, 4].some(n => n % 2 === 0);   // true  (4 er partall)

every — oppfyller alle elementer betingelsen?

every returnerer også boolean, men krever at alle elementer passerer testen. 

[2, 4, 6, 8].every(n => n % 2 === 0);  // true
[2, 4, 6, 7].every(n => n % 2 === 0);  // false

forEach — gjør noe for hvert element, uten å returnere

Når du ikke vil beholde, transformere eller spørre — bare kjøre noe per element — bruk forEach. Returverdien er undefined; vi gjør det for bivirkningen, f.eks. logging. 

['Norway', 'Sweden', 'Denmark'].forEach(country =>
  console.log('Country: ' + country)
);
// Country: Norway
// Country: Sweden
// Country: Denmark
Velg riktig metode

Trenger du en kortere liste? filter.
Trenger du en transformert liste? map.
Trenger du å vite om noen oppfyller noe? some.
Trenger du å vite om alle gjør det? every.
Trenger du bare å gjøre noe per element? forEach.

05 · Pensum 1.13.2

Scope — hvor en variabel er synlig

Scope avgjør hvor du har tilgang til en variabel eller funksjon. Er noe «utenfor scope» betyr det at du ikke kan lese det. Pensum nevner tre former for scope.

Eksamen — bruk let/const, ikke var

Pensumeksemplene under bruker var fordi det er det pensum gjør, og fordi var er funksjons-scopet (relevant for å forklare hva «lokalt scope» er). Men på eksamen bør du skrive let/const. Foreleser sa rett ut: «a very good practice is to not use var and use let every time» — fordi var tillater re-deklarering som lett gir bugs.

Globalt scope

Alt som er definert utenfor en funksjon, ligger i globalt scope og er tilgjengelig hvor som helst. 

var planet = 'earth';   // globalt — tilgjengelig overalt

Lokalt scope

Definerer du variabelen inne i en funksjon, er den begrenset til den funksjonens scope. Forsøk på å lese den utenfra gir ReferenceError. 

function universe() {
  var planet = 'earth';
}
console.log(planet);
// ReferenceError: planet is not defined

Leksikalsk scope

En funksjon som er definert inne i en annen funksjon, har tilgang til den ytre funksjonens variabler. Det er det leksikalske scopet. 

function universe() {
  var planet = 'earth';
  function milkyWay() {
    console.log(planet);   // ser 'earth'
  }
  milkyWay();
}
Hvorfor det er kontraintuitivt

Du tenker kanskje at en indre funksjon er «helt uavhengig» av den ytre. Men siden den er skrevet inne i den ytre, arver den scopet derfra. Dette er nyttig når du vil dele en variabel mellom flere indre funksjoner uten å gjøre den global.

06 · Pensum 1.13.3

Tid — utsette og gjenta

Vanligvis kjører JavaScript sekvensielt. Noen ganger vil du derimot vente eller gjenta noe. Pensum nevner to funksjoner.

setTimeout — kjør én gang etter en forsinkelse

setTimeout(fn, ms) planlegger funksjonen fn til å kjøres om ms millisekunder. Pensumeksempelet er en SpaceX-rakett som tennes etter ett minutt. 

setTimeout(() => {
  liftOff();
}, 60000);   // 60 000 ms = 1 minutt

setInterval — gjenta i et intervall

setInterval(fn, ms) kjører funksjonen igjen og igjen, hver ms millisekund. Merk: den kjører ikke umiddelbart, men ved første intervall. 

setInterval(() => {
  measureTemperature();
}, 60000);   // måler én gang per minutt
Vær obs

Andre argument er millisekunder (ikke sekunder!). 1 sekund = 1000 ms. En vanlig feil på eksamen er å skrive 60 og tro at det er ett minutt.

07 · Pensum 1.13.4

DOM — velg, endre, legg til, fjern

DOM (Document Object Model) er broa mellom HTML-elementene og JavaScript. Pensum dekker fire operasjoner mot DOM-treet.

Velge elementer

Pensum nevner fire metoder du kan kalle på document:

getElementsByTagName('li')
Alle li-elementer på siden.
getElementById('id')
Det ene elementet med gitt id.
getElementsByClassName('class')
Alle elementer med gitt klasse.
querySelector('selector')
Første element som matcher en CSS-selektor.

Pensumeksempelet med en handleliste:

<ul>
  <li>Milk</li>
  <li>Bread</li>
  <li>Cheese</li>
</ul>
const groceries = document.getElementsByTagName('li');
console.log(groceries[0].innerText);   // 'Milk'
console.log(groceries[1].innerText);   // 'Bread'
console.log(groceries[2].innerText);   // 'Cheese'

Endre elementer

Hver node har egenskaper du kan lese og skrive. Pensum bruker innerText for å bytte ut teksten. 

const groceries = document.getElementsByTagName('li');
groceries[0].innerText = 'Juice';   // Milk → Juice

I tillegg til innerText dukker disse opp i kode-oppgaver:

el.innerHTML
Hele HTML-innholdet i elementet — du kan sette inn tagger, ikke bare tekst. Pensum 1.13 (Ajax-eksempel) bruker den til å oppdatere siden dynamisk.
el.value
Verdien i et <input>, <textarea> eller <select>. Brukt i form-validering på alle tidligere eksamener.
el.style.egenskap
Endre CSS direkte fra JS. el.style.color = 'red', el.style.backgroundColor = '#eee'. Merk: camelCase for CSS-navn med bindestrek. Brukes i «endre farge ved fokus»-oppgaver.
const overskrift = document.getElementById('tittel');
overskrift.innerHTML = 'Hei <em>du</em>!';
overskrift.style.color = 'red';
overskrift.style.backgroundColor = '#eee';

const navn = document.getElementById('navn').value;   // les input fra felt

Legge til elementer

Nye noder lages med document.createElement og kobles inn i treet som barn av et element via appendChild. 

const groceryList = document.getElementsByTagName('ul')[0];
const soapItem = document.createElement('li');
soapItem.innerText = 'Soap';
groceryList.appendChild(soapItem);

Fjerne elementer

Du fjerner et barn ved å kalle removeChild på forelderen. Pensum bruker firstChild for å peke på første barn av lista. 

const groceryList = document.getElementsByTagName('ul')[0];
const milkItem = groceryList.firstChild;
groceryList.removeChild(milkItem);   // fjerner Milk
08 · Pensum 1.13.5

Events — sida reagerer på brukeren

Events signaliserer at noe har skjedd i DOM-en. Noen er bruker-genererte (mus, tastatur), andre er bivirkninger (en video som pauser, en animasjon som er ferdig).

For å reagere på en event registrerer du en event listener — en funksjon som planlegges til å kjøre når eventen fyres. Eventen sendes inn som argument til lytteren.

Click — den vanligste eventen

<button id="greeting">Howdy</button>
const greetingButton = document.getElementById('greeting');
greetingButton.addEventListener('click', (event) => {
  alert('... partner!');
});

Keypress — tastatur-events

Når brukeren trykker en tast, fyres som regel tre events:

  1. keydown — når tasten først blir trykt ned.
  2. keypress — kun hvis tasten ikke er en modifikator (f.eks. Shift).
  3. keyup — når brukeren slipper tasten.

Hver tast har en unik keyCode-verdi du leser av eventen for å skille hvilken tast som ble trykt. Pensum lister blant annet:

a / b / c
65 / 66 / 67
← (venstre)
37
→ (høyre)
38
↑ (opp)
39
↓ (ned)
40
F1 / F2 / F3
112 / 113 / 114

Eksempel fra pensum: alert hvis brukeren trykker en piltast hvor som helst i DOM-en. 

document.addEventListener('keydown', (event) => {
  if (event.keyCode > 36 && event.keyCode < 41) {
    alert('You pressed an arrow key.');
  }
});

Andre vanlige event-typer

Foreleser anbefaler eksplisitt å skrive disse event-navnene på juksearket («Event names — spell them right»). De registreres alle med addEventListener('navn', fn):

click
Brukeren klikker på elementet.
focus / blur
Et input-felt får / mister fokus. Validate on blur er forelesers eksempel.
submit
Skjemaet sendes — brukes oftest med preventDefault() for å validere først.
change
Verdien i et input-felt eller select endres.
keydown / keypress / keyup
Tre faser ved tastetrykk (se over).
Staving må være riktig

Event-navnet i addEventListener må være eksakt slik nettleseren venter det: click — ikke onClick eller onclick. Foreleser er streng på dette. on…-formen er for HTML-attributter (<button onclick="…">), ikke for addEventListener.

09 · Pensum 1.13.5 (events forts.)

Event propagation og bubbling

Dette er det mest sannsynlige langsvar-temaet på eksamen (10 %, 2 av 3 sett). Foreleser har påpekt at studenter taper poeng fordi de forklarer hva som skjer men glemmer å nevne de tre fasene.

Problemet: hvilket element «eier» eventen?

Tenk på en knapp som ligger inni en div som ligger inni body. Brukeren klikker på knappen. Skal eventen utløse lyttere registrert på knappen, på div-en, på body — eller alle tre? Svaret er: alle tre, men i en bestemt rekkefølge.

De tre fasene

Hver event-instans går gjennom DOM-treet i tre faser, alltid i samme rekkefølge:

  1. Capturing-fasen — eventen reiser nedover fra window gjennom document, html, body og videre ned til target-elementet. Lyttere registrert med tredje argument true kjører her.
  2. Target-fasen — eventen er framme ved elementet som faktisk ble klikket. Lyttere på selve target-elementet kjører.
  3. Bubbling-fasen — eventen bobler oppover fra target tilbake mot window. Lyttere registrert med false (eller utelatt tredje argument) kjører her. Dette er default-fasen.
Bubbling = «opp», capturing = «ned»

Navnet «bubbling» kommer av at eventen bobler oppover treet som en luftboble i vann. Capturing er det motsatte: eventen fanges på vei ned mot target.

Tredje argument til addEventListener

Default-oppførselen er bubbling. Du velger capturing ved å sende true som tredje argument: 

el.addEventListener('click', fn);          // bubbling (default)
el.addEventListener('click', fn, false);   // bubbling (eksplisitt)
el.addEventListener('click', fn, true);    // capturing

Konkret eksempel — nestede divs

<div id="ytre">
  <div id="indre">
    <button id="knapp">Klikk meg</button>
  </div>
</div>
document.getElementById('ytre').addEventListener('click', () => {
  console.log('ytre div ble klikket');
});
document.getElementById('indre').addEventListener('click', () => {
  console.log('indre div ble klikket');
});
document.getElementById('knapp').addEventListener('click', () => {
  console.log('knappen ble klikket');
});

Brukeren klikker på knappen. Konsollen viser (i denne rekkefølgen):

knappen ble klikket
indre div ble klikket
ytre div ble klikket

Selv om brukeren bare klikket på knappen, fyres alle tre lytterne fordi eventen bobler oppover gjennom forelderne. Hadde den ytterste lytteren vært registrert med true, ville den kjørt først (i capturing-fasen), før knappens egen lytter.

stopPropagation — stans flyten

Hvis du vil at bare det innerste elementet skal reagere på klikket, kaller du event.stopPropagation() i lytteren. Da stopper eventen og kjører ikke flere lyttere oppover (eller nedover, hvis du stopper i capturing-fasen). 

document.getElementById('knapp').addEventListener('click', (event) => {
  event.stopPropagation();   // ytre lyttere fyres ikke
  console.log('kun knappen');
});

preventDefault — blokker default-handling

preventDefault() stopper ikke propagation. Den stopper nettleserens default-oppførsel for eventen:

  • På et <a>-klikk hindrer den navigering.
  • På et submit-event hindrer den at skjemaet sendes.
  • På en høyreklikk-event hindrer den kontekstmenyen.
form.addEventListener('submit', (event) => {
  if (!validerInput()) {
    event.preventDefault();   // skjemaet sendes ikke
  }
});
Forskjellen i én setning

stopPropagation() sier «ikke fortsett til andre lyttere». preventDefault() sier «ikke gjør det nettleseren ellers ville gjort».

Eksamen — sjekkliste for langsvar

For full uttelling på «Forklar event propagation og bubbling» må svaret ditt inneholde:

  • Navngi alle tre fasene: capturing → target → bubbling
  • Capturing = ned fra window, bubbling = opp til window
  • Default-fasen er bubbling — tredje argument true bytter til capturing
  • Konkret eksempel med nestede elementer (en div inni en div, eller knapp i div)
  • Nevn stopPropagation() som måten å bryte flyten på

12 · Vanlige feller

Vanlige feller

Eksamensvarsel — staving av getElementById

Foreleser har sagt at getElementByID (med både I og D som store bokstaver) regnes som feil på eksamen. Riktig staving er getElementById — kun I er stor, d er liten. Samme prinsipp for de pluraliserte variantene: getElementsByClassName og getElementsByTagName — husk «s»-en i «Elements» når du henter flere. Disse navnene må staves nøyaktig riktig; foreleser strekker seg ikke på dette.

Eksamensvarsel — skriv kode, ikke pseudokode

Foreleser sa rett ut: «writing pseudocode instead of some code that will give me an idea of your thinking process is the biggest mistake». Engelsk prosa som «jeg ville hentet elementet og endret teksten» gir 0 poeng, selv om logikken er riktig. Skriv faktisk JS — stavefeil og småfeil i syntaksen straffes ikke så lenge koden er lesbar, men det må være kode.

Pass på
  • forEach returnerer ikke noe. Bruker du const x = arr.forEach(...) blir x alltid undefined. Bruk map eller filter hvis du trenger en ny array.
  • some/every gir boolean, ikke en array. Pensum understreker at returverdien skiller seg fra de andre.
  • Lese variabel utenfor scope gir ReferenceError. Sjekk om variabelen er definert inne i en funksjon.
  • setTimeout / setInterval bruker millisekunder. 60 betyr 60 ms, ikke 60 sekunder.
  • setInterval kjører ikke umiddelbart — første kjøring skjer etter første intervall.
  • removeChild kalles på forelderen, ikke på elementet selv. Du må først finne foreldernoden.
  • keypress fyres ikke for modifikatorer som Shift; bruk keydown hvis du må fange dem også.
  • Event-navn må staves eksakt: click, keyup, keydown, blur, focus, submit, change. Foreleser er streng på dette — onClick som event-streng i addEventListener("onClick", …) er feil.
  • stopPropagation ≠ preventDefault. stopPropagation stopper at andre lyttere fyres. preventDefault stopper bare nettleserens default-oppførsel (submit, navigering). Du kan trenge begge i en form-validator.
  • Bubbling er default. Glemmer du tredje argument er det bubbling-fasen lytteren reagerer i, ikke capturing. Ikke skriv «default er capturing» — det er feil.
  • Bubbling skjer alltid — selv når du ikke har registrert flere lyttere. Eventen bobler likevel oppover; det er bare ingen som lytter. Dette er viktig å forklare i langsvaret.
13 · Eksamensscenario

Bytt og utvid handlelista

Oppgave

Gitt følgende HTML:

<ul id="groceries">
  <li>Milk</li>
  <li>Bread</li>
  <li>Cheese</li>
</ul>
<button id="add-soap">Legg til Soap</button>

Skriv JavaScript som (a) bytter Milk til Juice ved sideoppstart, og (b) legger til Soap nederst i lista når brukeren klikker på knappen.

Forslag til løsning
Klikk for å se en mulig løsning.
// (a) Bytt Milk til Juice
const items = document.getElementsByTagName('li');
items[0].innerText = 'Juice';

// (b) Legg til Soap når knappen klikkes
const list = document.getElementById('groceries');
const button = document.getElementById('add-soap');

button.addEventListener('click', (event) => {
  const soap = document.createElement('li');
  soap.innerText = 'Soap';
  list.appendChild(soap);
});

Vi velger li-ene med getElementsByTagName og endrer første element via innerText. Knappen får en addEventListener('click', …) som lager en ny li med createElement, gir den tekst, og kobler den inn med appendChild.

14 · Test deg selv

Helhetsspørsmål

Femten spørsmål som dekker alle fem delkapitler i 1.13, inkludert event propagation.

1 · Lett · 1.13.1
Hva returnerer ['a','b','c'].forEach(x => x.toUpperCase())?
undefined. forEach returnerer ingenting — du gjør noe per element men får ikke en ny array. Hadde du brukt map, ville svaret vært ['A','B','C'].
2 · Lett · 1.13.1
Hva er forskjellen på some og every?
some returnerer true hvis minst ett element oppfyller betingelsen. every returnerer true bare hvis alle elementer gjør det. Begge returnerer boolean.
3 · Middels · 1.13.1
Hva blir resultatet av [1, 2, 3, 4].filter(n => n > 2).map(n => n * 10)?
[30, 40]. Først filtreres bort 1 og 2 (gir [3, 4]), deretter ganges hvert element med 10.
4 · Lett · 1.13.2
Hva skjer når koden under kjører?
function f() { var x = 10; }
console.log(x);

Hvilke tre events fyres typisk når brukeren trykker en bokstav-tast?

13 · Middels · Propagation
Nevn de tre fasene en event går gjennom, i riktig rekkefølge.
Capturing (ned fra window mot target) → target (på selve elementet) → bubbling (opp fra target tilbake til window). Default-lyttere kjører i bubbling-fasen.
14 · Vanskelig · Propagation
Gitt nestede divs <div id="A"><div id="B"><button id="C">Klikk</button></div></div> med en click-lytter på alle tre. Brukeren klikker på knappen. I hvilken rekkefølge logges «A», «B», «C» — og hvordan endrer det seg hvis A-lytteren registreres med tredje arg true?

Default (alle bubbling): C → B → A. Eventen treffer target (knappen) først, så bobler den oppover.

Med A som capturing (true): A → C → B. A fanger eventen på vei ned i capturing-fasen, før target nås. B er fortsatt bubbling, så den kjører etter C.

15 · Middels · Propagation
Hva er forskjellen på event.stopPropagation() og event.preventDefault()?
stopPropagation() stopper at eventen fortsetter til andre lyttere i DOM-treet — bubbling/capturing stanser. preventDefault() stopper nettleserens default-handling for eventen (f.eks. at et skjema sendes, eller at en lenke navigerer). De påvirker uavhengige ting og kan kombineres.

1.13.5 EventsLett

Hva er en event listener?

1.13.5 EventsMiddels

Hvilke tre events fyres når man trykker en bokstav-tast?

keydown → keypress → keyup. keypress hoppes over hvis tasten er en modifikator (f.eks. Shift).

  • A event.value
  • B event.keyCode
  • C event.text
  • D event.id

Piltastene har keyCode 37–40, så > 36 && < 41 matcher akkurat dem.

PropagationLett

Hva er de tre fasene i event propagation, i riktig rekkefølge?

1. Capturing (ned fra window) → 2. Target (på elementet selv) → 3. Bubbling (opp til window). Default-fasen for lyttere er bubbling.

PropagationMiddels

Hva gjør tredje argument til addEventListener(..., true)?

  • A Stopper bubbling
  • B Aktiverer capturing-fasen
  • C Fjerner default-handling
  • D Gjør lytteren passiv

B — aktiverer capturing. Default (false/utelatt) er bubbling. Med true fyres lytteren på vei ned mot target i stedet for på vei opp.

Riktig: B

PropagationMiddels

Forskjellen på stopPropagation() og preventDefault()?

stopPropagation stopper at eventen når andre lyttere i DOM-treet (bryter capturing/bubbling-kjeden). preventDefault stopper nettleserens default-handling (form submit, lenke-navigering, kontekstmeny). Uavhengige — kan brukes sammen.

PropagationVanskelig

Gitt <div id="ytre"><button id="indre"></button></div> med click-lytter på begge, og brukeren klikker på knappen. Hva logges først?

indre først, deretter ytre. Eventen treffer target (knappen) først, så bobler oppover gjennom forelderen. Skal ytre kjøre først, må den registreres med tredje argument true (capturing).
