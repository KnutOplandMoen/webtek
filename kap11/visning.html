Kapittel 11 · Underkapittel C · Pensum 1.14

Visning & JSON

Hvordan vise XML stylet i nettleseren med CSS, og hvordan samme data ser ut i JSON og XML side om side.

Pensum: 1.14 Forelesning 13
01 · Rå XML

Hva skjer uten styling?

Åpner du et XML-dokument i nettleseren uten å koble til et stilark, viser nettleseren råstoffet — gjerne fargelagt, men ellers akkurat slik koden er skrevet. XML har ingen innebygd presentasjon — taggene er dine egne og betyr ingenting visuelt.

For å vise data stylet kan du peke dokumentet på et CSS-stilark eller et XSLT-stilark. Pensum 1.14 fokuserer på CSS-varianten.

02 · Koble til CSS

En processing instruction

Etter prologen, før root-elementet, legger du inn en xml-stylesheet-instruksjon. Den ligner på XML-deklarasjonen i syntaks (<? … ?>):

persons.xml
<?xml version="1.0" encoding="UTF-8"?>
<?xml-stylesheet type="text/css" href="stylesheet.css"?>
<persons>
  <person>
    <name>Name Nameson</name>
    <height>173</height>
    <age>23</age>
  </person>
</persons>
stylesheet.css
persons { display: block; font-family: serif; }
person  { display: block; margin: 1em 0; padding: 0.5em;
          border: 1px solid #ccc; }
name    { display: block; font-weight: bold; font-size: 1.2em; }
height, age { display: inline-block; margin-right: 1em; color: #666; }

CSS-en kjenner ingen XML-tagg fra før, så du må selv si at person er en blokk, gi name fet skrift osv. Selektorene er bare element-navnene — alt annet i CSS er som vanlig.

Mental modell

Tenk på det som å bygge HTML fra grunnen av: ingen display: block arvet automatisk, ingen standardstiler. Du må eksplisitt si hvordan hver av dine egne tagger skal vises.

03 · JSON ↔ XML

Samme data, to former

XML og JSON brukes til samme jobb — utveksle data mellom systemer — men har ulik form. Pensum 1.14 viser hvordan de ser ut side om side. Det er nyttig å kunne oversette begge veier.

JSON
{
  "person": {
    "name": "Name Nameson",
    "height": 173,
    "age": 23
  }
}
XML
<person>
  <name>Name Nameson</name>
  <height>173</height>
  <age>23</age>
</person>

Reglene for oversettelse er enkle:

  • JSON-objekt → XML-element med barn
  • JSON-nøkkel → XML-tag-navn
  • JSON-verdi (string/number) → XML-tekstinnhold
  • JSON-array → flere søsken-elementer med samme tag-navn
JSON med array
{
  "persons": [
    { "name": "A", "age": 20 },
    { "name": "B", "age": 22 }
  ]
}
XML
<persons>
  <person>
    <name>A</name>
    <age>20</age>
  </person>
  <person>
    <name>B</name>
    <age>22</age>
  </person>
</persons>
04 · Forskjeller

Når hva?

JSON
Kompakt, parses direkte til JS-objekt med JSON.parse. Mest brukt på web. Skiller mellom tall, streng, boolean.
XML
Mer ordrik. Støtter attributter, kommentarer, namespaces, og strict validering via DTD/XSD. Vanlig i konfigurasjon, dokumenter, og eldre integrasjoner.

I praksis brukes JSON oftest til API-kall mellom nettleser og server. XML lever videre i RSS-feeds, SOAP-tjenester, Office-dokumenter (.docx er XML inni en zip), SVG-grafikk og konfigurasjonsfiler.

05 · Test deg selv

Multiple choice

1 · Lett
Hva skjer hvis du åpner et XML-dokument uten stilark i nettleseren?
  • Det vises som vanlig HTML
  • Det gir feilmelding
  • Råstoffet vises som det er — ingen automatisk styling
  • Nettleseren laster ned filen i stedet
XML har ingen innebygd presentasjon — taggene er dine egne. Uten stilark vises kilden direkte.
2 · Lett
Hvilken instruksjon kobler et CSS-stilark til et XML-dokument?
  • <link rel="stylesheet" href="…">
  • <?xml-stylesheet type="text/css" href="…"?>
  • <style src="…"/>
  • <css href="…"/>
Pensum 1.14: en processing instruction (samme <? … ?>-syntaks som prologen).
3 · Lett
Hvor plasseres xml-stylesheet-instruksjonen?
  • Inni root-elementet, som første barn
  • I prologen, etter XML-deklarasjonen og før root
  • Hvor som helst i dokumentet
  • I et eget <head>
Den hører til prologen — etter <?xml … ?> og før selve dataene.
4 · Middels
Hva er en god mental modell for å style XML med CSS?
  • Det fungerer akkurat som HTML, alt arves automatisk
  • Som å bygge fra grunnen av — du må eksplisitt si display: block osv. for hver tag
  • CSS funker ikke på XML
  • CSS gjelder bare for attributter
XML har ingen standard-stiler. Hver av dine egne tagger må få display, font, margin osv.
5 · Middels
Oversett { "person": { "name": "X", "age": 23 } } til XML.
  • <person name="X" age="23"/>
  • <person><name>X</name><age>23</age></person>
  • <person>{name: X, age: 23}</person>
  • <name>X</name><age>23</age>
JSON-objekt → element med barn. Hver nøkkel blir en egen barn-tag.
6 · Middels
Hvordan oversettes en JSON-array til XML?
  • Som ett element med komma-separerte verdier
  • Som flere søsken-elementer med samme tag-navn
  • Som et array-element med tall-attributter
  • Arrayer kan ikke uttrykkes i XML
F.eks. flere <person> inni en <persons>-wrapper.
7 · Vanskelig
Hvilken CSS-regel gjør at <person> vises som en blokk med kantlinje?
  • .person { display: block; border: 1px solid; }
  • #person { display: block; border: 1px solid; }
  • person { display: block; border: 1px solid; }
  • <person> { display: block; border: 1px solid; }
I XML er tag-navnet selektoren — uten . eller # foran. CSS bryr seg ikke om at «person» ikke er en HTML-tag.
8 · Vanskelig
Hvilken er en typisk fordel JSON har over XML for web-API?
  • Parses direkte til JS-objekt med JSON.parse, mer kompakt
  • Støtter validering med XSD
  • Har innebygde kommentarer
  • Bedre støtte for namespaces
JSON er kompakt og blir et JS-objekt direkte. XML er rikere (attributter, kommentarer, namespaces, validering) men mer ordrik.
