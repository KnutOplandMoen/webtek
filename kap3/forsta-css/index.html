01 · Pensum 1.6.1

Hva er CSS?

CSS står for Cascading Style Sheets. Mens HTML beskriver hva innholdet er (overskrift, avsnitt, lenke), beskriver CSS hvordan det skal se ut (farge, font, marg, kant). Pensum 1.6.1 kaller dette skillet mellom struktur og presentasjon.

Et stilark er en samling regler. Når reglene står i en egen .css-fil kan flere HTML-sider dele samme stil — endrer du én linje i stilarket, oppdateres utseendet på hele nettstedet samtidig. Det er det Don't Repeat Yourself (DRY) handler om.

Kjerneidé

HTML = struktur. CSS = utseende. Skillet gjør at du kan endre designet uten å røre innholdet, og at flere sider deler en og samme stil.

02 · Pensum 1.6.1

Hvert element bor i en boks

Pensum bruker en enkel mental modell: rundt hvert HTML-element ligger en usynlig boks. CSS-reglene bestemmer både hvordan boksen ser ut (bakgrunn, ramme, plass) og hvordan innholdet i den ser ut (font, farge, justering).

Det er denne boksen vi siden i kapittelet skal styre med padding, border og margin. For nå er det nok å huske at alt har en boks — også tekst-elementer som <p> og <span>.

Cascading Style Sheets — «kaskade» fordi flere regler kan treffe samme element, og en gitt rekkefølge bestemmer hvilken som vinner.
MCQ · 1.6.1
Hva er den primære oppgaven til CSS?
  • Definere strukturen i et HTML-dokument
  • Lagre data fra et skjema
  • Bestemme utseendet til HTML-elementer

CSS styrer presentasjonen. Strukturen er HTML, dynamikken er JavaScript, datalagring skjer på serveren.
03 · Pensum 1.6.2

Anatomien til en regel

En CSS-regel består av tre deler: selektor, egenskap og verdi. Selektoren peker ut hvilket element regelen gjelder for. Egenskap + verdi sier hva som skal endres. Sammen kalles egenskap-og-verdi-paret en erklæring (declaration). 

p {
  color: blue;
  font-size: 18px;
}
  • Selektor: p — peker ut alle <p>-elementer.
  • Erklæringsblokk: alt mellom { og }.
  • Erklæring: color: blue; er én erklæring. Vi har to her.
  • Egenskap: color. Verdi: blue.
  • Skilletegn: kolon mellom egenskap og verdi, semikolon etter hver erklæring.
Pensum-detalj

Det aller siste semikolonet i en blokk er teknisk sett valgfritt, men pensum (og god stil) sier: skriv det alltid. Da slipper du å glemme det neste gang du legger til en erklæring.

MCQ · 1.6.2
Hva kalles paret color: blue i en CSS-regel?
  • En selektor
  • En erklæring (declaration)
  • En kommentar
  • Et stilark
Egenskap + verdi = erklæring. Alle erklæringer mellom { og } kalles en erklæringsblokk.
MCQ · 1.6.2
Hvilket tegn skiller egenskap og verdi i en erklæring?
  • Likhetstegn =
  • Semikolon ;
  • Kolon :
  • Pipe |
Kolon skiller egenskap og verdi. Semikolon avslutter hele erklæringen.
MCQ · 1.6.2
Hvilken av disse linjene er syntaktisk korrekt som CSS-regel?
  • p = color: blue;
  • p { color = blue; }
  • p { color: blue; }
  • color(p): blue;
Selektor først, deretter erklæringsblokk i krøllparenteser. Kolon, ikke likhetstegn, mellom egenskap og verdi.
04 · Pensum 1.6.3

Kommentarer i CSS

Kommentarer skrives mellom /* og */ og kan dekke én eller flere linjer. Nettleseren ignorerer alt som står inni. Pensum nevner to typiske bruksområder:

  1. Forklare hvorfor en regel er som den er, særlig hvis verdien ikke er åpenbar (f.eks. en magisk piksel-verdi som matcher en bestemt designspec).
  2. Skru av regler midlertidig uten å slette dem — nyttig når du eksperimenterer eller debugger.
/* Hovedfargen — matcher logoen */
h1 {
  color: #1f4e79;
}

/* Skrudd av mens vi tester nytt design
.kort {
  border: 1px solid red;
}
*/
Pass på

I motsetning til mange programmeringsspråk har CSS ingen én-linje-kommentar (// finnes ikke i ren CSS — den hører til i preprosessorer som SCSS).

MCQ · 1.6.3
Hvordan skriver du en kommentar i CSS?
  • // dette er en kommentar
  • # dette er en kommentar
  • /* dette er en kommentar */
  • <!-- dette er en kommentar -->
/* … */ er den eneste kommentar-syntaksen i ren CSS. <!-- --> hører til HTML, og // finnes kun i preprosessorer som SCSS.
MCQ · 1.6.3
Hva skjer hvis du omslutter en hel CSS-regel med /* og */?
  • Regelen får høyere prioritet
  • Regelen blir ignorert (skrudd av)
  • Regelen kjøres bare i utviklerverktøy
  • Regelen blir konvertert til HTML-kommentar
Alt mellom /* og */ er en kommentar — nettleseren leser den ikke som CSS. En vanlig debug-teknikk.
05 · Vanlige feller

Vanlige feller

Pass på
  • Glemmer semikolon. Hver erklæring må avsluttes med ; — uten det «smitter» nettleseren neste linje sammen med den forrige og misforstår.
  • Bruker = i stedet for :. Klassisk feil hvis du kommer fra et språk der = er tilordning. CSS bruker kolon.
  • Skriver HTML-kommentar i CSS. <!-- … --> virker ikke i et stilark — bare /* … */.
  • Tror linjeskift erstatter semikolon. Det gjør det ikke. Linjeskift er bare for menneskets lesbarhet.
06 · Test deg selv

Avsluttende spørsmål

MCQ · Helhet
Hva er hovedforskjellen på HTML og CSS i tankemodellen pensum bruker?
  • HTML beskriver struktur, CSS beskriver utseende
  • HTML kjøres i nettleseren, CSS kjøres på serveren
  • HTML er for små sider, CSS er for store sider
  • HTML laster raskere enn CSS
«HTML = hva, CSS = hvordan.» Begge tolkes av nettleseren — server/klient-skillet er ikke relevant her.
MCQ · Helhet
Hvilken av disse er en gyldig erklæringsblokk?
  • { color: red; font-size: 16px } (uten selektor)
  • h1 { color: red; font-size: 16px; }
  • h1: color = red; font-size = 16px;
  • h1 [color: red, font-size: 16px]
En komplett regel = selektor + krøllparenteser + erklæringer separert med semikolon.
MCQ · Helhet
Hva er en typisk grunn til å legge regler i en ekstern .css-fil framfor i <style>?
  • Ekstern CSS støtter flere egenskaper
  • Flere HTML-sider kan dele samme stil — DRY
  • Inline-CSS finnes ikke
  • Ekstern CSS har høyere presedens enn inline
DRY: én fil med stiler kan deles på tvers av sider. Inline-stil har faktisk høyere presedens enn ekstern, ikke lavere.
