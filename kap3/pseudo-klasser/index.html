Pseudo-klasser

En pseudo-klasse er ikke et reelt attributt på elementet, men en tilstand: musen er over det, det er besøkt, det er avkrysset. CSS gir deg lov til å style disse tilstandene direkte.

01 · Pensum 1.6.12

Hva er en pseudo-klasse?

En pseudo-klasse skrives med et kolon etter en vanlig selektor: 

a:hover {
  color: red;
}

Den treffer ikke en bestemt klasse i HTML-en, men en tilstand elementet kan være i. Pensum lister tre typer tilstander:

  • Brukerinteraksjon: :hover (musen er over), :active (museknappen trykkes ned).
  • Historikk: :link (ikke besøkt), :visited (besøkt).
  • Element-status: :checked (avkrysset), :disabled, :focus.
MCQ · 1.6.12
Hva er en pseudo-klasse?
  • En usynlig HTML-tag
  • En selektor som matcher en tilstand på elementet, ikke et attributt
  • En selektor som ikke virker i moderne nettlesere
  • En måte å lage falske klasser i HTML
«Pseudo» = «som om». Den oppfører seg som en klasse, men matcher tilstand i stedet for attributt.
02 · Lenketilstander

De fire lenketilstandene

En lenke (<a href="…">) kan være i fire tilstander som hver har sin egen pseudo-klasse:

:link
Lenken er ikke besøkt enda. Standard farge i nettlesere er typisk blå.
:visited
Lenken er besøkt før (basert på nettleserhistorikk). Tradisjonelt lilla.
:hover
Musen ligger over lenken akkurat nå.
:active
Lenken er trykket på (museknappen er nede). Kortvarig tilstand.
a:link    { color: #0060A3; }
a:visited { color: #6b4ea8; }
a:hover   { color: red; text-decoration: underline; }
a:active  { color: orange; }
MCQ · 1.6.12
Hva matcher pseudo-klassen :visited?
  • Lenker som er åpne i en annen fane
  • Lenker brukeren har besøkt før (ifølge nettleserhistorikken)
  • Bilder med alt-tekst
  • Lenker som peker til samme domene
Tilstanden er basert på nettleserhistorikken — derfor kan ikke JavaScript lese den fritt (privatlivshensyn).
MCQ · 1.6.12
Når matcher :active?
  • Når en lenke er åpnet i en annen fane
  • Akkurat i øyeblikket museknappen er trykket ned på elementet
  • Når elementet har :focus
  • Når brukeren har scrollet til elementet
Kortvarig tilstand mens museknappen holdes nede. Slippes den, går elementet typisk over til :visited (hvis lenke).
03 · Rekkefølgen

Rekkefølgen er kritisk

Pseudo-klassene har lik spesifisitet — derfor avgjør «siste regel vinner» hvilken som er aktiv. Hvis du skriver :hover før :visited, vil :hover-stilen bli «slått ut» når lenken er besøkt.

Riktig rekkefølge er LVHA — Link, Visited, Hover, Active: 

a:link    { … }
a:visited { … }
a:hover   { … }
a:active  { … }
Pensum-poeng

Hvis du opplever at :hover ikke virker når du beveger musen over en besøkt lenke — sjekk rekkefølgen. :visited kommer da og overstyrer.

MCQ · 1.6.12
Hva er den korrekte rekkefølgen for lenke-pseudoklassene?
  • :hover, :active, :link, :visited
  • :link, :visited, :hover, :active
  • Rekkefølgen spiller ingen rolle
  • :active, :hover, :visited, :link
LVHA — huskeregel: «LoVe HAte». Pensum-spørsmål.
MCQ · 1.6.12
Hvorfor er rekkefølgen viktig?
  • Fordi nettleseren parser CSS bakover
  • Fordi pseudo-klassene har lik spesifisitet, så «siste regel vinner» avgjør hvilken som er aktiv
  • Fordi :link er den eneste som faktisk fungerer
  • Det er ikke viktig — bare en stilkonvensjon
Når flere kan være sanne samtidig (f.eks. :visited og :hover), bestemmer rekkefølgen i kildefila hvilken som overstyrer.
04 · Andre pseudo-klasser

Andre nyttige pseudo-klasser

:checked
Matcher <input type="checkbox"> eller <input type="radio"> som er avkrysset, eller <option> som er valgt.
:focus
Matcher elementet som har tastaturfokus akkurat nå (typisk en <input> man tabber til).
:disabled
Matcher form-felter som er deaktivert med disabled-attributtet.
:first-child / :last-child
Matcher det første eller siste barnet av forelderen.
MCQ · 1.6.12
Hva matcher input:checked?
  • Alle input-felter
  • Avkryssede checkbox-er eller radioknapper
  • Felter med tekst i
  • Felter med required-attributt
:checked matcher status av et form-element — typisk checkbox som er huket av.
05 · Vanlige feller

Vanlige feller

Pass på
  • Feil rekkefølge på :hover og :visited. Klassisk eksamensfelle. Sett :hover sist (etter :visited).
  • Glemmer :focus. Hvis du kun styler :hover, mister tastaturbrukere visuell tilbakemelding når de tabber gjennom siden. God praksis: style :hover, :focus sammen.
  • Bruker for sterke effekter på :visited. Av personvernhensyn er det begrenset hva du kan endre — typisk bare farge, ikke størrelse eller bakgrunn.
  • Skriver :hover uten kolon. Det gjør den til etterkommer-selektor av et tenkt hover-element.
06 · Test deg selv

Avsluttende spørsmål

MCQ · Helhet
Hva er forskjellen på :link og :visited?
  • Ingen — de er synonymer
  • :link matcher lenker som ikke er besøkt; :visited matcher dem som er besøkt
  • :link er for interne lenker; :visited for eksterne
  • :link matcher knapper, :visited matcher lenker
Tilstanden er basert på brukerens nettleserhistorikk.
MCQ · Helhet
Hvilken huskeregel hjelper deg å huske rekkefølgen for lenke-pseudoklassene?
  • «Hover Hates Visited»
  • «LoVe HAte» (LVHA — Link, Visited, Hover, Active)
  • «ABC: Active Before Cascade»
  • «Visited Always Last»
«LoVe HAte» — første bokstav i hver pseudo-klasse i riktig rekkefølge.
MCQ · Helhet
Du skriver a:hover { color: red; } over a:visited { color: purple; } i CSS-en. Hva skjer når du beveger musen over en besøkt lenke?
  • Lenken blir rød
  • Lenken blir lilla — fordi :visited står sist og overstyrer
  • Det blir feilmelding
  • Begge fargene blandes
Klassisk feil. Bytt rekkefølgen så :hover kommer sist (LVHA).
