Boks-modellen og sentrering

Hvert HTML-element er en boks med fire lag: innhold, padding, border og margin. Forstår du lagene, kan du både gi luft inni elementet, skille bokser fra hverandre, og sentrere hele oppsettet horisontalt med margin: auto.

01 · Pensum 1.6.8

De fire lagene

Boks-modellen er den viktigste mentale modellen i hele kapittel 3. Fra innerst til ytterst:

  1. Content — selve innholdet (tekst, bilde).
  2. Padding — luft mellom innholdet og rammen. Bakgrunnen til elementet strekker seg ut til padding-kanten.
  3. Border — selve rammen rundt boksen.
  4. Margin — luft utenfor rammen, som skiller boksen fra naboene. Margin er alltid gjennomsiktig.
┌─────────────────────────────────┐  ← margin (utenfor)
│  ┌──────────────────────────┐   │
│  │     border (rammen)      │   │
│  │  ┌────────────────────┐  │   │
│  │  │     padding        │  │   │
│  │  │  ┌──────────────┐  │  │   │
│  │  │  │   content    │  │  │   │
│  │  │  └──────────────┘  │  │   │
│  │  └────────────────────┘  │   │
│  └──────────────────────────┘   │
└─────────────────────────────────┘
MCQ · 1.6.8
Hva er rekkefølgen på lagene i boks-modellen, fra innerst til ytterst?
  • Margin → padding → content → border
  • Content → padding → border → margin
  • Border → margin → padding → content
  • Padding → content → margin → border
Innholdet er innerst, så luft (padding), så ramme (border), og helt ytterst luften som skiller naboene (margin).
MCQ · 1.6.8
Hvor strekker bakgrunnsfargen til en boks seg?
  • Bare innenfor content
  • Til og med padding (men ikke margin)
  • Til og med margin
  • Bare utenfor border
Bakgrunnen dekker content og padding. Margin er utenfor boksen og er alltid gjennomsiktig.
Innhold-størrelse

Pensum nevner at når box-sizing er satt til standard, styrer egenskapene width, min-width, max-width, height, min-height og max-height størrelsen på content-laget — ikke hele boksen inkludert padding og border. Setter du ingen av dem, vil boksen tilpasse seg innholdet sitt.

02 · Padding

Padding — luft inni boksen

Padding er rommet mellom innholdet og rammen. Du bruker det når teksten din ligger «klemt opp» mot kanten av elementet. 

.kort {
  padding-top: 16px;
  padding-right: 20px;
  padding-bottom: 16px;
  padding-left: 20px;
}

Eller med shorthand: 

.kort { padding: 16px 20px; }   /* topp/bunn 16, venstre/høyre 20 */
MCQ · 1.6.8
Hva styrer padding?
  • Avstanden mellom boksen og naboene
  • Avstanden mellom innholdet og rammen
  • Tykkelsen på rammen
  • Bredden på elementet
Padding = innvendig luft. Margin = utvendig luft.
03 · Margin

Margin — luft mellom bokser

Margin er rommet utenfor rammen — det skiller boksen fra andre bokser. Som regel det første du bruker for å gi avstand mellom avsnitt eller seksjoner. 

p { margin-bottom: 1em; }
section { margin: 32px 0; }
Utenfor pensum: margin collapse

Når to vertikale marger møtes (f.eks. margin-bottom på et avsnitt og margin-top på det neste), «kollapser» de til den største av de to. Pensum 1.6.8 nevner ikke dette eksplisitt, men det er greit å være obs på når et tilsynelatende stort mellomrom plutselig blir mindre enn forventet.

MCQ · 1.6.8
Hva er forskjellen på margin og padding?
  • De gjør det samme
  • Padding er innvendig luft (mellom innhold og ramme), margin er utvendig (mellom ramme og naboer)
  • Padding er for tekst, margin er for bilder
  • Margin støttes ikke i moderne nettlesere
Padding er inni rammen. Margin er utenfor rammen.
04 · Shorthand

Shorthand — én linje, fire sider

Både margin og padding har shorthand. Antall verdier bestemmer hvilke sider som settes:

1 verdi
padding: 10px; — alle fire sider får 10px.
2 verdier
padding: 10px 20px; — topp/bunn 10, venstre/høyre 20.
3 verdier
padding: 10px 20px 30px; — topp 10, sider 20, bunn 30.
4 verdier
padding: 10px 20px 30px 40px; — topp, høyre, bunn, venstre. (Klokkeretningen fra topp.)
MCQ · 1.6.8
margin: 10px 20px 30px 40px; — hva blir høyre margin?
  • 10px
  • 20px
  • 30px
  • 40px
Rekkefølgen er klokkeretningen fra toppen: topp (10), høyre (20), bunn (30), venstre (40).
05 · Pensum 1.6.9

Sentrer et element

For å sentrere en boks horisontalt i nettleservinduet (eller i foreldrelementet) trengs to ting:

  1. Boksen må ha en bredde mindre enn 100 % — f.eks. width: 600px; eller width: 80%;. En full-bredde-boks kan ikke sentreres horisontalt fordi det ikke er noe rom å fordele.
  2. margin-left: auto; og margin-right: auto; (eller margin: 0 auto;). Verdien auto gjør at nettleseren fordeler resterende plass likt på begge sider.
.kort {
  width: 600px;
  margin-left: auto;
  margin-right: auto;
  /* eller kortere: margin: 0 auto; */
}
Pensum-poeng

margin: auto sentrerer kun horisontalt for blokk-elementer. Det sentrerer ikke tekst inni elementet — for det trenger du text-align: center;.

MCQ · 1.6.9
Hva skal til for å sentrere et blokk-element horisontalt med margin?
  • text-align: center på elementet
  • position: center
  • En bredde mindre enn 100 %, og margin-left: auto; margin-right: auto;
  • Det går ikke uten flexbox
Bredde + auto-margin på begge sider. Uten en bredde mindre enn forelder, har auto ingenting å fordele.
MCQ · 1.6.9
Hva sentrerer text-align: center?
  • Selve elementet i nettleservinduet
  • Innholdet (tekst og inline-elementer) inni elementet
  • Marger på begge sider
  • Border-stilen
text-align styrer innholdet — ikke selve boksen. Lett å forveksle med margin: auto.
06 · Vanlige feller

Vanlige feller

Pass på
  • Forveksle margin og padding. Hvis bakgrunnsfargen «smitter» til naboen, har du sannsynligvis brukt padding der du ville hatt margin.
  • Glemmer width ved sentrering. margin: 0 auto alene gjør ingenting hvis elementet allerede er 100 % bredt.
  • Bruke text-align: center på et bilde og lure på hvorfor det ikke virker. Bilder er inline, så det fungerer hvis du setter text-align: center på foreldrelementet. Eller: gjør bildet til blokk og bruk margin: 0 auto.

Hva betyr padding: 12px;?
  • 12 piksler padding på alle fire sider
  • 12 piksler padding bare på toppen
  • 12 piksler margin
  • Det er ugyldig — du må oppgi alle fire

Hvilken egenskap skiller en boks fra naboboksene sine?

  • margin
  • border-radius
  • line-height
