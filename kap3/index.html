CSS — presentasjonens språk

Hver HTML-tag lever inni en usynlig boks. CSS er reglene som styrer hvordan boksen og innholdet ser ut: farger, fonter, marger, kanter og hvilken regel som vinner når flere prøver å gjelde samtidig.

Pensum: 1.6.1 – 1.6.12 Slides: css.md, selectors.md Forelesning 5–6
Eksamensfokus · Kap 3 Høy prioritet

CSS-selektorer er nesten garantert kode-oppgave

Hvert eksamenssett har en stor selektor-oppgave (8–10 %): du får et HTML-tre og må skrive CSS som treffer spesifikke elementer. I tillegg dukker forklarings-spørsmål om specificity og stilstrategier opp jevnlig. Dette er det mest poengrike CSS-kapittelet.

Må kunne

  • Selektor-typer: descendant (div p), child (div>p), adjacent (div+p), klasse, id, attributt
  • Pseudo-klasser: :hover, :first-child, :nth-child(n)
  • Pseudo-elementer: ::first-letter, ::first-line
  • Specificity-regler: id (100) > class (10) > element (1), !important overstyrer
  • External / Internal / Inline CSS — når brukes hva (langsvar 6 %)
  • Box-model rekkefølge: margin → border → padding → content (utenfra inn)
  • border-width 4 verdier = top right bottom left

Kan nedprioriteres

  • Fonter, font-fallbacks, web-fonts — testes ikke direkte
  • Detaljerte fargemodeller (HSL, CMYK)
  • CSS-variabler (custom properties) — ikke eksamenstoff
  • Animations / transitions — ikke testet

01 · Helhetsbilde

Hvorfor CSS?

CSS står for Cascading Style Sheets. Det er regler som bestemmer hvordan et element — eller innholdet i et element — skal se ut. Tankemodellen er enkel: rundt hvert HTML-element ligger en usynlig boks, og CSS styrer både boksen og innholdet i den.

Når reglene står i en egen fil kan flere HTML-sider dele samme stil. Da slipper du å skrive den samme stilen flere steder, og du kan endre hele sidens utseende på ett sted. Dette kalles ofte Don't repeat yourself (DRY) i pensum.

Kjerneidé

HTML beskriver hva innholdet er. CSS beskriver hvordan det skal se ut. Skill struktur fra presentasjon — én HTML, mange stiler.

Kilde: pensum 1.6.1 (Understanding CSS), 1.6.4 (Cascade — embedded vs. external).

02 · Mental modell

Kaskaden — hvem vinner?

Når flere regler treffer det samme elementet, må nettleseren velge én. Pensum 1.6.4 sier at rekkefølgen avgjøres av tre ting:

  1. Spesifisitet. En mer spesifikk selektor vinner over en mindre spesifikk. p er mer spesifikk enn *, og p b er mer spesifikk enn p. ID-er er mer spesifikke enn klasser.
  2. Siste regel. Hvis to regler er like spesifikke og treffer samme egenskap på samme element, vinner den som er skrevet sist (lengst ned).
  3. !important. En verdi merket !important overstyrer andre regler for samme element. Skal brukes med varsomhet.

I tillegg har det betydning hvor stilen er definert. Pensum lister fem nivåer i synkende rekkefølge:

  1. Inline-stil. Skrevet rett på elementet via style="…". Høyest presedens.
  2. Embedded styles. Inni <style>…</style> i HTML-dokumentet, gjerne i <head>.
  3. External style sheet. Egen .css-fil koblet inn med <link>.
  4. Brukerdefinerte regler. Stilark satt opp av sluttbrukeren i nettleseren. Lavere presedens enn dine egne stilark.
  5. Browser default. Nettleserens standardstil — brukes når ingen andre nivåer har sagt noe.
Praktisk konsekvens

Hvis du undres over hvorfor en farge ikke endrer seg: sjekk i rekkefølge — finnes det inline-stil? Finnes det en mer spesifikk selektor lenger ned i filen? Står det !important noe sted?

Kilde: pensum 1.6.4 (Cascade), 1.6.5 (Inheritance).

03 · Reisen

Fra naken side til stylet

En typisk vei fra usidet HTML til ferdig stylet side, slik pensum 1.6.1–1.6.12 går igjennom det:

  1. Koble inn et stilark. Lag style.css, og legg <link href="style.css" rel="stylesheet"> i <head>. (1.6.4 — External style sheets)
  2. Skriv den første regelen. Selektor + { + erklæring(er) + }. Hver erklæring er egenskap: verdi; og avsluttes med semikolon. (1.6.2)
  3. Velg riktig selektor. Type, klasse .x, ID #y, etterkommer p a, barn li > a, eller universell *. (1.6.6)
  4. Sett farger. color for tekst, background-color for bakgrunn. Bruk hex (#0060A3), navn (cyan) eller rgb/rgba. (1.6.7)
  5. Forstå boks-modellen. Innhold → padding → border → margin. Juster avstand inne i boksen med padding, og avstand mellom bokser med margin. (1.6.8)
  6. Sentrer den ytre boksen. Sett bredde mindre enn 100 %, så margin-left: auto; og margin-right: auto;. (1.6.9)
  7. Tegn rammer. border-width, border-style, border-color — eller kortformen border: 4px dashed #0060A3;. (1.6.10)
  8. Stil typografien. font-family (med fallback), font-weight, font-size, line-height, text-align, color. (1.6.11)
  9. Reager på brukeren. Pseudo-klasser: :link, :visited, :hover, :active — i den rekkefølgen for lenker. (1.6.12)
  10. Bruk arv smart. Sett font-family og color på body; barna arver verdien. Tving arv ved behov med inherit. (1.6.5)

Nøkkelbegreper

Selektor + erklæringsblokk. Bestemmer stil for ett eller flere elementer. Pensum 1.6.2.

Mønsteret før { som peker ut hvilke elementer regelen gjelder. Pensum 1.6.2 / 1.6.6.

Kaskade

Arv
Visse egenskaper (f.eks. font-family, color) arves fra forelder. Andre, som border, arves ikke. Pensum 1.6.5.
Boks-modellen
Content + padding + border + margin. Pensum 1.6.8.
Inline / embedded / external
Tre måter å koble CSS: i style-attributtet, i <style>-tag, eller via <link>. Pensum 1.6.4.
Pseudo-klasse
Et stikkord etter selektoren som matcher tilstand: :hover, :visited, :link, :active, :checked. Pensum 1.6.12.
!important
Overstyrer andre regler for samme egenskap. Pensum advarer mot misbruk. Pensum 1.6.4.
DRY
Don't Repeat Yourself — embedded og external stilark gjør at samme stil ikke må skrives flere steder. Pensum 1.6.4.
Web-safe font
Font installert på alle klienter (f.eks. Arial, Georgia). Brukes som fallback. Pensum 1.6.11.

08 · Vanlige feller

Vanlige feller

Pass på
  • Glemmer semikolon. Hver erklæring må avsluttes med ;. Uten det forstår ikke nettleseren at erklæringen er ferdig (1.6.2).
  • Tror at klasse alltid vinner over type. Det stemmer som tommelfingerregel, men siste regel og !important kan snu det. Husk: ID > klasse > type, gitt at ikke noe overstyrer (1.6.4).
  • Setter border på body og forventer at p arver det. border arves ikke. Bruk inherit hvis du virkelig vil ha det (1.6.5).
  • Pseudo-klasser i feil rekkefølge for lenker. :link → :visited → :hover → :active. Skriver du :hover før :visited, vil :hover-stilen ikke vises (1.6.12).
  • Misbruk av !important. Pensum advarer eksplisitt: bruk det med varsomhet, ellers blir kaskaden uleselig (1.6.4).
  • Glemmer fallback i font-family. Pensum krever at du oppgir generisk familie (serif/sans-serif/monospace) til slutt, så nettleseren har noe å falle tilbake på (1.6.11).
  • Sentrering uten å sette bredde. margin: 0 auto; virker bare hvis elementet er smalere enn vinduet (1.6.9).
  • Lav kontrast tekst/bakgrunn. Pensum 1.6.11 påpeker at lav kontrast gjør teksten vanskelig å lese, særlig for synshemmede.
09 · Eksamensscenario

Hvilken regel vinner?

Klassisk eksamensoppgave: gitt et HTML-fragment og noen CSS-regler — hvilken farge får teksten?

HTML
<div id="boks" class="varsel">
  Hei!
</div>
CSS
div          { color: black;  }
.varsel      { color: orange; }
#boks        { color: red;    }
.varsel      { color: green;  }
Tankegangen
  1. List opp regler som matcher. Alle fire treffer div-en (type, klasse to ganger, ID).
  2. Sammenlign spesifisitet. ID er mest spesifikk, klasse mer spesifikk enn type. #boks slår de andre.
  3. Konklusjon. Teksten blir rød. De to .varsel-reglene konkurrerer kun seg imellom (siste regel vinner: green), men begge taper mot #boks.

Hva hvis vi legger til color: blue !important; på .varsel? Da vinner klassen, fordi !important overstyrer vanlig spesifisitet. Teksten blir blå.

Andre typiske oppgaver fra pensum 1.6:

  • «Tegn boks-modellen og marker margin, border, padding og content.» (1.6.8)
  • «Skriv en CSS-regel som sentrerer en div med klasse kort og 600 px bredde.» (1.6.9)
  • «Hva blir bakgrunnsfargen til en p hvis body har background-color: gray;?» Svar: ingenting — background-color er ikke listet som arvet i pensum, og standard er gjennomsiktig.
  • «Skriv pseudo-klassene for å style en lenke i alle fire tilstander, i riktig rekkefølge.» (1.6.12)
10 · Test deg selv

Helhetsspørsmål

12 spørsmål som dekker pensum 1.6.1–1.6.12. Klikk «Se svar» når du har tenkt selv.

1 · Lett · Pensum 1.6.2
Hva heter de tre delene av en CSS-regel?
Selektor, egenskap (property) og verdi. Et par av egenskap+verdi kalles en erklæring. Alle erklæringene mellom { og } kalles en erklæringsblokk.
2 · Lett · Pensum 1.6.3
Hvordan skriver du en kommentar i CSS?
/* dette er en kommentar */. Kan også omslutte CSS-regler for midlertidig å skru dem av.
3 · Lett · Pensum 1.6.7
Nevn de tre måtene å oppgi en farge på i CSS som pensum lister.
Hex-kode (#0060A3), fargenavn (cyan) og rgb/rgba-verdier (rgb(0, 96, 163) eller rgba(0, 96, 163, 0.5)).
4 · Middels · Pensum 1.6.4
Hvilke fem nivåer av stiler finnes, og hvilken har høyest presedens?
Inline → embedded → external → user-specified → browser default. Inline-stil har høyest presedens.
5 · Middels · Pensum 1.6.4
To regler treffer samme element og setter samme egenskap. De er like spesifikke. Hvilken vinner?
Den som er skrevet sist (lengst ned) i CSS-en. Det er regelen om «last rule» i kaskaden.
6 · Middels · Pensum 1.6.5
Hvorfor blir hele dokumentet ofte i samme font selv om du bare har skrevet font-family på body?
font-family er en av egenskapene som arves fra forelder til barn. Alle barn av body arver fonten med mindre de selv overstyrer den. Det samme gjelder color.
7 · Middels · Pensum 1.6.6
Forskjell på p a og p > a?
p a er en etterkommer-selektor — treffer alle <a> som ligger inni en <p>, uansett hvor dypt. p > a er en barn-selektor — treffer kun <a> som er direkte barn av <p>.
8 · Middels · Pensum 1.6.8
Tegn lagene i boks-modellen fra innerst til ytterst.
Innerst: content. Så padding rundt innholdet. Så border som kanten. Ytterst: margin, som skiller boksen fra naboene.
9 · Middels · Pensum 1.6.9
Hvordan sentrerer du et element horisontalt i nettleservinduet?
Sett en bredde mindre enn 100 %, og bruk margin-left: auto; sammen med margin-right: auto;.
10 · Middels · Pensum 1.6.10
Skriv shorthand for en 4 px stiplet, blå ramme.
border: 4px dashed #0060A3;. Rekkefølgen er border-width, border-style, border-color.
11 · Vanskelig · Pensum 1.6.12
I hvilken rekkefølge må :link, :visited, :hover og :active stå i CSS-en for at alle skal virke?
a:link → a:visited → a:hover → a:active. Skriver du f.eks. :hover før :visited, vil :hover-stilen ikke ta effekt fordi :visited overstyrer den (siste regel + lik spesifisitet).
12 · Vanskelig · Pensum 1.6.4 + 1.6.6
Gitt <p class="advarsel" id="topp">Hei</p> og reglene: p { color: black; }, .advarsel { color: orange; }, #topp { color: red; }. Hvilken farge får teksten?
Rød. ID-selektoren #topp er mer spesifikk enn klasse, som igjen er mer spesifikk enn type. Rekkefølgen i filen spiller ingen rolle her, fordi spesifisitet avgjør først.

Hvilken usynlig figur omgir hvert HTML-element ifølge pensum?

En boks. CSS-regler styrer både boksen og innholdet i boksen.

Hva kalles paret egenskap: verdi;?

En erklæring (declaration). Alle erklæringer mellom { og } kalles en erklæringsblokk.

Pensum 1.6.2

1.6.2 · SyntaksLett

Hvilket tegn må stå etter hver erklæring?

Semikolon (;). Det forteller nettleseren at erklæringen er ferdig.

Pensum 1.6.2

1.6.3 · KommentarerLett

Hvordan ser en CSS-kommentar ut?

/* … */. Brukes til å forklare eller for midlertidig å skru av regler.

Hvilke tre faktorer avgjør hvilken regel som vinner?

1) Spesifisitet, 2) siste regel (rekkefølge), 3) !important.

Pensum 1.6.4

List de fem stilnivåene i fallende presedens.

Inline → embedded → external → user-specified → browser default.

Pensum 1.6.4

1.6.4 · KaskadenMiddels

Hvorfor er p b mer spesifikk enn p?

Den krever to ting samtidig (en <b> inni en <p>), ikke bare en <p>. Mer spesifikk selektor vinner.

Pensum 1.6.4 / 1.6.6

1.6.4 · KaskadenMiddels

Hva gjør !important, og hva er pensumets råd?

Markerer en erklæring som viktigere enn andre regler for samme egenskap. Pensum sier: bruk med varsomhet.

Pensum 1.6.4

1.6.4 · KaskadenLett

Hvordan kobler du inn et eksternt stilark?

<link href="path/to/stylesheet.css" rel="stylesheet"> i <head>.

Pensum 1.6.4

1.6.5 · ArvMiddels

Nevn to egenskaper som arves automatisk, og én som ikke arves.

Arves: font-family, color. Arves ikke: border (eksempel fra pensum).

Pensum 1.6.5

1.6.5 · ArvMiddels

Hvordan tvinger du en egenskap til å arve verdien fra forelder?

Sett verdien til inherit, f.eks. border-radius: inherit;.

Pensum 1.6.5

1.6.6 · SelektorerLett

Hvordan refereres en klasse, og hvordan refereres en ID i CSS?

Klasse: .klassenavn. ID: #id-navn. ID kan kun forekomme én gang per side; klasse kan brukes på flere elementer.

Pensum 1.6.6

1.6.6 · SelektorerMiddels

Hva treffer den universelle selektoren *?

Alle elementer i dokumentet.

Pensum 1.6.6

1.6.6 · SelektorerMiddels

Hva er forskjellen mellom li > a og p a?

li > a er barn-selektor — direkte barn. p a er etterkommer-selektor — alle a nestet inni en p, uansett dybde.

Pensum 1.6.6

1.6.6 · SelektorerVanskelig

Hva treffer h1+p og h1~p?

h1+p = adjacent sibling — den første <p> rett etter en <h1>. h1~p = general sibling — alle <p> som er søsken etter en <h1>.

Pensum 1.6.6

1.6.7 · FargerLett

Hvilken farge gir #000000, og hvilken gir #FFFFFF?

#000000 = svart (alle tre kanaler 0). #FFFFFF = hvit (alle tre kanaler maks).

Pensum 1.6.7

1.6.7 · FargerMiddels

Hva gjør den fjerde verdien i rgba(0, 96, 163, 0.5)?

Det er alfa-kanalen (opasitet). 0 = helt gjennomsiktig, 1 = helt ugjennomsiktig. 0.5 = 50 % opasitet.

Pensum 1.6.7

1.6.7 · FargerLett

Hvor mange forhåndsdefinerte fargenavn gjenkjenner nettlesere ifølge pensum?

147 fargenavn — f.eks. blue, cyan, lightgoldenrodyellow.

Pensum 1.6.7

1.6.8 · Boks-modellenLett

List opp de fire delene av boks-modellen fra innerst til ytterst.

Content → padding → border → margin.

Pensum 1.6.8

1.6.8 · Boks-modellenMiddels

Hva er forskjellen på padding og margin?

padding er rommet mellom innholdet og rammen — bakgrunnen til elementet strekker seg ut hit. margin er det tomme rommet utenfor rammen som skiller boksen fra naboene.

Pensum 1.6.8

1.6.9 · SentreringMiddels

Hvordan sentrerer du ytre element i nettleservinduet?

Sett en bredde mindre enn 100 %, og bruk margin-left: auto; + margin-right: auto;.

Pensum 1.6.9

1.6.10 · BordersLett

Hvilke tre egenskaper styrer en ramme?

border-width, border-style, border-color. Eller shorthand: border: 4px dashed #0060A3;.

Pensum 1.6.10

1.6.10 · BordersMiddels

Nevn fem mulige verdier for border-style.

F.eks. none, hidden, dotted, dashed, solid, double.

Pensum 1.6.10

1.6.10 · BordersMiddels

Hvordan setter du forskjellig stil per side på en ramme?

Med -top, -right, -bottom, -left: f.eks. border-top-style: solid; og border-left-style: dashed;.

Pensum 1.6.10

1.6.11 · Tekst og fonterLett

Hvilke tre typeface-klassifiseringer nevner pensum?

Serif, sans-serif og monospace.

Pensum 1.6.11

1.6.11 · Tekst og fonterMiddels

Hvorfor avsluttes font-family-listen typisk med en generisk familie som serif?

Det er en fallback: hvis ingen av de foregående fontene finnes på klienten, skal nettleseren bruke sin standard serif-font.

Pensum 1.6.11

1.6.11 · Tekst og fonterMiddels

Hvilke fire verdier kan text-align ta?

left, right, center, justify. Justify strekker linjene så hver linje er like bred (som i aviser).

Pensum 1.6.11

1.6.11 · Tekst og fonterVanskelig

Hvilken tommelfingerregel gir pensum for line-height?

Pensum 1.6.11

1.6.12 · Pseudo-klasserLett

Hva betyr :hover?

En tilstand som matcher når musepekeren er over elementet. Skrives som selektor:hover { … }.

Pensum 1.6.12

1.6.12 · Pseudo-klasserVanskelig

Hvorfor må pseudo-klassene for lenker stå i en bestemt rekkefølge?

Pensum 1.6.12
