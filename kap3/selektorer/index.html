Kapittel 3 · Underkapittel 1.6.6

Selektorer — velg ut elementene

En selektor er mønsteret før {. Den bestemmer hvilke HTML-elementer en regel gjelder for. CSS gir mange måter å peke på elementer — fra det helt generelle * til presise kombinasjoner som nav > ul > li.aktiv a.

01 · Enkle selektorer

De fire enkle selektorene

Universell *
Treffer alle elementer i dokumentet. Brukes ofte til en «reset» av margin/padding.
Type p
Treffer alle elementer av en bestemt type (alle <p>, alle <a> osv.).
Klasse .varsel
Treffer alle elementer som har attributtet class="varsel" (eller listen inneholder varsel).
ID #topp
Treffer det ene elementet som har id="topp". ID skal være unik på siden.
* { box-sizing: border-box; }      /* universell */
p { line-height: 1.6; }            /* type       */
.varsel { color: orange; }         /* klasse     */
#topp { background: #f4f1ea; }     /* id         */
MCQ · 1.6.6
Hva treffer den universelle selektoren *?
  • Bare html-elementet
  • Alle elementer i dokumentet
  • Bare elementer uten klasse eller id
  • Tomme elementer
Den universelle selektoren matcher absolutt alt — derfor er den den minst spesifikke.
02 · Klasse vs. ID

Når bruker du klasse, og når bruker du ID?

Klasse og ID er begge attributter på HTML-elementet, men brukes forskjellig:

  • Klasse kan brukes på flere elementer på samme side, og ett element kan ha flere klasser samtidig (mellomrom-separert: class="varsel viktig"). I CSS brukes punktum: .varsel.
  • ID skal være unik per side. Brukes typisk for ett bestemt element du vil peke ut — gjerne som ankerlenke (href="#topp"). I CSS brukes hash: #topp.
Praktisk regel

Bruk klasse som standardvalg. Bruk ID bare når du virkelig peker på ett unikt element — typisk for navigasjon eller layout-landemerker.

MCQ · 1.6.6
Hvilket prefiks brukes for en klasse-selektor i CSS?
  • Hash #
  • Punktum .
  • Kolon :
  • Apenklamme @
Punktum for klasse, hash for ID, kolon for pseudo-klasse (kommer i 1.6.12), @ for at-regler som @media.
MCQ · 1.6.6
Hva er den viktigste forskjellen mellom klasse og ID?
  • Klasser virker bare i Chrome
  • En ID skal være unik per side; en klasse kan brukes på flere elementer
  • ID er for tekst, klasse er for bokser
  • Klasser kan ikke kombineres med andre selektorer
ID-attributtet er unikt — det er derfor det også fungerer som ankermål for href="#…".
03 · Kombinerte selektorer

Etterkommer og barn

Selektorer kan kjedes med mellomrom eller >:

Etterkommer p a
Treffer alle <a> som ligger inni en <p>, uansett hvor dypt.
Direkte barn li > a
Treffer kun <a> som er direkte barn av en <li> — ikke barnebarn.
<ul>
  <li><a>Direkte</a>       <!-- treffes av  li > a  OG  ul a  -->
    <span><a>Dypt</a></span> <!-- treffes kun av  ul a  (ikke li > a) -->
  </li>
</ul>
MCQ · 1.6.6
Hva er forskjellen på p a og p > a?
  • p a treffer alle nivåer dypt; p > a treffer kun direkte barn
  • De er identiske
  • p > a finnes ikke i CSS
  • p a betyr «p eller a»
Mellomrom = etterkommer (alle nivåer). > = barn (kun ett nivå ned).
04 · Søsken-selektorer

Element ved siden av et annet

To selektorer adresserer søsken — elementer som har samme forelder:

Adjacent h1 + p
«Naboen»: den første <p>-en rett etter en <h1>.
General h1 ~ p
Alle <p> som er søsken etter en <h1>, uavhengig av hvor mange søsken som ligger imellom.
<h1>Tittel</h1>
<p>Første</p>     <!-- treffes av  h1 + p  OG  h1 ~ p -->
<p>Andre</p>      <!-- treffes kun av  h1 ~ p  -->
MCQ · 1.6.6
Hva treffer h2 + p?
  • Alle <p> inni <h2>
  • Den første <p> som kommer rett etter en <h2> som søsken
  • Alle <h2> og alle <p>
  • Det er ugyldig CSS
+ er adjacent sibling: kun den nærmeste søsken som matcher.
05 · Gruppering

Komma — flere selektorer, samme regel

Hvis du vil at flere selektorer skal dele samme erklæringsblokk, separerer du dem med komma. Det sparer duplisering. 

h1, h2, h3 {
  font-family: 'Newsreader', Georgia, serif;
  color: #1f4e79;
}

Husk: komma betyr «eller». Det er forskjellig fra mellomrom, som betyr «inni».

MCQ · 1.6.6
Hva betyr h1, h2 { color: red; }?
  • Alle <h2> inni en <h1>
  • Både <h1> og <h2> får farge rød
  • Ugyldig syntaks
  • Bare <h1> får farge
Komma grupperer selektorer slik at samme regel gjelder for hver av dem.
06 · Vanlige feller

Vanlige feller

Pass på
  • Glemmer punktum/hash. varsel { … } uten punktum forsøker å matche en <varsel>-tag (som ikke finnes), ikke klassen.
  • Bytter mellomrom og komma. h1, h2 = «h1 eller h2». h1 h2 = «h2 inni h1». Helt forskjellige resultat.
  • Tror p > * matcher dypt. Det treffer kun direkte barn av <p>. For etterkommere: p *.
  • Bruker ID på flere elementer. HTML-validatoren godtar det ofte, men selektoren #x er tenkt som unik. Bruk klasse i stedet.
07 · Test deg selv

Avsluttende spørsmål

MCQ · Helhet
Selektoren article p matcher hvilke elementer?
  • Alle <p> i hele dokumentet
  • Alle <p> som ligger inni et <article>, uansett dybde
  • Bare <article>
  • Bare <p> som er direkte barn av <article>
Mellomrom er etterkommer-selektor — alle nivåer dypt. For direkte barn ville det vært article > p.
MCQ · Helhet
Hvor mange elementer kan ha samme klasse på en side?
  • Maks ett
  • Maks ti
  • Så mange du vil — det er hele poenget med klasser
  • Bare ett, ellers blir CSS-en ignorert
Klasser er gjenbrukbare etiketter. ID-en er den unike — én ID per side.
MCQ · Helhet
Selektoren div.kort matcher hva?
  • Alle <div> som har klassen kort som første ord
  • <div>-elementer som har klassen kort
  • En <div> inni .kort
  • Det er ugyldig CSS
Når selektorer kjedes uten mellomrom betyr det at samme element må ha begge — her: en div som også har klassen kort.
