01 · Pensum 1.11.7

if, else if, else

Kontrollstruktur betyr «kode som velger hvilken vei programmet skal gå». Det vanligste verktøyet er if-blokken. Den evaluerer uttrykket i parentesen — hvis det er true kjøres koden i krøllparentesene; ellers hoppes den over. 

const alder = 17;

if (alder >= 18) {
  console.log("Voksen");
} else if (alder >= 13) {
  console.log("Tenåring");
} else {
  console.log("Barn");
}
// "Tenåring"

Reglene for kjeden er enkle:

  • Den første betingelsen som er true kjører — resten hoppes over.
  • else if er valgfritt og kan repeteres mange ganger.
  • else er valgfri «catch-all» som kjøres bare hvis ingen andre var true.
Truthy / falsy

JS har «sannhets-aktige» og «falske-aktige» verdier. Disse evalueres som false i en if-test: false, 0, "", null, undefined, NaN. Alt annet er true.

MCQ · 1.11.7
Hva logges? if (0) { console.log("a"); } else { console.log("b"); }
  • "a"
  • "b"
  • Begge
  • Ingen
0 er falsy, så if-blokken hoppes over og else kjøres.
MCQ · 1.11.7
I en lang if / else if-kjede, hva skjer hvis flere betingelser er true?
  • Alle blokkene som matcher kjøres
  • Kun den første som matcher kjøres
  • Kun den siste som matcher kjøres
  • Det blir SyntaxError
Kjeden evalueres ovenfra. Den første true-grenen kjører; resten hoppes over.
02 · Pensum 1.11.8

Arrays — ordnet liste

En array er en ordnet samling av verdier. Den deklareres med firkantparenteser, og hvert element har en indeks som starter på 0: 

const farger = ["rød", "grønn", "blå"];

farger[0];        // "rød"
farger[2];        // "blå"
farger.length;    // 3
farger[farger.length - 1];  // "blå" (siste element)

De viktigste metodene

.length
Antall elementer.
.push(x)
Legg x til slutten.
.pop()
Fjern og returner siste element.
arr[i] = x
Endre verdi på indeks i.
Indeks fra 0

Et array med n elementer har indeksene 0 til n - 1. Det betyr at arr[arr.length] er én utenfor arrayen og gir undefined.

MCQ · 1.11.8
Gitt const a = [10, 20, 30]. Hva returnerer a[a.length]?
  • 30
  • 10
  • undefined
  • RangeError
a.length er 3, og gyldige indekser er 0–2. Merk: pensum 1.11.8 påstår at indeks utenfor området «would give an error» — det er feil i pensum. JavaScript returnerer faktisk undefined uten å kaste exception. Ingen vanlig nettleser eller Node-versjon kaster RangeError her.
MCQ · 1.11.8
Hva gjør arr.push(5)?
  • Setter første element til 5
  • Legger 5 til som siste element
  • Fjerner det siste elementet
  • Sjekker om 5 finnes i arrayet
.push() legger til på slutten og returnerer ny lengde. Motsvaret er .pop().
03 · Pensum 1.11.9

Objekter — nøkkel-verdi-par

Et objekt grupperer relatert data. I motsetning til en array er det ikke ordnet etter posisjon, men etter nøkkel: 

const person = {
  navn: "Anna",
  alder: 22,
  student: true
};

person.navn;          // "Anna" (dot-notasjon)
person["alder"];      // 22     (bracket-notasjon)
person.epost;         // undefined (property fins ikke)

// Legg til ny property:
person.epost = "anna@example.com";

// Endre eksisterende:
person.alder = 23;

Dot vs bracket

  • Dot: obj.navn — kortere, men nøkkelen må være et gyldig identifier-navn.
  • Bracket: obj["navn"] — fungerer alltid, og lar deg bruke variabler som nøkkel: obj[key].
MCQ · 1.11.9
Hvordan legger man til en ny property på et objekt?
  • Bruke obj.add("nyKey", verdi)
  • Bare tilordne den: obj.nyKey = verdi
  • Bruke obj.push({ nyKey: verdi })
  • Det går ikke etter at objektet er deklarert
Tilordning oppretter property om den ikke fins fra før. Det fungerer også på const-objekter (referansen er låst, ikke innholdet).
MCQ · 1.11.9
Når må du bruke bracket-notasjon framfor dot?
  • Når property-en er en string
  • Når nøkkelen er i en variabel, eller har spesialtegn/mellomrom
  • Når objektet er stort
  • Når man henter siste property
Dot krever et statisk, gyldig identifier-navn. Bracket lar deg bruke en variabel (obj[key]) eller en streng som ikke er gyldig som identifier (obj["full navn"]).
04 · Pensum 1.11.10

for-loop

Når du vet på forhånd hvor mange iterasjoner du trenger, bruker du en for-loop. Parentesen har tre deler skilt med semikolon: 

for (let i = 0; i < 5; i++) {
  console.log(i);
}
// 0, 1, 2, 3, 4
  1. Initialisering: let i = 0 — kjøres én gang, før loopen starter.
  2. Betingelse: i < 5 — sjekkes før hver iterasjon. Loopen kjører så lenge dette er true.
  3. Oppdatering: i++ — kjøres etter hver iterasjon.

Iterere gjennom et array

const navn = ["Anna", "Bo", "Cecilie"];
for (let i = 0; i < navn.length; i++) {
  console.log(navn[i]);
}
MCQ · 1.11.10
Hva er korrekt loop-grense for å iterere gjennom hele en array?
  • i <= arr.length
  • i < arr.length
  • i < arr.length + 1
  • i <= arr.length - 1 er feil
Indeks går fra 0 til length - 1. i < arr.length stopper riktig. i <= arr.length går én for langt og gir undefined i siste iterasjon.
04 · Pensum 1.11.10

while-loop

Når du ikke vet hvor mange iterasjoner som trengs, men har en betingelse som skal holde, bruker du en while-loop: 

let svar = "";
while (svar === "") {
  svar = prompt("Hva heter du?");
}

Loopen kjører så lenge betingelsen er true. Hvis betingelsen aldri blir false får du en uendelig loop — typisk bug. Pass på at noe inni loopen endrer det betingelsen sjekker.

Uendelig loop

Skriver du while (true) { … } uten break inni, kjører loopen evig og fryser nettleseren. Bruk break hvis du må avslutte basert på en hendelse inni.

06 · Pensum 1.11.10

break og continue

Inni en loop kan du styre flyten med to nøkkelord:

break
Avbryter hele loopen umiddelbart. Hopper rett til linjen etter loopen.
continue
Hopper over resten av én iterasjon og går videre til neste.
for (let i = 0; i < 10; i++) {
  if (i === 5) break;       // stopper helt når i er 5
  if (i % 2 === 0) continue; // hopper over partall
  console.log(i);
}
// 1, 3
MCQ · 1.11.10
Hva er forskjellen på break og continue?
  • Ingen — de gjør det samme
  • break avbryter hele loopen, continue hopper over én iterasjon
  • continue avbryter hele loopen, break hopper over én iterasjon
  • break finnes bare i switch
break = ut av loopen helt. continue = hopp til neste iterasjon, men loopen fortsetter.
07 · Vanlige feller

Vanlige feller

Pass på
  • Off-by-one: i <= arr.length går én for langt. Skal være i < arr.length.
  • Endre array under iterasjon: .push() i en loop som leser arr.length kan gi uendelig loop.
  • Lese property som ikke fins: gir undefined, ikke feil. Bruk if (obj.x !== undefined) hvis du må sjekke.
  • Glemme oppdatering i while: hvis variabelen i betingelsen aldri endres, blir loopen uendelig.
  • Bruke = i stedet for === i if: if (x = 5) tilordner og er alltid true. Skal være if (x === 5).
08 · Test deg selv

Avsluttende spørsmål

MCQ · Helhet
Hva er det siste elementet i [10, 20, 30]?
  • arr[3]
  • arr[arr.length - 1]
  • arr.length
  • arr.last
Indeksering starter på 0, så siste element ligger på length - 1.
MCQ · Helhet
Når er en for-loop riktig valg framfor while?
  • Når man har et objekt
  • Når man vet antall iterasjoner på forhånd
  • Når man ikke vet antall iterasjoner
  • Når man trenger break
For-loop = kjent antall (f.eks. iterere gjennom array). While-loop = ukjent antall (f.eks. vente på input).
MCQ · Helhet
Hvilken syntaks oppretter en property by med verdien "Oslo" på et eksisterende objekt p?
  • p.add("by", "Oslo")
  • p.push({by: "Oslo"})
  • p.by = "Oslo"
  • p["by"]: "Oslo"
Bare tilordne. Property opprettes om den ikke finnes. p["by"] = "Oslo" fungerer også.
