01 · Pensum 1.11.3

De tre grunnleggende datatypene

JavaScript kjenner mange typer, men pensum 1.11.3 trekker fram tre du møter hele tiden:

String
Tekst i fnutter — "Hei" eller 'Hei'. Begge fungerer; vær konsistent.
Number
Alle tall. 3, 3.14, -7. Ingen forskjell på heltall og desimaltall i JS.
Boolean
true eller false. Brukes i tester og logiske uttrykk.

Strings — konkatenering

+-operatoren slår strings sammen som de er — du må selv legge inn mellomrom hvis du vil ha det: 

const navn = "Anna";
console.log("Hei " + navn + "!"); // Hei Anna!

Numbers — spesielle verdier

  • NaN («Not a Number») — returneres ved umulige beregninger som 0/0 eller Number("abc"). Pussig nok er typeof NaN === "number".
  • Infinity — større enn alle vanlige tall. Infinity + 1 er fortsatt Infinity.
  • % (modulo) — resten etter heltallsdivisjon. 7 % 5 er 2.
MCQ · 1.11.3
Hva er resultatet av "3" + 4?
  • 7
  • "34"
  • NaN
  • SyntaxError
Når én av operandene er en string, gjør + konkatenering — tallet 4 konverteres til "4" og slås sammen til "34".
MCQ · 1.11.3
Hva returnerer 10 % 3?
  • 3.33
  • 3
  • 1
  • 0
Modulo gir resten etter heltallsdivisjon. 10 / 3 = 3 rest 1.
02 · Pensum 1.11.4

Logiske operatorer

Operatorer brukes til å sammenligne og kombinere verdier. Pensum 1.11.4 deler dem i tre grupper:

Sammenlikning

< <= > >=
Mindre/større enn (eventuelt eller lik).
==
Likhet med type-koersjon: "5" == 5 er true.
===
Streng likhet — verdi og type må stemme: "5" === 5 er false.
!=
Ulikhet med type-koersjon.
!==
Streng ulikhet — verdi eller type skiller.
Tommelfingerregel

Bruk === som standard. Bruk == bare hvis du aktivt vil at JS skal konvertere typer for deg.

Logiske kombinasjoner

  • && (AND): begge sider må være true.
  • || (OR): minst én side må være true.
  • ! (NOT): negerer en boolean. !true blir false.
const alder = 22;
const harBillett = true;

if (alder >= 18 && harBillett) {
  console.log("Velkommen inn");
}
MCQ · 1.11.4
Hva returnerer 0 == false?
  • true
  • false
  • NaN
  • TypeError
== gjør type-koersjon: false blir til 0, så 0 == 0 er true. 0 === false ville vært false (ulik type).
MCQ · 1.11.4
Hva returnerer (true || false) && !true?
  • true
  • false
  • undefined
  • SyntaxError
(true || false) er true. !true er false. true && false er false.
03 · Pensum 1.11.5

var, let og const

En variabel er et navn vi gir en verdi så vi kan bruke den senere. JavaScript har tre nøkkelord for å lage variabler — og de oppfører seg ulikt.

var
Funksjons-scopet. Kan re-deklareres. Den «gamle» måten — fra før ES6.
let
Block-scopet (mellom {}). Kan re-tilordnes, men ikke re-deklareres i samme blokk. Innført i ES6.
const
Block-scopet. Kan ikke re-tilordnes. Innført i ES6.
var a = 1;       // funksjons-scopet
let b = 2;       // block-scopet, kan re-tilordnes
const c = 3;     // block-scopet, kan ikke re-tilordnes

b = 20;          // OK
c = 30;          // TypeError: Assignment to constant variable
Misforståelse

const betyr at referansen ikke kan re-tilordnes — ikke at innholdet er låst. En const-array kan fortsatt endres med .push(): const liste = [1,2]; liste.push(3); er gyldig — men liste = [] ville feilet.

Navngivningsregler

  • Ikke et reservert keyword (som if eller return).
  • Ingen mellomrom.
  • Tall er tillatt, men ikke som første tegn.
  • Av punktuasjon er kun $ og _ tillatt.
  • Konvensjon: camelCase — myAge, userName.
MCQ · 1.11.5
Hvilket nøkkelord er blokk-scopet og kan ikke re-tilordnes?
  • var
  • let
  • const
  • final
const = blokk-scopet, ikke re-tilordnes. let er blokk-scopet men kan re-tilordnes. final finnes ikke i JS.
MCQ · 1.11.5
Hva skjer med const liste = [1, 2]; liste.push(3);?
  • Det er gyldig — listen blir [1, 2, 3]
  • Det gir TypeError fordi liste er const
  • Det gir SyntaxError
  • .push finnes ikke på const-arrays
const låser referansen, ikke innholdet. Du kan endre objektet/arrayet, men ikke peke liste til noe nytt.
MCQ · 1.11.5
Hvilket variabelnavn er ugyldig?
  • my_age
  • $value
  • name2
  • 2name
Variabelnavn kan ikke begynne med et tall. $ og _ er tillatt; bokstaver og tall etter første tegn er greit.
04 · Pensum 1.11.5

"use strict" — streng modus

Skriver du "use strict"; øverst i et script (eller øverst i en funksjon) aktiverer du streng modus. Det fanger flere tabber tidlig:

  • U-deklarerte variabler (uten var/let/const) blir feil i stedet for global variabel.
  • Re-deklarering i samme blokk gir feil.
  • Visse «silent» feil blir kastet som unntak.
"use strict";

x = 5;  // ReferenceError: x er ikke deklarert

let x = 5; // OK
Hvorfor

Uten streng modus blir x = 5 automatisk en global variabel. Det er nesten alltid feil — og bugger som er vanskelige å spore. Streng modus tvinger deg til å være tydelig.

05 · Pensum 1.11.6

Når noe ikke fins

JavaScript har to verdier som representerer «ingenting» — og forskjellen er viktig.

undefined
«Det er ikke satt». Variabel deklarert men ikke tildelt verdi. Eller en property som ikke fins på et objekt.
null
«Det er bevisst tomt». Programmereren har eksplisitt satt verdien til ingenting.
let a;             // a er undefined (ikke tildelt)
let b = null;      // b er bevisst tom

const obj = { navn: "Anna" };
obj.alder;         // undefined (property fins ikke)

typeof undefined;  // "undefined"
typeof null;       // "object"   ← klassisk egenart i språket

undefined == null;   // true
undefined === null;  // false (ulik type)
Egenart

typeof null === "object" er en gammel feil i språket som aldri ble fikset. Sjekk for null eksplisitt med x === null.

MCQ · 1.11.6
Hva returnerer typeof null?
  • "null"
  • "undefined"
  • "object"
  • "boolean"
typeof null er "object" — en kjent egenart i JS.
MCQ · 1.11.6
Gitt const p = { navn: "Anna" }, hva returnerer p.alder?
  • null
  • undefined
  • ReferenceError
  • 0
Lesing av en property som ikke fins gir undefined — ingen feil. null brukes bare når noen aktivt har satt det.
06 · Vanlige feller

Vanlige feller

Pass på
  • == i stedet for ===: "" == 0 er true men "" === 0 er false. Bruk === som standard.
  • Tro at const betyr immutable: arrays og objekter kan fortsatt endres innenfra.
  • Glemme let/const uten "use strict": variabelen blir global og kan kollidere med annet.
  • Sjekke for null med typeof: det er "object", ikke "null". Bruk x === null.
  • Variabel-navn med bindestrek: my-age er ikke gyldig — JS tolker - som minus.
07 · Test deg selv

Avsluttende spørsmål

MCQ · Helhet
Hva er forskjellen på == og ===?
  • Ingen forskjell — begge er likhet
  • === krever lik type i tillegg til lik verdi
  • == brukes bare for tall
  • === brukes bare for objekter
== gjør type-koersjon, === krever streng likhet.
MCQ · Helhet
Hvilken kombinasjon stemmer?
  • var = blokk-scopet, let = funksjons-scopet
  • var = funksjons-scopet, let = blokk-scopet
  • const = global-scopet, let = funksjons-scopet
  • Alle tre er global-scopet
var følger funksjonen, let og const følger nærmeste blokk { }.
MCQ · Helhet
Hva er forskjellen på undefined og null?
  • Ingen — de er identiske
  • undefined = ikke satt; null = bevisst tom
  • null kan ikke brukes i variabler
  • undefined finnes bare i ES6+
undefined er JS sin standard for «verdien er ikke satt». null er programmerens måte å si «her skal det stå ingenting».
