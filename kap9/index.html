01 · Helhetsbilde

Hvorfor JavaScript?

Hvorfor

HTML strukturerer, CSS styler. Begge er statiske. JavaScript er programmeringsspråket som lar siden reagere, regne, hente data og endre seg — uten en ny request til tjeneren. Uten JS er en webside bare tekst og dekorasjon.

Forelesningen starter helt på begynnelsen — hvor du skriver JS, datatyper, variabler, kontrollstruktur, arrays, objekter, loops og funksjoner.

Kilde: slides presentasjoner/basic_js.md (delkapittel 1.11.1 — 1.11.11). JSON (1.11.12) og Ajax (1.12) ble ikke gjennomgått i forelesningen og er ikke i pensum.

02 · Mental modell

Et script kjører ovenfra og ned

Modell

Nettleseren leser HTML-dokumentet ovenfra og ned. Når den treffer en <script>-tag, kjøres koden umiddelbart. JavaScript kan referere til og manipulere HTML-elementer, men elementet må være rendret før JS prøver å nå det.

«You can't yell at a baby before it is born.»

Du har tre måter å unngå problemet på:

  • Legg <script> nederst i <body> — DOM-en er da ferdig rendret.
  • Pakk koden inn i window.onload = function() { … }.
  • Skill ut JS i en egen fil og last den med <script src="path/to/script.js"></script>.

Kilde: pensum 1.11.1.

08 · Vanlige feller

Vanlige feller

Eksamensvarsel — unngå var

Foreleser sa rett ut: «var is not common to use anymore. We use let.» Problemet med var er at den tillater re-deklarering av samme variabelnavn — det gjør debugging vanskeligere. Bruk let for variabler som endrer seg, og const for de som ikke gjør det. Sensor vil ikke trekke poeng for korrekt var, men koden din er tryggere og mer idiomatisk uten den.

Pass på
  • == i stedet for ===: "" == 0 er true, men "" === 0 er false. Bruk === hvis du ikke har en grunn til å la JS gjøre type-koersjon.
  • Referere et HTML-element før det er rendret: hvis scriptet ligger i <head> over elementet, finnes elementet ikke ennå. Legg scriptet nederst i <body> eller bruk window.onload.
  • Glemme let/const: uten keyword blir variabelen automatisk global — det er nesten alltid feil.
  • Tro at const betyr immutable: en const-array kan fortsatt endres med .push() — det er bare referansen som er låst.
  • Off-by-one i loop: i <= array.length går én for langt. Skal være i < array.length.
  • Lese et element som ikke fins: en array-indeks utenfor lengden gir feil; en property som ikke fins på et objekt gir undefined (ikke feil).
  • Glemme parentes ved funksjonskall: square er en referanse til funksjonen, square(5) kjører den.
09 · Eksamensscenario

Skriv en funksjon

Oppgave

Skriv en JavaScript-funksjon summerPositive(tall) som tar et array av tall som argument, og returnerer summen av kun de positive (større enn 0). Bruk en for-loop og en if-test.

Forslag til løsning
function summerPositive(tall) {
  let sum = 0;
  for (let i = 0; i < tall.length; i++) {
    if (tall[i] > 0) {
      sum += tall[i];
    }
  }
  return sum;
}

console.log(summerPositive([3, -1, 4, -2, 5])); // 12

Hva sensor sjekker: riktig function-syntaks, parameter brukt (ikke en hardkodet array), korrekte loop-grenser (i < tall.length, ikke <=), if-test mot 0, return utenfor loopen, let/const brukt riktig.

10 · Test deg selv

Helhetsspørsmål

Et utvalg flervalgs- og forklaringsspørsmål. Alle hentet fra pensum 1.11.1 — 1.11.11.

Lett · Datatyper
Hva returnerer typeof "hei"?
  • "text"
  • "string"
  • "char"
  • "object"
typeof returnerer en streng som beskriver typen — for tekst er det "string". Andre verdier: "number", "boolean", "undefined", "object", "function". Pensum 1.11.4.
Lett · Likhet
Hva returnerer "" == 0?
  • true
  • false
  • SyntaxError
  • undefined
== gjør type-koersjon. Tom streng evalueres til false, og 0 evalueres til false — så false == false er true. Med === ville svaret vært false. Pensum 1.11.4.
Lett · Variabler
Hvilken variabeltype er blokk-scopet og kan ikke re-tilordnes?
  • var
  • let
  • const
  • function
const er blokk-scopet og kan ikke re-tilordnes. let er også blokk-scopet, men kan re-tilordnes. var er funksjons-scopet. Pensum 1.11.5.
Middels · Arrays
Gitt const a = ["x", "y", "z"]. Hva er verdien av a[a.length - 1]?
  • "x"
  • "y"
  • "z"
  • undefined
Arrayens lengde er 3, så indeks 3 - 1 = 2, som er det siste elementet ("z"). Husk at indeksering starter på 0. Pensum 1.11.8.
Middels · Objekter
Gitt const p = { navn: "Anna" }. Hva returnerer p.alder?
  • null
  • undefined
  • ReferenceError
  • 0
Å lese en property som ikke fins, gir undefined — ikke feil. Pensum 1.11.9.
Middels · Loops
Hvilket nøkkelord avbryter en loop fullstendig?
  • continue
  • break
  • return
  • stop
break avbryter loopen. continue hopper bare over resten av én iterasjon. Pensum 1.11.10.
Vanskelig · Funksjoner
Hva logges?
function f(x) { return x * x; }
console.log(f);
  • undefined
  • NaN
  • Selve funksjonsdefinisjonen
  • SyntaxError
Uten parentes refererer f til selve funksjonsobjektet, ikke et resultat. f(5) ville logget 25. Pensum 1.11.11.
Forklaring · Lett
Forklar forskjellen på undefined og null.
undefined betyr at en variabel er deklarert, men ikke tildelt en verdi. null er en bevisst tom verdi som programmerer har satt eksplisitt. Begge representerer «fravær av data», men kommer fra forskjellig kilde. Pensum 1.11.6.
Forklaring · Middels
Forklar forskjellen mellom parameter og argument.
En parameter er navnet i funksjonsdeklarasjonen — en variabel funksjonen forventer. Et argument er den faktiske verdien som sendes inn når funksjonen kalles. I function square(x){…} er x parameteren; i square(5) er 5 argumentet. Pensum 1.11.11.
Forklaring · Vanskelig
Forklar hva som skjer hvis du legger en script-tag i <head> som referer til et HTML-element i <body>.
Nettleseren leser HTML ovenfra og ned. Når den treffer scriptet i <head>, er <body> ikke rendret enda — så elementet finnes ikke i DOM-en. Referansen feiler. Løsninger: legg scriptet nederst i <body>, pakk koden i window.onload = function() {…}, eller bruk en ekstern .js-fil som lastes etter DOM. Pensum 1.11.1.

1.11.1 · Hvor skrive JSLett

Hva er de to måtene å plassere JavaScript i et HTML-dokument?

Inline (embedded) inne i en <script>-tag, eller eksternt med <script src="path/to/script.js"></script>. Begge varianter kan stå i <head> eller <body>.

1.11.1 · DOM-rekkefølgeMiddels

Hvorfor kan et script feile hvis det refererer et HTML-element?

Fordi nettleseren leser HTML ovenfra og ned. Hvis scriptet kjøres før elementet er rendret, finnes elementet ikke ennå.

«You can't yell at a baby before it is born.»

1.11.1 · window.onloadLett

Hva gjør window.onload = function(){ … }?

Sørger for at koden inni først kjøres etter at hele DOM-en er ferdig rendret. Brukes for å unngå at scriptet refererer til elementer som ikke fins ennå.

1.11.2 · KommentarerLett

Hva er de to kommentar-syntaksene i JavaScript?

// kommentar — gjelder til linjeslutt.

/* kommentar */ — kan strekke seg over flere linjer.

1.11.3 · DatatyperLett

Hva er de tre grunnleggende datatypene i JavaScript?

String (tekst i fnutter), Number (alle tall), Boolean (true/false).

1.11.3 · StringsLett

Hva gjør +-operatoren mellom to strings?

Konkatenerer dem (slår dem sammen) som de er. "Hei " + "verden" blir "Hei verden" — ingen mellomrom legges til automatisk.

1.11.3 · NumbersMiddels

Hva er NaN, og hvordan oppstår det?

NaN = «Not a Number». Returneres når en numerisk operasjon ikke gir et meningsfullt resultat, f.eks. 0 / 0. Selv om navnet sier «not a number», er typen faktisk number.

1.11.3 · ModuloLett

Hva returnerer 7 % 5?

2. Modulo-operatoren (%) returnerer resten etter heltallsdivisjon.

1.11.4 · LikhetMiddels

Forskjellen på == og ===?

== sjekker likhet med automatisk type-koersjon ("" == 0 er true).

=== sjekker streng likhet — verdiene må ha samme type ("" === 0 er false). Bruk === som standard.

1.11.4 · Logiske operatorerLett

Hva er && og || brukt til?

&& (AND): begge sider må være true for å returnere true.

|| (OR): minst én side må være true.

Begge bruker «short-circuit»: andre operand evalueres ikke hvis svaret allerede er bestemt av første.

1.11.4 · NotLett

Hva gjør !-operatoren?

Negerer en boolean-verdi. !true blir false. !!"hei" konverterer en verdi til boolean ved å negere to ganger — en streng med innhold blir true.

1.11.5 · varMiddels

Hvordan er var scopet?

var er funksjons-scopet: variabelen er synlig i hele funksjonen den ble deklarert i. Også synlig globalt hvis deklarert utenfor en funksjon.

1.11.5 · letMiddels

Hvordan er let scopet?

let er blokk-scopet: variabelen er kun synlig i den nærmeste omsluttende blokken (mellom { og }). Innført i ES6.

1.11.5 · constMiddels

Hva betyr const — er verdien immutable?

const betyr at referansen ikke kan re-tilordnes. Men hvis verdien er en array eller objekt, kan innholdet fortsatt endres med f.eks. .push() eller property-tilordning.

1.11.5 · NavngivningLett

Hvilke regler gjelder for variabelnavn i JavaScript?

Ikke et reservert keyword. Ingen mellomrom. Tall er tillatt, men ikke som første tegn. Av punktuasjon er kun $ og _ tillatt. Konvensjonen er camelCase (myAge).

1.11.6 · undefinedLett

Hva betyr undefined?

En variabel som er deklarert, men ikke har fått en verdi tildelt. Returneres også når man prøver å lese en property som ikke fins på et objekt.

1.11.6 · nullLett

Forskjellen på undefined og null?

undefined = variabelen er deklarert, men ikke tildelt en verdi (JS satte den). null = programmereren har satt verdien til «ingenting» bevisst.

1.11.4 · LikhetMiddels

Hva er forskjellen på "" == 0 og "" === 0?

"" == 0 er true — == gjør type-koersjon (begge evalueres til false-ish).

"" === 0 er false — === krever samme type, og string ≠ number.

1.11.7 · if/elseLett

Hvilken rekkefølge evalueres en if/else if/else-kjede?

Ovenfra og ned. Den første betingelsen som er true kjører — resten hoppes over. else (uten betingelse) er «catch all» og kjøres bare hvis ingen av de andre var true.

1.11.7 · switchMiddels

Hva gjør break i en switch-blokk?

Avslutter switch-blokken så ikke neste case kjører også («fall-through»). Hver case-gren trenger sin egen break hvis den ikke skal renne over i neste.

default er valgfri «catch all» når ingen case matchet.

1.11.8 · ArraysLett

Hvordan henter du første og siste element i en array?

Første: arr[0]. Siste: arr[arr.length - 1] (fordi indeksering starter på 0).

1.11.8 · push/popLett

Hva gjør .push() og .pop() på en array?

.push(x) legger x til som siste element.

.pop() fjerner og returnerer det siste elementet.

1.11.9 · ObjekterLett

Hvordan deklarerer man et objekt i JavaScript?

Med krøllparenteser, og nøkkel-verdi-par adskilt med komma:

const p = { navn: "Anna", alder: 22 };

1.11.9 · TilgangLett

Hva er de to måtene å hente en property fra et objekt på?

Dot-notasjon: obj.navn

Bracket-notasjon: obj["navn"]

Begge gir samme resultat.

1.11.9 · Legge til propertyMiddels

Hvordan legger du til en property på et eksisterende objekt?

Bare tilordne den: obj.nyKey = "verdi"; eller obj["nyKey"] = "verdi";. Property opprettes om den ikke fins fra før.

1.11.10 · for-loopMiddels

Hva er de tre delene i parentesen til en for-loop?

1. Initialisering — let i = 0.

2. Betingelse — i < 10 (loopen kjører så lenge dette er true).

3. Oppdatering — i++ (kjøres etter hver iterasjon).

1.11.10 · whileLett

Når bruker man en while-loop fremfor en for-loop?

Når man ikke vet på forhånd hvor mange iterasjoner som trengs, men har en betingelse som skal holde — f.eks. summere tall til en grense er nådd.

1.11.10 · for...inMiddels

Hva itererer en for...in-loop over?

Over nøklene i et objekt — eller over indeksene i en array. for (let x in obj) { … obj[x] … }. Foreleser viser det både på objekter og arrays.

1.11.10 · break/continueMiddels

Forskjellen på break og continue?

break avslutter hele loopen umiddelbart.

continue hopper over resten av én iterasjon og går til neste.

1.11.11 · FunksjonerLett

Hva er de to måtene å skrive en funksjon på?

Declaration: function navn(p){ … }

Expression: const navn = function(p){ … }

Begge er likeverdige; valg er personlig preferanse. (Foreleser fraråder var — bruk let/const.)

1.11.11 · Parameter vs argumentMiddels

Forskjellen på parameter og argument?

Parameter: navnet i funksjonsdeklarasjonen — variabelen funksjonen forventer.

Argument: verdien som faktisk sendes inn ved kall.

function f(x){…} — x er parameter. f(5) — 5 er argument.

1.11.11 · returnLett

Hva gjør return i en funksjon?

Returnerer verdien som følger etter return til den som kalte funksjonen. Resten av funksjonen kjøres ikke. Uten return returnerer funksjonen undefined.

1.11.9 · deleteLett

Hvordan fjerner du en property fra et objekt?

Med delete-operatoren: delete obj.alder;. Etterpå er obj.alder undefined — som om property-en aldri fantes.

1.11.9 · Metoder i objektMiddels

Hva betyr this i en metode på et objekt?

Referansen til objektet metoden ble kalt på. p.hilsen() → inni hilsen peker this på p, slik at this.navn er p.navn.

const p = {
  navn: "Ada",
  hilsen: function() {
    return "Hei " + this.navn;
  }
};
1.11.9 · KonstruktørfunksjonVanskelig

Hva gjør new foran en konstruktørfunksjon?

Lager et nytt objekt, setter this til det nye objektet inne i funksjonen, og returnerer objektet automatisk.

function Teacher(first, last) {
  this.firstName = first;
  this.lastName = last;
}
const t = new Teacher("K", "S");

Konvensjon: konstruktørfunksjoner får stor forbokstav.

Date · KonstruktørLett

Hva er forskjellen på new Date() og new Date(2025, 11, 25)?

new Date() gir nåtidspunktet.

new Date(2025, 11, 25) gir 25. desember 2025 — måned er 0-indeksert, så 11 er desember.

Andre former: new Date("2025-12-25"), new Date(millisekunder).

Date · Differanse til julVanskelig

Hvordan regner du antall dager til neste 25. desember?

Subtraher to Date-objekter → får differansen i millisekunder. Del på ms/dag og rund opp:

const now = new Date();
const xmas = new Date(now.getFullYear(), 11, 25);
const ms = xmas - now;
const dager = Math.ceil(ms / (1000*60*60*24));

Foreleser har trukket fram dette eksempel for eksamen.

Date · GettersMiddels

Hvilke getter-metoder finnes på et Date-objekt?

getFullYear() (yyyy), getMonth() (0–11), getDate() (1–31), getDay() (0–6, søn=0), getHours(), getMinutes(), getSeconds(), getMilliseconds(), getTime() (ms siden 1970).
