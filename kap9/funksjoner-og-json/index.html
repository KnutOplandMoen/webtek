01 · Pensum 1.11.11

Hvorfor funksjoner?

En funksjon er en navngitt blokk med kode du kan kalle igjen og igjen. I stedet for å gjenta de samme tre linjene fem steder, skriver du dem én gang og kaller funksjonen fem ganger.

To grunner til å bruke funksjoner:

  1. Gjenbruk: samme kode trenger bare å skrives én gang.
  2. Lesbarhet: et godt funksjonsnavn forteller leseren hva koden gjør, ikke bare hvordan.
Mental modell

En funksjon er en maskin med inn- og utgang. Den tar inn noen verdier (argumenter), gjør noe med dem, og kan returnere et resultat.

02 · Pensum 1.11.11

To måter å skrive en funksjon

Function declaration

function square(x) {
  return x * x;
}

square(5); // 25

Function expression

const square = function(x) {
  return x * x;
};

square(5); // 25

Begge oppfører seg likt for våre formål. Velg den du synes er ryddigst. Pensum nevner at declaration-formen «hoistes» til toppen av scopet, men det er en detalj du kan kapsle inn ved å deklarere før du kaller.

MCQ · 1.11.11
Hvilken syntaks er en gyldig function expression?
  • function square(x) { return x * x; }
  • const square = function(x) { return x * x; };
  • square: function(x) { return x * x; }
  • def square(x) { return x * x; }
Function expression: en anonym funksjon tilordnes en variabel. Function declaration starter med function og har et navn direkte.
03 · Pensum 1.11.11

Parameter vs argument

To begreper som ofte forveksles — pensum vil at du holder dem fra hverandre:

Parameter
Navnet i funksjonsdeklarasjonen. En variabel funksjonen forventer å motta.
Argument
Den faktiske verdien som sendes inn når funksjonen kalles.
function greet(navn) {  // navn er PARAMETER
  console.log("Hei " + navn);
}

greet("Anna");          // "Anna" er ARGUMENT
Huskeregel

Parameter står i deklarasjonen — som en plassholder. Argument er den aktuelle verdien ved kall.

MCQ · 1.11.11
I function add(a, b) { return a + b; } add(3, 5);, hva er 3 og 5?
  • Parametere
  • Argumenter
  • Variabler
  • Konstanter
a og b er parametere (i deklarasjonen). 3 og 5 er argumenter (verdiene som sendes inn).
04 · Pensum 1.11.11

return — gi noe tilbake

return avslutter funksjonen og sender en verdi tilbake til den som kalte. Resten av funksjonen kjøres ikke. Uten return returnerer funksjonen undefined. 

function square(x) {
  return x * x;
  console.log("denne kjøres aldri");
}

const a = square(4); // a = 16

function logge(x) {
  console.log(x);
  // ingen return — returnerer undefined
}

const b = logge("hei"); // b = undefined

Funksjonsnavn vs funksjonskall

square (uten parentes) er en referanse til selve funksjonen. square(5) (med parentes) kjører funksjonen og gir resultatet: 

console.log(square);    // Selve funksjonsdefinisjonen
console.log(square(5)); // 25
MCQ · 1.11.11
Hva returnerer en funksjon uten return-uttrykk?
  • null
  • 0
  • ""
  • undefined
Standard returverdi er undefined. Det er det samme som om funksjonen avsluttet med en bar return;.
MCQ · 1.11.11
Hva logges? function f() { return 1; return 2; } console.log(f());
  • 1
  • 2
  • 1, 2
  • SyntaxError
Første return avslutter funksjonen umiddelbart — andre return nås aldri.
05 · Pensum 1.11.9 + 1.11.11

Metoder i objekter

En funksjon som er lagret som en property på et objekt kalles en metode. Inni metoden refererer this til objektet metoden ble kalt på, slik at den kan lese eller endre objektets egne properties: 

const teacher = {
  firstName: "Kshitij",
  lastName: "Sharma",
  nickName: function() {
    return this.firstName.charAt(0) + this.firstName.charAt(1);
  }
};

console.log(teacher.nickName()); // "Ks"
Mental modell

this er ikke parameter — det blir automatisk satt til objektet før prikken når metoden kalles. teacher.nickName() → inni er this = teacher.

MCQ · 1.11.9
Hva refererer this til inni en metode kalt med obj.hilsen()?
  • Det globale window-objektet
  • Objektet obj (det før prikken)
  • Forrige funksjon i kallstakken
  • Selve metoden hilsen
this settes til det som står før prikken i kallet. Inni obj.hilsen() er this === obj.
06 · Pensum 1.11.9 + 1.11.11

Konstruktørfunksjoner med new

Skal du lage mange objekter med samme struktur, kan du skrive en konstruktørfunksjon. Konvensjonen er stor forbokstav. Bruk new foran kallet — det lager et nytt tomt objekt, setter this til det, og returnerer det automatisk. 

function Teacher(first, last, course, year) {
  this.firstName = first;
  this.lastName  = last;
  this.course    = course;
  this.numYear   = year;
}

const t1 = new Teacher("Kshitij", "Sharma", "IT2805", 2024);
const t2 = new Teacher("Magnus", "Ouren",  "IT2805", 2024);

console.log(t1.firstName); // "Kshitij"

Konstruktøren er en helt vanlig funksjon — det er new som gjør den til en «factory». Glemmer du new, vil this peke på det globale objektet (eller være undefined i strict mode), og du får ingen ny instans.

MCQ · 1.11.9
Hva returnerer en konstruktørfunksjon kalt med new?
  • undefined
  • Det nye objektet som ble laget
  • Et tall (antall properties)
  • Konstruktøren selv (referansen)
new lager objektet og returnerer det automatisk — du trenger ikke skrive return.
07 · Vanlige feller

Vanlige feller

Pass på
  • Glemme parentes ved funksjonskall: square er en referanse; square(5) kjører funksjonen.
  • Glemme return: uten return får kalleren bare undefined, selv om funksjonen beregner noe.
  • Forveksle parameter og argument: sensor sjekker at du bruker begrepene riktig.
