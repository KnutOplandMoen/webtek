01 · Pensum 1.11.11

Hvorfor funksjoner?

En funksjon er en navngitt blokk med kode du kan kalle igjen og igjen. I stedet for å gjenta de samme tre linjene fem steder, skriver du dem én gang og kaller funksjonen fem ganger.

To grunner til å bruke funksjoner:

  1. Gjenbruk: samme kode trenger bare å skrives én gang.
  2. Lesbarhet: et godt funksjonsnavn forteller leseren hva koden gjør, ikke bare hvordan.
Mental modell

En funksjon er en maskin med inn- og utgang. Den tar inn noen verdier (argumenter), gjør noe med dem, og kan returnere et resultat.

02 · Pensum 1.11.11

To måter å skrive en funksjon

Function declaration

function square(x) {
  return x * x;
}

square(5); // 25

Function expression

const square = function(x) {
  return x * x;
};

square(5); // 25

Begge oppfører seg likt for våre formål. Velg den du synes er ryddigst. Pensum nevner at declaration-formen «hoistes» til toppen av scopet, men det er en detalj du kan kapsle inn ved å deklarere før du kaller.

MCQ · 1.11.11
Hvilken syntaks er en gyldig function expression?
  • function square(x) { return x * x; }
  • const square = function(x) { return x * x; };
  • square: function(x) { return x * x; }
  • def square(x) { return x * x; }
Function expression: en anonym funksjon tilordnes en variabel. Function declaration starter med function og har et navn direkte.
03 · Pensum 1.11.11

Parameter vs argument

To begreper som ofte forveksles — pensum vil at du holder dem fra hverandre:

Parameter
Navnet i funksjonsdeklarasjonen. En variabel funksjonen forventer å motta.
Argument
Den faktiske verdien som sendes inn når funksjonen kalles.
function greet(navn) {  // navn er PARAMETER
  console.log("Hei " + navn);
}

greet("Anna");          // "Anna" er ARGUMENT
Huskeregel

Parameter står i deklarasjonen — som en plassholder. Argument er den aktuelle verdien ved kall.

MCQ · 1.11.11
I function add(a, b) { return a + b; } add(3, 5);, hva er 3 og 5?
  • Parametere
  • Argumenter
  • Variabler
  • Konstanter
a og b er parametere (i deklarasjonen). 3 og 5 er argumenter (verdiene som sendes inn).
04 · Pensum 1.11.11

return — gi noe tilbake

return avslutter funksjonen og sender en verdi tilbake til den som kalte. Resten av funksjonen kjøres ikke. Uten return returnerer funksjonen undefined. 

function square(x) {
  return x * x;
  console.log("denne kjøres aldri");
}

const a = square(4); // a = 16

function logge(x) {
  console.log(x);
  // ingen return — returnerer undefined
}

const b = logge("hei"); // b = undefined

Funksjonsnavn vs funksjonskall

square (uten parentes) er en referanse til selve funksjonen. square(5) (med parentes) kjører funksjonen og gir resultatet: 

console.log(square);    // Selve funksjonsdefinisjonen
console.log(square(5)); // 25
MCQ · 1.11.11
Hva returnerer en funksjon uten return-uttrykk?
  • null
  • 0
  • ""
  • undefined
Standard returverdi er undefined. Det er det samme som om funksjonen avsluttet med en bar return;.
MCQ · 1.11.11
Hva logges? function f() { return 1; return 2; } console.log(f());
  • 1
  • 2
  • 1, 2
  • SyntaxError
Første return avslutter funksjonen umiddelbart — andre return nås aldri.
05 · Pensum 1.11.12

JSON — JavaScript Object Notation

JSON er et tekstformat for å representere data. Det er bygget på JS-objekter, men er strengere — og brukes uavhengig av JS som standard for å sende data mellom systemer. 

{
  "navn": "Anna",
  "alder": 22,
  "kurs": ["IT2805", "TDT4140"],
  "aktiv": true
}

Forskjellen fra et JS-objekt

  • Nøkler må stå i doble fnutter — "navn", ikke navn.
  • Strenger må bruke doble fnutter — "Anna", ikke 'Anna'.
  • Ingen kommentarer tillatt.
  • Kun enkle data: string, number, boolean, null, array, objekt. Ingen funksjoner, ingen variabler, ingen beregninger.
Hvorfor

JSON er strengt fordi det skal kunne leses pålitelig av programmer skrevet i ethvert språk — Python, Java, C# — ikke bare JavaScript. Strengheten er funksjonen.

MCQ · 1.11.12
Hvilket av disse er ikke tillatt i JSON?
  • Tall
  • Strenger med doble fnutter
  • Arrays
  • Kommentarer
JSON er enkle data — ingen kommentarer, ingen funksjoner, ingen variabler.
MCQ · 1.11.12
Hvilken nøkkel-syntaks er gyldig JSON?
  • navn: "Anna"
  • 'navn': 'Anna'
  • "navn": "Anna"
  • navn = "Anna"
Doble fnutter rundt både nøkkel og string-verdi. Enkle fnutter er ikke tillatt i JSON.
06 · Pensum 1.11.12

JSON.stringify og JSON.parse

JS har to innebygde funksjoner for å oversette mellom JS-objekter og JSON-strenger:

JSON.stringify(obj)
Tar et JS-objekt → returnerer en JSON-streng. Brukes f.eks. når du sender data til en server.
JSON.parse(str)
Tar en JSON-streng → returnerer et JS-objekt. Brukes når du mottar data fra en server.
const person = { navn: "Anna", alder: 22 };

const tekst = JSON.stringify(person);
// '{"navn":"Anna","alder":22}'

const tilbake = JSON.parse(tekst);
// { navn: "Anna", alder: 22 }
tilbake.navn; // "Anna"
Pass på

JSON.parse kaster SyntaxError hvis strengen ikke er gyldig JSON. Bruk try/catch hvis du tar imot data du ikke stoler på.

MCQ · 1.11.12
Hvilken funksjon brukes for å gjøre et JS-objekt om til en streng som kan sendes over nett?
  • JSON.stringify
  • JSON.parse
  • obj.toString()
  • String(obj)
JSON.stringify serialiserer objektet til JSON-tekst. JSON.parse går motsatt vei.
07 · Vanlige feller

Vanlige feller

Pass på
  • Glemme parentes ved funksjonskall: square er en referanse; square(5) kjører funksjonen.
  • Glemme return: uten return får kalleren bare undefined, selv om funksjonen beregner noe.
  • Forveksle parameter og argument: sensor sjekker at du bruker begrepene riktig.
  • Enkle fnutter i JSON: ikke gyldig. Doble fnutter alltid.
  • Kommentarer i JSON: JSON.parse vil feile.
  • Komma etter siste element: {"a": 1,} er ikke gyldig JSON. JS-objekter tillater det.
08 · Test deg selv

Avsluttende spørsmål

MCQ · Helhet
Hva er hovedforskjellen på et JS-objekt og JSON?
  • Ingen forskjell
  • JSON er strengere — doble fnutter på alle nøkler, ingen kommentarer, kun enkle data
  • JS-objekter kan kun ha tall som verdier
  • JSON kan inneholde funksjoner, JS-objekter kan ikke
JSON er en strengere undermengde av JS-objekt-syntaks, designet for trygg utveksling mellom systemer.
MCQ · Helhet
Hva logges? const f = function(n) { return n + 1; }; console.log(f);
  • undefined
  • NaN
  • Selve funksjonsdefinisjonen
  • SyntaxError
Uten parentes refererer f til selve funksjonen, ikke et resultat. f(5) ville logget 6.
MCQ · Helhet
Hvilken funksjon parser tekst-strengen '{"a":1}' til et JS-objekt?
  • JSON.stringify
  • JSON.parse
  • eval
  • Object.from
JSON.parse tar JSON-streng → JS-objekt. JSON.stringify går motsatt vei.
