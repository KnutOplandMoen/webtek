Spørsmål 1 3 poeng Kap. 1 — HTTP-metoder

Hvilken av følgende HTTP-metoder er idempotent — altså gir samme tilstand på serveren uansett hvor mange ganger den kalles?

  • A POST
  • B GET
  • C CONNECT
  • D PATCH
Vis fasit
Riktig svar: B

GET er idempotent — å hente samme ressurs ti ganger endrer ikke tilstanden på serveren. Det samme gjelder PUT og DELETE (kall N ganger gir samme resultat som ett kall).

Hvorfor de andre er feil: POST oppretter typisk en ny ressurs hver gang (to like POST-er kan lage to like rader). PATCH endrer delvis og er ikke garantert idempotent. CONNECT brukes til tunneler og er ikke idempotent.

Spørsmål 2 2 poeng Kap. 9 — Likhets-operatorer

Hva er hovedforskjellen mellom == og === i JavaScript?

  • A == sammenligner referanser; === sammenligner verdier.
  • B == utfører type-coercion før sammenligning; === krever samme type og samme verdi.
  • C === brukes bare for strenger; == for alle andre typer.
  • D De er identiske; === er bare en gammel skrivemåte.
Vis fasit
Riktig svar: B

=== ("strict equality") krever at både type og verdi er like — derfor er 0 === "0" falskt og 1 === true falskt. == ("loose equality") konverterer typer først, så 0 == "0" er sant og 1 == true er sant.

Hvorfor de andre er feil: A blander med Object.is / referanse-sammenligning. C er konfabulasjon — begge fungerer for alle typer. D er feil — semantikken er reelt forskjellig, derfor anbefales === i moderne JS.

Spørsmål 7 3 poeng Kap. 10 — Event-kontroll

Hva er forskjellen mellom event.preventDefault() og event.stopPropagation()?

  • A preventDefault() stanser eventets bubbling til foreldre-elementer; stopPropagation() hindrer nettleserens standardoppførsel.
  • B preventDefault() hindrer nettleserens standardoppførsel (f.eks. skjema-submit eller lenke-navigering); stopPropagation() hindrer eventet i å boble videre til foreldre.
  • C Begge metodene gjør det samme — den ene er bare et alias for den andre.

Vis fasit
Riktig svar: B

Spørsmål 11 4 poeng Kap. 9 — parseInt

Hva skriver dette ut?

console.log(parseInt("3.7px"));
Vis fasit
Riktig svar: 3

Mekanisme: parseInt leser tegn fra venstre og stopper ved første ikke-gyldige tegn for et heltall. "3" er gyldig, "." er ikke et heltallstegn — så parsing stopper og resultatet er 3.

Kontraster å huske:
parseFloat("3.7px") → 3.7 (godtar desimaltegn).
Number("3.7px") → NaN (krever at hele strengen er et gyldig tall).
parseInt("abc") → NaN (ingen gyldige tegn i starten).

Spørsmål 12 4 poeng Kap. 9 — for...in vs verdier

Hva skriver dette ut (på én linje, mellomrom mellom)?

let result = "";
for (let x in [10, 20, 30]) {
  result += x + " ";
}
console.log(result.trim());
Vis fasit
Riktig svar: 0 1 2

Mekanisme — den klassiske for-in-fellen: for...in itererer over nøklene/indeksene i et objekt eller array. For arrayet [10, 20, 30] er nøklene strengene "0", "1", "2" — ikke verdiene.

Kontrast — for...of: Hadde koden brukt for (let x of [10, 20, 30]), ville outputen blitt 10 20 30. Tommelfingerregel: in = indeks/key, of = verdi. For arrays foretrekkes nesten alltid for...of eller forEach.

Spørsmål 13 3 poeng Kap. 9 — Closures

Hva skriver dette ut (siste linje)?

function lagTeller() {
  let n = 0;
  return function () {
    n += 1;
    return n;
  };
}
const tell = lagTeller();
tell();
tell();
console.log(tell());
Vis fasit
Riktig svar: 3

Mekanisme — closure: Den indre funksjonen "lukker" rundt variabelen n fra ytre scope. Selv etter at lagTeller() har returnert, holder den indre funksjonen liv i n. Hver tell()-kall øker samme n med 1: 1, 2, 3.

Vanlig misforståelse: Mange tror at n resettes til 0 ved hvert kall, fordi let n = 0 står inni lagTeller. Men lagTeller kjøres bare én gang — den returnerer en funksjon som hver gang refererer til samme lukkede n.

Spørsmål 14 4 poeng Kap. 9 — Boolean coercion

Hva skriver dette ut?

console.log(Boolean("false") && !!"0");
Vis fasit
Riktig svar: true

Mekanisme: Begge uttrykk konverterer en ikke-tom streng til boolean. I JS er alle ikke-tomme strenger truthy — inkludert "false" og "0". Tegnene betyr ingenting for konverteringen; bare lengde > 0.

  • Boolean("false") → true (strengen er ikke tom).
  • !!"0" → true (samme prinsipp; dobbel-negasjon er en kortform for boolean-cast).
  • true && true → true.

Falsy-verdier i JS: false, 0 (nummer, ikke streng!), "" (tom streng), null, undefined, NaN. Alt annet er truthy.

Forklar forskjellen mellom getElementById, getElementsByClassName og querySelector / querySelectorAll. Hva returnerer hver, og når foretrekker du hva?

Vis fasit

Rubrikk (4 poeng):

Spørsmål 17 3 poeng Kap. 4 — Skjuling av elementer

Forklar forskjellen mellom display: none, visibility: hidden og opacity: 0. Hva skjer med plassen elementet opptar, og kan brukeren klikke på det?

Vis fasit

Rubrikk (3 poeng — 1 per verdi):

Spørsmål 18 3 poeng Kap. 9 — let vs var

Forklar tre konkrete forskjeller mellom let og var i JavaScript.

Vis fasit

Sensor leter etter minst tre av disse (1 poeng hver, maks 3):

  • Scope: let har blokk-scope ({ ... }), var har function-scope. let x inni en if-blokk er ikke synlig utenfor; en var x samme sted er synlig i hele funksjonen.
  • Hoisting: var-deklarasjoner heves til toppen av funksjonen og initialiseres til undefined (kan brukes før deklarasjons-linja, men er undefined). let heves også, men er i en "temporal dead zone" — bruker du den før deklarasjonen får du en ReferenceError.
  • Redeklarering: var x = 1; var x = 2; i samme scope er lovlig. let x = 1; let x = 2; kaster SyntaxError.
  • Global property: Globalt deklarert var foo blir window.foo i nettleseren; globalt let foo blir ikke en property på window-objektet.

Anbefaling: I moderne JS brukes const som standard, let når verdien skal endres, og var aldri.

Spørsmål 20 3 poeng Kap. 1 — HTTP-metoder

Beskriv kort hva de fire HTTP-metodene GET, POST, PUT og DELETE brukes til. Bruk gjerne én setning per metode.

Vis fasit

Rubrikk (3 poeng — 0.75 per metode):

  • GET — hente en ressurs uten å endre serveren. Idempotent. Brukes for lesning/visning (f.eks. åpning av en nettside, søk, filtre).
  • POST — sende data for å opprette en ny ressurs eller utløse en handling. Ikke idempotent (to POST-er = to nye rader). Brukes for skjema-innsending, ny bruker, nytt innlegg.
  • PUT — erstatte en ressurs helt med en ny versjon. Idempotent: samme PUT N ganger gir samme tilstand. Brukes for full oppdatering.
  • DELETE — slette en ressurs. Idempotent: slett-én-gang gir samme resultat som slett-N-ganger (ressursen er borte uansett).

Spørsmål 21 8 poeng Kap. 10 — Tegn-teller

HTML-en under er gitt. Skriv JavaScript som lager en tegn-teller for textarea-feltet:

  1. (3 poeng) Mens brukeren skriver i textarea (event input eller keyup), oppdater <span id="count"> til å vise antall tegn brukeren har skrevet.
  2. (3 poeng) Hvis antall tegn overstiger 100, sett tellerens tekstfarge til rød (red) ved hjelp av style.color. Når tallet faller under 100 igjen, settes fargen tilbake til sort/default.
  3. (2 poeng) Initialiser telleren slik at den viser 0 ved sideinnlasting (uten at brukeren har skrevet noe).
Vis fasit

Referanseløsning:

const melding = document.getElementById("melding");
const teller = document.getElementById("count");

function oppdater() {
  const n = melding.value.length;
  teller.textContent = n;
  teller.style.color = (n > 100) ? "red" : "";
}

melding.addEventListener("input", oppdater);
oppdater();  // initialiser ved load

Kommentarer:

  • Event input vs keyup: input trigger ved enhver verdiendring (også paste, drag-drop, autocomplete). keyup trigger bare for tastetrykk — så hvis brukeren limer inn tekst, kan telleren bli feil. Begge er godkjent i denne oppgaven, men input er det robuste valget.
  • Initialisering: Å kalle oppdater() én gang ved load sørger for at telleren viser 0 fra start. Alternativt kan man sette teller.textContent = "0" eksplisitt.
  • Tilbakestilling av farge: Sett style.color = "" (tom streng) for å fjerne inline-stilen og falle tilbake til CSS-arv. Hadde man satt "black" hardkodet, ville evt. tema-CSS bli overstyrt.

Vanlige feil: Bruke onkeyup-attribute inline i HTML (godtatt, men ikke moderne). Glemme initial-oppdateringen så telleren er tom inntil første tastetrykk. Sjekke === 100 i stedet for > 100 — da blir den aldri rød.

Spørsmål 22 10 poeng Kap. 8+9 — Kalkulator

HTML-en under er gitt. Skriv JavaScript som lager en enkel kalkulator med fire knapper:

  1. (3 poeng) Når en av knappene klikkes, les verdiene fra #a og #b, konverter til tall, og skriv resultatet til <span id="res">.
  2. (3 poeng) Knappene støtter operasjonene: +, −, ×, ÷. Bruk en data-attributt (data-op) for å skille mellom dem, og en felles handler (ikke fire separate funksjoner).
  3. (2 poeng) Ved divisjon med 0, skriv teksten "Kan ikke dele på 0" i resultat-feltet i stedet for Infinity.
  4. (2 poeng) Hvis ett av feltene ikke er et gyldig tall (f.eks. tomt eller "abc"), skriv "Ugyldig input".
Vis fasit

Referanseløsning (én idiomatisk variant):

const knapper = document.querySelectorAll(".op");
const res = document.getElementById("res");

knapper.forEach(function (btn) {
  btn.addEventListener("click", function () {
    const a = parseFloat(document.getElementById("a").value);
    const b = parseFloat(document.getElementById("b").value);
    if (isNaN(a) || isNaN(b)) {
      res.textContent = "Ugyldig input";
      return;
    }
    const op = btn.dataset.op;
    let svar;
    switch (op) {
      case "+": svar = a + b; break;
      case "-": svar = a - b; break;
      case "*": svar = a * b; break;
      case "/":
        if (b === 0) { res.textContent = "Kan ikke dele på 0"; return; }
        svar = a / b;
        break;
    }
    res.textContent = svar;
  });
});

Kommentarer:

  • Felles handler: Vi itererer over alle knapper med querySelectorAll(".op") og fester samme handler — operasjonen avgjøres av btn.dataset.op (som leser data-op-attributtet). Dette er DRY (don't repeat yourself).
  • parseFloat vs Number: parseFloat er forgivende ("3.7px" → 3.7). Number("3.7px") ville gitt NaN. Begge er gyldige for denne oppgaven — viktigste er isNaN-sjekken.
  • Division-by-zero: JS gir Infinity (ikke en exception) ved x / 0. Vi må sjekke eksplisitt for b === 0.
  • Alternative tilnærminger: Et objekt-lookup som const ops = { "+": (a,b) => a+b, ... } er en elegant variant — men switch er like korrekt.

Vanlige feil: Glemme parseFloat så "5" + "3" blir "53" (string-concat). Lage fire separate funksjoner — fungerer, men er ikke det oppgaven ber om. Skrive btn.getAttribute("data-op") i stedet for btn.dataset.op (begge funker, første er mer verbose).

});

Kommentarer:

  • Backdrop-klikk: Sjekken e.target === modal er nøkkelen. Klikker du på .modal-box inni, bobler klikket opp til #modal, men e.target er fortsatt .modal-box — så sjekken filtrerer bort interne klikk. Alternativt kunne man kalt e.stopPropagation() på modal-box-klikk.
  • Escape-key: e.key er den moderne string-baserte API-en ("Escape", "Enter", "ArrowLeft"). Den gamle e.keyCode (27) er deprecated.
  • classList.add/remove: Mer leselig enn å sette .className = "..." manuelt, og det fjerner ikke andre klasser ved et uhell.

Vanlige feil: Glemme sjekken på backdrop og lukke modalen ved klikk inni boksen også. Lytte på keypress i stedet for keydown (Escape sender ikke keypress). Festes Escape-handler på modal i stedet for document — fungerer ikke når modalen er lukket og ikke har fokus.

Spørsmål 24 8 poeng Kap. 10 — Bilde-galleri

HTML/CSS for et lite galleri er gitt. Skriv JavaScript som lar brukeren bytte hoved-bildet ved å klikke på en thumbnail:

  1. (3 poeng) Når brukeren klikker på en thumbnail (.thumb), skal src-attributtet til <img id="stor"> bytte til samme bildet (les src fra thumbnailen).
  2. (2 poeng) Den klikkede thumbnailen får klassen active; alle andre mister den. Default skal første thumbnail være aktiv.
  3. (2 poeng) Bruk event delegation: én listener på .gallery-containeren, ikke én listener per thumbnail.
  4. (1 poeng) Bruk addEventListener (ikke inline onclick).
Vis fasit

Referanseløsning:

const galleri = document.querySelector(".gallery");
const stor = document.getElementById("stor");

galleri.addEventListener("click", function (e) {
  if (!e.target.classList.contains("thumb")) return;

  // Bytt hoved-bilde
  stor.src = e.target.src;

  // Oppdater active-klassen
  galleri.querySelectorAll(".thumb").forEach(function (t) {
    t.classList.remove("active");
  });
  e.target.classList.add("active");
});

Kommentarer:

  • Event delegation: Én listener på .gallery — vi sjekker e.target.classList.contains("thumb") for å filtrere. Hvis vi seinere legger til flere thumbnails dynamisk, fungerer det fortsatt uten ekstra arbeid.
  • src-attributtet: e.target.src gir den absolutte URL-en til thumbnailen — perfekt for å overføre direkte til stor.src.
  • Oppdater active: Vi rydder bort active fra alle, og legger den til på den klikkede. Alternativt kan man bruke galleri.querySelector(".active").classList.remove("active"), men da må man være sikker på at det alltid finnes én.

Vanlige feil: Feste klikk-handler på hver .thumb (fungerer, men ikke event delegation som oppgaven ber om). Glemme å fjerne active fra de andre — så alle blir aktive til slutt. Skrive til stor.setAttribute("src", ...) — fungerer, men .src er enklere.

HTML er gitt. Skriv kode som lar brukeren bytte mellom lyst og mørkt tema:

  1. (2 poeng) Skriv CSS-regler slik at body.dark får mørk bakgrunn (#222) og lys tekst (#eee). Default er lys.
  2. (3 poeng) Klikk på #bytt toggler klassen dark på body (legg til hvis den ikke er der, fjern hvis den er).
  3. (1 poeng) Knappens tekst skal vise nåværende handling: "Bytt til mørkt" når lyst er aktivt, "Bytt til lyst" når mørkt er aktivt.
Vis fasit

Referanseløsning:

/* CSS */
body.dark {
  background-color: #222;
  color: #eee;
}

/* JS */
const knapp = document.getElementById("bytt");

knapp.addEventListener("click", function () {
  const erMork = document.body.classList.toggle("dark");
  knapp.textContent = erMork ? "Bytt til lyst" : "Bytt til mørkt";
});

Kommentarer:

  • classList.toggle() returnerer boolean: true hvis klassen ble lagt til, false hvis den ble fjernet. Perfekt for å vite hvilken tilstand vi er i etter toggle.
  • Alternativ med contains: 
    if (document.body.classList.contains("dark")) {
  document.body.classList.remove("dark");
  knapp.textContent = "Bytt til mørkt";
} else {
  document.body.classList.add("dark");
  knapp.textContent = "Bytt til lyst";
}
    Mer verbose, men også korrekt.
  • CSS-arv: Når vi setter color på body, arver alle barn (h2, p) den med mindre de overstyrer den selv.

Vanlige feil: Hardkode at klassen alltid legges til (uten å sjekke nåværende tilstand) — etter første klikk gjør den ingenting nytt. Skrive om hele className-strengen i stedet for å bruke classList — kan overskrive andre klasser.
