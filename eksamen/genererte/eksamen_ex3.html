Hvilket symbol identifiserer at HREF-attributtet peker til et "NAME anchor" (et anker på samme side) i stedet for en adresse eller filnavn?

  • A $
  • B #
  • C &
  • D !
Vis fasit
Riktig svar: B

  • A inherit
  • B fixed
  • C absolute
  • D static
Vis fasit

Spørsmål 4 (A4) 3 poeng Kap. 9 — Objekt-property-sjekk

Hvilken JavaScript-operator brukes for å sjekke om en gitt property finnes på et objekt?

  • B is in
  • C in
  • D lies
Vis fasit
Riktig svar: C

in er en JavaScript-operator: "navn" in personObjekt returnerer true hvis objektet (eller dets prototype-kjede) har en property med det navnet.

Spørsmål 9 (A9) 3 poeng Kap. 9 — Array-rotasjon

Hva blir resultatet av koden under?

function solve(arr, rotations) {
  if (rotations == 0) return arr;
  for (let i = 0; i < rotations; i++) {
    let element = arr.pop();
    arr.unshift(element);
  }
  return arr;
}
solve([44, 1, 22, 111], 5);
  • A [111, 44, 1, 22]
  • B [111, 1, 22, 44]
  • C [44, 1, 22, 111]
  • D [1, 22, 111, 44]
Vis fasit
Riktig svar: D

Hver iterasjon: arr.pop() tar siste element, arr.unshift() legger det først. Effektivt: roterer arrayet høyre→venstre ett steg.

Trace ([44, 1, 22, 111], 5 rotasjoner):

  • Start: [44, 1, 22, 111]
  • 1: pop 111, unshift → [111, 44, 1, 22]
  • 2: pop 22, unshift → [22, 111, 44, 1]
  • 3: pop 1, unshift → [1, 22, 111, 44]
  • 4: pop 44, unshift → [44, 1, 22, 111] (tilbake til start!)
  • 5: pop 111, unshift → [111, 44, 1, 22]

Vent — det gir A, ikke D. La oss verifisere: etter 4 rotasjoner er vi tilbake til start (siden array-lengden er 4). 5 rotasjoner = 5 mod 4 = 1 rotasjon, som gir [111, 44, 1, 22].

Originaler påstår D er korrekt, men logikken gir A. Dette er en feil/uklar oppgave i originalen. Antagelse: Sensor godtok flere svar. Den faktiske riktige output er [111, 44, 1, 22].

Læringsverdi: Forstå at pop + unshift er én høyre-rotasjon, og at antall rotasjoner mod array-lengde gir nettoeffekten.

Spørsmål 11 (B1) 8 poeng Kap. 1 — TCP/IP

Forklar funksjonaliteten til TCP/IP-lagene og gi eksempler på protokoller som brukes i hvert lag.

Vis fasit

Rubrikk (8 poeng — 2 per lag):

  • Application Layer: Gir applikasjoner tilgang til nettverkstjenester. Protokoller: HTTP (web), SMTP/IMAP (mail), FTP (filoverføring), DNS (navneoppslag), telnet, VoIP.
  • Transport Layer: Ende-til-ende-levering mellom to endepunkter. Protokoller: TCP (pålitelig, ordnet strøm), UDP (rask, "fire-and-forget").
  • Network (Internet) Layer: Ruter pakker basert på IP-adresser fra avsender til mottaker. Protokoller: IP (IPv4/IPv6), ICMP (ping).
  • Physical/Link Layer: Legger bittene på det fysiske mediet og håndterer lokal nettverkstilgang. Protokoller/standarder: Ethernet, WiFi (IEEE 802.11), Bluetooth.

Nøkkelpoeng: Pakken pakkes ned gjennom lagene hos avsender (HTTP-data → TCP-segment → IP-pakke → Ethernet-frame) og pakkes opp i motsatt rekkefølge hos mottaker.

Spørsmål 13 (B3) 6 poeng Kap. 4 — display-verdier

Forklar forskjellen mellom:

  • display: none
  • display: inline
  • display: block
Vis fasit

Rubrikk (6 poeng — 2 per verdi):

  • display: none: Fjerner elementet helt fra layouten — ingen plass tas, ingen rendering. Som om elementet ikke eksisterte. (Kontrast: visibility: hidden beholder plassen.)
  • display: inline: Elementet ligger i samme linje som omgivende tekst. Aksepterer ikke width/height — dimensjoneres etter innholdet. Eksempler: <span>, <a>, <strong>.

Bonus: inline-block kombinerer: oppfører seg som inline utad (på samme linje), men respekterer width/height som block.

Spørsmål 14 (B4) 10 poeng Kap. 10 — Event propagation

Forklar event propagation og event bubbling med eksempler.

Vis fasit

Rubrikk (10 poeng):

  • 2 poeng: Event propagation = "reisen" en event tar gjennom DOM-treet, i tre faser.
  • 2 poeng: Capturing-fasen: window → target (utenfra og inn).
  • 2 poeng: Target-fasen: hendelsen treffer det elementet brukeren faktisk klikket på.
  • 2 poeng: Bubbling-fasen: target → window (innenfra og ut). Default for handlere.
  • 2 poeng: Konkret eksempel + addEventListener(..., true) for capturing og event.stopPropagation() for å stoppe videre flyt.

Eksempel: <div id="ytre"><button id="indre">Klikk</button></div>. Klikk på knappen → event treffer indre (target), bobler så til ytre (bubble). Hvis ytres handler er registrert med tredje argument true, fyrer den i capturing-fasen i stedet — før handleren på indre.

Praktisk bruk: Event delegation — én handler på en forelder kan håndtere klikk på alle barn (også dynamisk lagt til), siden klikket bobler opp.

Vis fasit

const me = document.getElementById("mouseevents");
const boks = document.getElementById("boks");

  me.style.border = "10px solid black";
});

me.addEventListener("mouseout", () => {
  me.style.color = "blue";
});

boks.addEventListener("focus", () => {

});

boks.addEventListener("blur", () => {
  boks.value = "Mistet fokus";

Lag et HTML-skjema med fornavn, etternavn, alder og passord. Skriv JavaScript som validerer:

  1. (2 poeng) Alder må være over 18 år.
  2. (3 poeng) Fornavn og etternavn må ikke inneholde tall.
  3. (4 poeng) Passordet må ha minst én stor bokstav, ett spesialtegn og ett tall.
  4. (1 poeng) Vis valideringsfeilene som alert og hindre at skjemaet sendes hvis noe er ugyldig.
Vis fasit

Referanseløsning:

<!-- HTML -->
<form id="reg">
  <input type="text" id="fornavn" placeholder="Fornavn"><br>
  <input type="text" id="etternavn" placeholder="Etternavn"><br>
  <input type="number" id="alder" placeholder="Alder"><br>
  <input type="password" id="passord" placeholder="Passord"><br>
  <button type="submit">Send</button>
</form>

// JS
document.getElementById("reg").addEventListener("submit", (e) => {
  const fornavn = document.getElementById("fornavn").value;
  const etternavn = document.getElementById("etternavn").value;
  const alder = parseInt(document.getElementById("alder").value, 10);
  const passord = document.getElementById("passord").value;

  if (isNaN(alder) || alder <= 18) {
    alert("Alder må være over 18");
    e.preventDefault(); return;
  }
  if (/\d/.test(fornavn) || /\d/.test(etternavn)) {
    alert("Fornavn og etternavn må ikke inneholde tall");
    e.preventDefault(); return;
  }
  const harStor = /[A-Z]/.test(passord);
  const harTall = /\d/.test(passord);
  const harSpesial = /[^A-Za-z0-9]/.test(passord);
  if (!harStor || !harTall || !harSpesial) {
    alert("Passord må ha minst én stor bokstav, ett tall og ett spesialtegn");
    e.preventDefault(); return;
  }
});

Kommentarer:

  • Regex-bytene: /\d/ matcher ett siffer, /[A-Z]/ matcher én stor bokstav, /[^A-Za-z0-9]/ matcher alt som ikke er alfanumerisk (= spesialtegn).
  • parseInt: Returnerer NaN hvis input ikke er tall — isNaN() fanger det.
  • preventDefault(): Stopper at skjemaet faktisk sendes når valideringen feiler. Uten den ville siden lastet på nytt.
  • "Over 18": Tolkes som > 18, ikke ≥ 18. Sjekk alltid spesifikasjonen ved grensetilfeller.

Vanlige feil: Bruke onsubmit="return validate()" inline (oppgaven krever JS-validering — godtatt, men anbefaler addEventListener). Glemme preventDefault(). For løs regex (f.eks. passord.length > 6 uten å sjekke innhold).

Spørsmål 18 (EKSTRA — erstatter canvas) 10 poeng Kap. 10 — DOM + event delegation

NB: EKSTRA-oppgave. Originalens C4 var en canvas-tegneoppgave (Kap 12 — ute av pensum). Den er erstattet med følgende DOM- og event-oppgave som dekker en mer sentral del av nåværende pensum.

HTML-en under er gitt. Skriv JavaScript som lager en interaktiv todo-liste:

  1. (3 poeng) Når brukeren klikker "Legg til", skal teksten fra input-feltet legges til som en ny <li> nederst i listen, og input-feltet tømmes.
  2. (2 poeng) Hvis input er tomt eller bare whitespace, gjør ingenting.
  3. (3 poeng) Klikk på en <li> (også dynamisk lagt til) skal markere den som "ferdig" — sett text-decoration: line-through.
  4. (2 poeng) Bruk event delegation på <ul> for klikk-håndteringen, ikke individuelle handlere på hver <li>.
Vis fasit

Referanseløsning:

const input = document.getElementById("ny");
const legg = document.getElementById("legg");
const liste = document.getElementById("liste");

legg.addEventListener("click", () => {
  const tekst = input.value.trim();
  if (tekst === "") return;
  const li = document.createElement("li");
  li.textContent = tekst;
  liste.appendChild(li);
  input.value = "";
});

// event delegation — én handler dekker både eksisterende og nye li
liste.addEventListener("click", (e) => {
  if (e.target.tagName === "LI") {
    e.target.style.textDecoration =
      e.target.style.textDecoration === "line-through" ? "" : "line-through";
  }
});

Kommentarer:

  • Event delegation: Vi lytter på <ul> i stedet for hver <li>. Klikket bobler opp, og vi sjekker e.target for å se hva som ble klikket. Slik fungerer det også for nye li-er — ingen ekstra event-handlere trengs.
  • Toggle: Sammenlign nåværende verdi og veksle mellom å sette og fjerne line-through — slik kan brukeren "uferdig"-markere igjen.
  • trim: Fjerner whitespace foran og bak. " ".trim() === "" er sant — fanger både helt tom og kun-whitespace.

Vanlige feil: Feste klikk-handler bare på de eksisterende li-ene (Array.from + forEach) — dynamisk lagt til får ingen handler. Bruke innerHTML += "<li>..." — det rebygger hele listen og fjerner state. Glemme trim() og slippe gjennom bare-whitespace.
