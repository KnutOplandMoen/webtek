Vi ønsker å plassere fem DIVer slik at DIV4 ligger over DIV1 i stable-rekkefølgen. Hvilken CSS-property styrer rekkefølgen i stacken?

  • A x-index
  • B d-index
  • C s-index
  • D z-index
Vis fasit
Riktig svar: D

z-index kontrollerer stable-rekkefølgen (z-aksen / dybden) for posisjonerte elementer. Høyere verdi ligger over lavere verdi.

Hvilket alternativ matcher TCP/IP-lagene riktig med deres funksjon?

Spørsmål 10 (B1) 4 poeng Kap. 8 — POST vs GET

Hva er forskjellen mellom GET og POST som brukes i HTML-skjemaer?

Vis fasit

Rubrikk (4 poeng — 1 poeng per godt punkt, maks 4):

  • Plassering: GET legger parametere i URL-en (etter ?); POST legger dem i request-body.
  • Caching / bookmarks: GET kan caches og bookmarkes (GET er en henteoperasjon, så gjentakelse er trygg); POST caches normalt ikke fordi POST endrer state på serveren.
  • Størrelse: GET har URL-grense (~2 KB i praksis); POST har ingen praktisk grense.
  • Synlighet/sikkerhet: GET vises i URL/history/logger — uegnet for sensitive data; POST skjuler dem i body.
  • Bruksområder: GET for henting/lesing (søk, filtre); POST for å endre serverens tilstand (registrering, opplasting).

Spørsmål 11 (B2) 4 poeng Kap. 4 — display vs visibility

Forklar forskjellen mellom display: block og visibility: hidden.

Vis fasit

Rubrikk (4 poeng):

  • 1 poeng: display: block gjør elementet til et blokk-element (full bredde, linjeskift før og etter). Det er synlig.
  • 1 poeng: visibility: hidden gjør elementet usynlig, men det tar fortsatt plass i layouten — det er som en tom rektangel.
  • 1 poeng: Kontrast: display: none fjerner elementet helt fra layouten (ingen plass) — det er forskjellig fra visibility: hidden.
  • 1 poeng: Det riktige sammenligningsparet er display: none ↔ visibility: hidden. display: block handler om hvordan elementet vises, ikke om det vises.

Spørsmål 12 (B3) 4 poeng Kap. 9 — Array-mutasjon

Når denne websiden lastes, hva blir resultatet på siden?

const arrayOfOddNumbers = [1, 3, 5];
document.write(arrayOfOddNumbers.length);
arrayOfOddNumbers[100] = 199;
document.write(arrayOfOddNumbers.length);
Vis fasit
Riktig svar: 3101

Mekanisme: Første document.write skriver 3 — lengden av [1, 3, 5]. Så settes arrayOfOddNumbers[100] = 199 — JavaScript tillater "sparse" arrays (hull), og length blir max(eksisterende length, indeks + 1). Indeks 100 → length blir 101.

Viktig: Selv om vi har et hull mellom indeks 3 og indeks 100 (verdiene er undefined der), teller alle med i length. Strengene konkateneres på siden, så resultatet er "3" + "101" = "3101".

Vanlige feil: Anta at length bare teller faktisk satte elementer (101 er ikke). Eller anta at arrayOfOddNumbers[100] = ... kaster en feil fordi indeks 100 ikke finnes (det gjør den ikke — arrays vokser dynamisk).

Spørsmål 13 (B4) 8 poeng Kap. 10 — DOM-querying

Forklar forskjellen mellom de følgende, med eksempler:

  • getElementById
  • getElementsByName
  • getElementsByClassName
  • getElementsByTagName
Vis fasit

Rubrikk (8 poeng — 2 per metode):

  • getElementById("x"): Returnerer ett element (eller null) som matcher id="x". IDer skal være unike, så ett element. Eksempel: document.getElementById("header").
  • getElementsByName("x"): Returnerer en NodeList (live-collection) av elementer som matcher attributtet name="x". Brukes ofte for radio-knapper. Eksempel: document.getElementsByName("kjonn").
  • getElementsByClassName("x"): Returnerer en HTMLCollection (live-collection) av elementer med class="x". Iterer med løkke eller spread. Eksempel: document.getElementsByClassName("aktiv").
  • getElementsByTagName("p"): Returnerer en HTMLCollection av alle <p>-elementer på siden. Eksempel: document.getElementsByTagName("li").

Nøkkelpoeng: Den første returnerer ett element direkte; de tre andre returnerer en collection man må iterere. Moderne alternativ: querySelector (ett) og querySelectorAll (statisk NodeList).

Spørsmål 15 (C1) 5 poeng Kap. 9 — Armstrong-tall

Skriv en JavaScript-funksjon som avgjør om et gitt positivt tall er et Armstrong-tall eller ikke. Et n-sifret positivt heltall er et Armstrong-tall hvis summen av hvert siffer opphøyd i n-te potens er lik tallet selv.

Eksempler:

  • 153 = 13 + 53 + 33 = 1 + 125 + 27 = 153 ✓
  • 1634 = 14 + 64 + 34 + 44 = 1 + 1296 + 81 + 256 = 1634 ✓
  1. (2 poeng) Finn antall siffer i tallet.
  2. (2 poeng) Summér hvert siffer opphøyd i n-te potens.
  3. (1 poeng) Sammenlign med originaltallet og returner true eller false.
Vis fasit

Referanseløsning:

function erArmstrong(n) {
  const sifre = String(n).split("");
  const antall = sifre.length;
  let sum = 0;
  for (const s of sifre) {
    sum += Math.pow(parseInt(s, 10), antall);
  }
  return sum === n;
}

document.getElementById("sjekk").addEventListener("click", () => {
  const n = parseInt(document.getElementById("tall").value, 10);
  const svar = erArmstrong(n)
    ? n + " er et Armstrong-tall"
    : n + " er IKKE et Armstrong-tall";
  document.getElementById("utskrift").textContent = svar;
});

Kommentarer:

  • Konvertering til siffer: String(153).split("") gir ["1","5","3"]. Hver streng må parses tilbake til tall med parseInt.
  • Antall siffer: sifre.length gir n. Alternativ: Math.floor(Math.log10(n)) + 1 (men split er enklere).
  • Math.pow vs **: Math.pow(s, n) og s ** n gir samme resultat. Begge er akseptert.

Vanlige feil: Bruke n.length direkte på et tall (tall har ikke length). Glemme parseInt ved summering — strenger blir konkatenerte. Bruke n^antall som er XOR, ikke potens.

Spørsmål 17 (C3) 8 poeng Kap. 9 — Date + primtall

Skriv en JavaScript-funksjon som regner antall dager til neste års jul (ikke i år), sjekker om antallet er et primtall eller ikke, og viser resultatet i en alert. Eksempler:

  • "4 dager igjen, 4 er ikke primtall"
  • "11 dager igjen, 11 er primtall"
  1. (3 poeng) Beregn antall dager til 24/25 desember neste år.
  2. (3 poeng) Skriv en hjelpefunksjon erPrimtall(n).
  3. (2 poeng) Sett dem sammen i en alert med leselig tekst.
Vis fasit

Referanseløsning:

function erPrimtall(n) {
  if (n < 2) return false;
  for (let i = 2; i <= Math.sqrt(n); i++) {
    if (n % i === 0) return false;
  }
  return true;
}

function dagerTilNesteJul() {
  const naa = new Date();
  const nesteAar = naa.getFullYear() + 1;
  const jul = new Date(nesteAar, 11, 25);  // måned 11 = desember
  const ms = jul - naa;
  const dager = Math.ceil(ms / (1000 * 60 * 60 * 24));
  const ord = erPrimtall(dager) ? "primtall" : "ikke primtall";
  alert(dager + " dager igjen, " + dager + " er " + ord);
}

document.getElementById("vis").addEventListener("click", dagerTilNesteJul);

Kommentarer:

  • Måned-index: Desember er måned 11 (0-indeksert). Et vanlig "off-by-one"-snublepunkt.
  • "Neste års": getFullYear() + 1. Hvis du bruker dette årets jul, vil tallet bli negativt etter 25. desember.
  • Primtall-test: Sjekk bare opp til Math.sqrt(n) — faktorer over kvadratroten har alltid en partner under, så vi har sett dem allerede.
  • Math.ceil: Runder opp — bedre enn floor her, så "12.3 dager" blir 13 (ikke 12).

Vanlige feil: Bruke måned 12 for desember (det blir januar neste år). Glemme +1 på året. Inkludere 1 som primtall (det er ikke). Bruke i < n i primtall-loop (treig for store n).
