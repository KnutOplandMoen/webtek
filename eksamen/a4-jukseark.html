Boks-modellen

utenfra → inn:
margin → border → padding → content

  • border-width: 10 5 20 1 = top, right, bottom, left

  • 2 verdier: top/bottom, side

display / visibility

  • none — fjernet, ingen plass
  • block — full bredde, ny linje, kan ha w/h
  • inline — i tekstflyt, ignorerer w/h
  • inline-block — w/h ja, samme linje
  • flex — 1D layout-context
  • grid — 2D layout-context
  • visibility:hidden — usynlig, beholder plass
  • opacity:0 — gjennomsiktig, fanger fortsatt klikk

position

  • static (default) — normal flyt
  • relative — flyttes fra normalposisjon
  • absolute — relativ til nærmeste posisjonerte forelder
  • fixed — relativ til viewport
  • sticky — relative til scroll passerer

z-index virker kun når position ≠ static

Arv (inheritance)

  • Arves: color, font-*, line-height, text-align, visibility
  • Arves ikke: border, margin, padding, width, background
  • Tving arv: inherit

Armstrong-tall (1/3 sett)

function armstrong(n){
  let s = String(n);
  let p = s.length;
  let sum = 0;
  for (let c of s){
    sum += Math.pow(+c, p);
  }
  return sum === n;
}
// armstrong(153) → true
//   1³+5³+3³ = 1+125+27 = 153

Forklaringssvar-mal (TCP/IP)

TCP/IP-modellen organiserer nettverkskommunikasjon i fire lag. Application-laget gir tjenester til programmer (HTTP, DNS). Transport sørger for ende-til-ende-overføring (TCP pålitelig, UDP raskt). Network ruter pakker mellom maskiner (IP). Physical/Link håndterer faktisk signal (Ethernet, WiFi). Data pakkes inn lag for lag på sender og pakkes ut på mottaker.

Forklaringssvar-mal (Bubbling)

Event propagation har tre faser: capturing (fra window ned til target), target-fasen (på selve elementet), bubbling (oppover treet til document). Som standard kjører handlere i bubbling-fasen. Med addEventListener(..., true) kjører den i capture-fasen. e.stopPropagation() stopper videre forplantning. Eksempel: klikk på en knapp inni en div trigger først knapp-handler, så div-handler (med mindre stopProp kalles).
