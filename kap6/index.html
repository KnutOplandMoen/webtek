Responsivt webdesign

Én CSS-fil, mange skjermer. Med @media introduserer du breakpoints og deler stilene som er felles mellom mobil og desktop — DRY-prinsippet i praksis.

Pensum: 1.10 1.10.1 — 1.10.3 Forelesning 12
Eksamensfokus · Kap 6 Lav prioritet

Ikke representert i ex2–ex5

Responsivt webdesign er bra praksis og dukker opp i øvingene, men ingen eksamenssett tester det direkte. Det betyr ikke null sjanse — en MCQ om @media-syntaks kan komme — men det er det eneste realistiske scenariet. Bruk minimalt med tid her, sett heller to-tre linjer på A4-arket.

Hyggelig å kjenne igjen

  • Media query-syntaks: @media (max-width: 768px) { ... }
  • Mobile first = start med små skjermer, legg til regler for større
  • Viewport-meta: <meta name="viewport" content="width=device-width, initial-scale=1.0">
  • Fluide enheter: %, em, rem, vw/vh

Kan hoppes over

  • Standard breakpoints utenat (320, 768, 1024 osv.)
  • picture-elementet og srcset
  • orientation/print-media-queries
  • Container queries

01 · Helhetsbilde

Hvorfor responsivt?

I dag kjører nettsider på alt fra laptop og nettbrett til mobil og phablet. En desktop-layout på en mobilskjerm gir typisk for liten eller for stor font — brukeropplevelsen lider. Tidligere ble dette løst ved å publisere flere stylesheets eller flere versjoner av siden for små skjermer, men det bryter med don't repeat yourself: identisk styling havnet i flere filer som måtte vedlikeholdes parallelt.

CSS3 introduserte media queries. Med dem kan vi sette breakpoints i én og samme CSS-fil og dele kode som er felles mellom mobil og desktop, mens regler som er spesifikke for et gitt skjermområde står inne i en @media-blokk.

Kilde

Pensum 1.10 (Responsive web design).

02 · Mental modell

Anatomien til en media query

En media query starter med @media, etterfulgt av et uttrykk som evalueres til true eller false. Inne i blokken legger du vanlige CSS-regler. Reglene anvendes kun når uttrykket er sant for nettleseren akkurat nå. 

p {
  font-size: 22px;
}

@media (max-width: 600px) {
  p {
    font-size: 14px;
  }
}

I eksempelet over er standardstørrelsen 22 px, men når nettleservinduet er 600 px eller smalere overstyres verdien til 14 px. max-width: 600px er en media feature — én av flere uttrykk som kan stå i en spørring.

  1. Start med nøkkelordet @media.
  2. Legg til null eller flere uttrykk (media types og/eller media features). Alle må evaluere til true for at blokken skal anvendes.
  3. Skriv CSS-regler inne i blokkens krøllparenteser. Disse følger normal kaskade når de er aktive.
03 · Reisen

Evaluering, kaskade og kobling

Pensum understreker fire egenskaper ved hvordan media queries oppfører seg:

true / false
Hele uttrykket må resolvere til true før blokken anvendes.
Null eller flere uttrykk
En spørring kan ha flere features kombinert med and — alle må være sanne.
Normal kaskade
Når en blokk er aktiv, deltar reglene i den vanlige spesifisitets- og rekkefølge-kaskaden.
Case insensitive
Store og små bokstaver i selve spørringen behandles likt.
Ukjent media type
Spørringer med ukjent media type evalueres alltid til false.

Du kan også koble en media type direkte på et stylesheet via media-attributtet på <link>: 

<link rel="stylesheet" media="screen" href="/css/master.css">

Det samme settet av media types som brukes i @media-blokker gjelder her.

04 · Hovedtemaer

De tre delene

A · TYP
Media types
all, screen, print. Bestemmer hvilken type enhet stilen gjelder for. Pensum 1.10.1.
B · FTR
Media features
width, height, resolution, orientation. De tre første tar min-/max--prefiks. Pensum 1.10.2.
C · LOG
Logiske operatorer
and, not, only. Kombinerer og endrer betydningen av en spørring. Pensum 1.10.3.

A · Media types (1.10.1)

Tre media types er pensum:

all
Passer for alle enheter — anvendes både på skjerm og print.
screen
For dataskjermer (laptop, mobil, nettbrett).
print
For utskrifter og dokumenter vist i print preview.

De fleste sider vises på skjerm, så det er ikke alltid nødvendig å eksplisitt skrive screen, men når en stil kun skal gjelde for skjerm kan du avgrense slik: 

@media screen {
  body {
    background-color: #FFFFFF;
  }
}

Når noen stiler vil ødelegge utskrift (f.eks. en navbar du ikke vil ha med på papir) bruker du print: 

@media print {
  #nav-area { display: none; }
}

Bruker du all gjelder stilen både på skjerm og print — effektivt det samme som å droppe media type: 

@media all {
  body {
    font-size: 22px;
  }
}

B · Media features (1.10.2)

width, height og resolution kan ha min- og max- som prefiks for å uttrykke henholdsvis nedre og øvre grense. orientation tar verdiene portrait eller landscape. 

/* Eksakt bredde */
@media (width: 300px) { /* … */ }

/* Bredde minst */
@media (min-width: 600px) { /* … */ }

/* Bredde høyst */
@media (max-width: 1200px) { /* … */ }

/* Eksakt høyde */
@media (height: 300px) { /* … */ }

/* Høyde minst */
@media (min-height: 600px) { /* … */ }

/* Høyde høyst */
@media (max-height: 800px) { /* … */ }

/* Print, høy oppløsning */
@media print and (min-resolution: 300dpi) { /* … */ }

/* Orientering */
@media (orientation: portrait) { /* … */ }

Verdier for bredde og høyde må alltid skrives som tall pluss en lengdeenhet (px, em).

C · Logiske operatorer (1.10.3)

and
Kombinerer flere features. Hver del må være true.
not
Negerer hele spørringen — kan ikke negere én enkelt feature.
only
Hindrer at eldre nettlesere uten støtte for media features anvender stilen.
/* and: begge må være sanne */
@media (max-width: 600px) and (orientation: landscape) { /* … */ }

/* not: gjelder hele spørringen og evalueres SIST */
@media not all and (monochrome) { /* … */ }
/* tolkes som: */
@media not (all and (monochrome)) { /* … */ }
/* IKKE som: */
@media (not all) and (monochrome) { /* … */ }

/* only: skjuler stilen for eldre nettlesere */
@media only screen { /* … */ }

Nøkkelbegreper

min-/max-prefiks

portrait / landscape

and

only
Skjuler stilen for nettlesere uten media-feature-støtte.
media-attributt
Attributt på <link>-tag som tar samme media types som @media.

08 · Vanlige feller

Vanlige feller

Pensum 1.10
  • Glemme enhet på width/height. Pensum krever et tall pluss en lengdeenhet (px, em). @media (min-width: 600) er ugyldig.
  • Tro at not kan negere én feature. not gjelder hele spørringen — den evalueres til slutt.
  • Tro at en spørring må ha en media type. Den kan ha null eller flere uttrykk; @media (min-width: 600px) alene er gyldig.
  • Bruke screen der det egentlig skal være print. Hvis stilen kun skal gjelde for utskrift, må media type være print.
  • Skrive ukjente media types. Pensum sier eksplisitt at en query med ukjent media type alltid evalueres til false — stilen vil aldri anvendes.
09 · Eksamensscenario

Skriv en media query

Oppgave

En p har font-size: 22px som standard. Skriv en CSS-regel som setter font-størrelsen til 14px når nettleservinduet er 600 px eller smalere.

Løsning
@media (max-width: 600px) {
  p {
    font-size: 14px;
  }
}

Hentet rett fra pensum 1.10. Vær obs på at enheten px er obligatorisk i media-feature-verdier.

Variant

«Skriv en spørring som kun anvendes på skjermer i landscape som er høyst 600 px brede.» — kombiner and med en feature og en orientation: 

@media screen and (max-width: 600px) and (orientation: landscape) {
  /* … */
}
10 · Test deg selv

Helhetsspørsmål

Spørsmål direkte fra pensum 1.10.

1 · Lett
Hvilket nøkkelord starter en media query?
  • @query
  • @media
  • @responsive
  • @screen
Pensum 1.10: «You write a media query as such, in your CSS document» — alltid med @media.
2 · Lett
Hvilke tre media types skal du kunne fra pensum?
  • screen, mobile, desktop
  • tv, screen, print
  • all, screen, print
  • device, screen, paper
Pensum 1.10.1 lister eksplisitt all, screen og print.
3 · Middels
Hvilke media features kan ha både min- og max- som prefiks?
  • orientation og width
  • width, height og resolution
  • kun width og height
  • alle media features
Pensum 1.10.2: «With the features below height, width and resolution can be used with the min- and max- prefixes.»
4 · Middels
Hva evalueres @media not all and (monochrome) til?
  • not (all and (monochrome)) — not gjelder hele spørringen
  • (not all) and (monochrome) — not gjelder kun «all»
  • syntaksfeil — not kan ikke kombineres med and
  • alltid true
Pensum 1.10.3: «the not is evaluated last in the following query». Hele spørringen blir negert som en helhet.
5 · Middels
Hva er hovedmotivasjonen for media queries i følge pensum?
  • Bedre SEO på mobile søk
  • DRY — unngå å duplisere stiler i flere stylesheets
  • Raskere lasting på mobil
  • Krav i WCAG-standarden
Pensum 1.10: tidligere ble dette løst med separate stilark, noe som «obviously defy the DRY principle».
6 · Vanskelig
Hva er only-nøkkelordet til for?
  • Bare anvende stilen én gang
  • Begrense til én bestemt enhet
  • Hindre eldre nettlesere uten støtte for media features fra å anvende stilen
  • Sørge for at kun den ene blokken kjører selv om flere matcher
Pensum 1.10.3: «The only keyword prevents older browsers that do not support media queries with media features from applying the given styles.»
7 · Reveal
Hva skjer hvis en media query refererer til en ukjent media type?
Spørringen evalueres alltid til false, og stilen anvendes derfor aldri. Pensum 1.10: «Media queries involving unknown media types are always false.»
8 · Reveal
Skriv en media query som anvender en stil bare når viewport er minst 600 px bred OG i landscape.
@media (min-width: 600px) and (orientation: landscape) {
  /* … */
}
Begge features må evaluere til true (jf. and-operatoren, pensum 1.10.3).
